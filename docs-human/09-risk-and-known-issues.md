# 风险与已知问题

## 数据缺口

| 问题 | 影响 | 缓解方案 |
|---|---|---|
| 非东京/大阪城市数据稀疏 | 北海道、冲绳等城市的餐厅/景点数据不足 | Claude AI 生成补全（force_ai=true） |
| Tabelog 反爬加强 | 餐厅评分数据采集困难 | 降级到 SerpAPI 搜索提取 |
| Google Places 额度限制 | 批量采集时容易触顶 | OSM + AI 生成作为备选 |
| 酒店实时价格获取困难 | 当前仅存参考价，无法实时报价 | 标注为"预算参考"，不做精确承诺 |

## 图片治理

| 问题 | 当前状态 | 计划 |
|---|---|---|
| 无自有图片库 | 依赖 Unsplash/Google 外链 | 逐步积累自拍+授权图 |
| 图片版权风险 | 未做系统化版权审查 | 建立审核流程，优先使用 CC0 |
| CDN 缺失 | 图片直接从外部 URL 加载 | 接入 Cloudflare R2 / Vercel Blob |
| 图片质量不一 | 部分实体无高质量封面图 | 编辑标注系统中增加图片审核 |

## 翻译覆盖

| 问题 | 影响 | 状态 |
|---|---|---|
| 部分实体缺少中文名 | 用户看到日文名看不懂 | 待完成 batch_translate.py |
| 翻译质量不稳定 | 机器翻译的地名/菜名偶有错误 | 人工审核种子数据中的翻译 |
| 实体描述只有日文 | 文案润色时需要 AI 理解日文 | AI 直接用日文生成中文文案 |

## 技术债务

| 问题 | 优先级 | 说明 |
|---|---|---|
| 交付页使用 mock 数据 | 高 | `plan/[id]/page.tsx` 中 PLAN 是硬编码，需对接真实 API |
| 无自动化测试 CI | 中 | 测试文件存在（tests/）但未接入 CI |
| 日志无结构化 | 中 | 使用 print 和 logging，待迁移到 structlog |
| 重试策略手写 | 低 | 待迁移到 tenacity |
| .env 可能曾被 commit | 高 | 需检查 git 历史并清理 |
| 无 rate limiting | 中 | API 无请求频率限制 |

## 业务风险

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| AI 生成内容质量不稳定 | 高 | 用户体验差 | 人工审核环节 + 护栏检查 |
| 竞品跟进（AI 旅行规划） | 中 | 市场竞争加剧 | 持续优化数据质量 + 本地化深度 |
| 微信封号风险 | 低 | 私域流量中断 | 多账号备份 + 公众号沉淀 |
| 日本旅行政策变化 | 低 | 签证/入境规则变化 | 系统内置政策更新提醒 |
