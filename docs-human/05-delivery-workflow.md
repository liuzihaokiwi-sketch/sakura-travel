# 交付流程

## 完整用户旅程

### 1. 首页（`web/app/page.tsx`）

用户通过小红书/搜索引擎访问首页，看到转化漏斗式的 10 个模块：
Hero → 痛点共鸣 → 解决方案 → 免费预览 → ¥248 主推 → ¥888 锚点 → 攻略展示 → 信任背书 → FAQ → 最终 CTA

核心 CTA：「🆓 先免费看一天」→ 跳转问卷。

### 2. 问卷（`web/app/quiz/page.tsx`）

5 步渐进式问卷，每步一个问题：
1. 去哪里（单选：东京/大阪+京都/北海道/冲绳/其他）
2. 几天（单选：3-8+天）
3. 和谁（单选：独行/情侣/带娃/带父母/朋友）
4. 旅行风格（多选，最多3个，可跳过）
5. 微信号（文本输入）

提交后 POST `/quiz` → 后端创建 `TripRequest` + 发送企业微信通知。

### 3. 提交成功（`web/app/submitted/page.tsx`）

显示"已收到你的需求"+ 三步引导：
1. 添加规划师微信（显示微信号 + 二维码）
2. 2 小时内收到第一天行程
3. 满意再决定

### 4. 微信私域沟通

规划师通过企业微信收到通知后：
- 添加用户微信
- 确认需求细节
- 触发行程生成
- 发送免费预览链接

### 5. 免费预览（`/plan/[id]?mode=preview`）

用户看到 Day 1 完整行程（逐时段路线 + 拍照提示 + 避坑提醒），剩余天数模糊锁定。
锁定区域显示：「还有 X 天精彩内容，¥248 解锁完整方案」+ 微信号复制按钮。

### 6. 付费 → 完整交付（`/plan/[id]`）

用户付费后，规划师在管理后台审核方案，通过后将链接发给用户。完整版包含：
- 全部天数行程
- 住宿建议（区域 + 理由 + 预算）
- 交通方案
- 出行准备清单（可勾选）
- 精调入口 + 升级管家版入口

### 7. 精调与升级

- ¥248 包含 2 次精调：用户提出修改需求 → 规划师在后台调整 → 重新生成受影响天数
- ¥888 升级：不限次精调 + 出行期间实时答疑

## 订单状态流转

```
pending → paid → processing → review → delivered
                                ↑         ↓
                           generating ← (rejected, 打回重做)
                                         ↓
                                      refunded (退款)
```
