# 03. 旅行注意事项模块设计

> 面向：AI 程序员 / 内容装配 / QA / 运营  
> 目标：把“旅行注意事项”从散乱的补充说明升级为**静态底座 + 动态风险模块**体系。  
> 定位：回答“旅途中要特别注意什么，以及哪些风险会因为目的地、天气、活动或用户画像而升级”。

---

## 1. 模块目标

旅行注意事项不应该只是“再补几条提醒”。

它的职责是：

1. **提前暴露风险**：把真正会影响安全、健康、执行、费用的风险讲清楚。
2. **按触发条件展示**：不是所有用户都看同一套提醒。
3. **能和行程挂钩**：哪一天、哪项活动触发了哪条注意事项，应可追踪。
4. **能动态更新**：天气、健康通知、交通和活动风险变化时，模块能替换或降级。

因此本模块要拆成两层：

- **通用注意事项**：大多数行程都成立
- **动态注意事项模块**：由活动 / 目的地 / 季节 / 用户画像触发

---

## 2. 输出形式

建议输出为：

```ts
interface CautionModule {
  caution_id: string;
  title: string;
  category: string;
  severity: "info" | "watch" | "important" | "critical";
  trigger_type: "global" | "destination" | "activity" | "season" | "profile" | "fact";
  trigger_reason: string[];
  applies_to_day_indexes?: number[];
  body_items: CautionItem[];
  action_items?: string[];
  source_snapshot_ids?: string[];
}

interface CautionItem {
  item_id: string;
  label: string;
  body: string;
  risk_if_ignored?: string;
}
```

程序展示规则：

- 通用模块进入总纲的“总注意事项”
- 动态模块进入总纲摘要 + 对应天的 `applied_caution_ids`
- 同一模块不重复展示长文；总纲展示摘要，日页展示与当天相关的执行型提醒

---

## 3. 通用注意事项底座

以下 8 类建议作为默认底座。

### 3.1 证件、法律与当地规则

应提醒：

- 证件与复印件分开放
- 了解当地法律、海关与禁带限制
- 不要默认国外规则与国内相同
- 转机地法律也可能适用

适用：

- 所有跨境行程

### 3.2 食品与饮水

应提醒：

- 洗手或用含酒精免洗消毒产品
- 谨慎选择生食、未密封饮料、冰块、路边高风险食物
- 容易肠胃不适时准备补液与常用药

适用：

- 所有旅行，尤其跨境和高温地区

### 3.3 夜间与财物安全

应提醒：

- 夜间尽量走明亮、人多、交通明确的路线
- 手机、护照、钱包不要集中放一处
- 使用正规出租车/网约车
- 醉酒状态下减少单独移动

适用：

- 所有城市旅行，独行 / 夜间抵达 / 夜生活场景优先

### 3.4 交通安全

应提醒：

- 系安全带
- 过马路遵守当地方向和规则
- 骑行 / 摩托 / 滑板等佩戴头盔
- 谨慎使用不熟悉的交通工具

适用：

- 所有旅行

### 3.5 天气与温差

应提醒：

- 山区、海边、沙漠、冬季景点温差更大
- 强日晒、高温、暴雨、寒潮都会影响行程和健康
- 出发前和当日都应查天气

适用：

- 所有旅行

### 3.6 医疗与紧急求助

应提醒：

- 记录急救、报警、保险救援、使领馆联系方式
- 若发生严重过敏、胸痛、呼吸困难、明显脱水、高反加重，应尽快就医
- 高风险地区需提前确认医疗可及性

适用：

- 所有旅行

### 3.7 虫咬与动物接触

应提醒：

- 不要随意喂食、抚摸动物
- 在有蚊虫或蜱虫风险的区域使用驱虫剂和遮盖衣物
- 被咬伤、抓伤或被动物舔到破损皮肤后尽快就医评估

适用：

- 户外、热带、郊野、海岛、动物园、农庄等场景

### 3.8 保险与费用风险

应提醒：

- 确认保险覆盖医疗、转运、取消改签、财物、计划活动
- 自驾、滑雪、徒步、潜水等高风险活动常需额外覆盖
- 出险材料、报警记录、票据要及时保存

适用：

- 所有跨境行程和高风险活动行程

## 4. 动态注意事项模块总览

建议首批做 12 个动态模块。

1. 徒步 / 爬山 / 山区徒步
2. 高海拔 / 高反
3. 自驾 / 长途驾驶
4. 水上活动 / 海边 / 游泳 / 浮潜 / 潜水
5. 冬季 / 滑雪 / 冰雪路面
6. 高温 / 热浪 / 沙漠 / 强紫外线
7. 虫媒 / 蚊虫 / 蜱虫
8. 动物接触 / 野生动物 / 猴子 / 狗猫 / 蝙蝠
9. 大型活动 / 演唱会 / 节庆 / 人群聚集
10. 远郊 / 无信号 / 医疗资源弱
11. 夜生活 / 饮酒 / 深夜回程
12. 画像触发模块（儿童 / 老人 / 孕期 / 独行）

---

## 5. 动态模块详细设计

### 5.1 徒步 / 爬山 / 山区徒步模块

#### 触发条件

- 行程包含 hike / trek / mountain / trail / summit / national park / canyon 等标签
- 步行强度高且离城市急救资源远
- 日出登山、夜间下撤、长距离徒步

#### 核心提醒

- 提前查看路线长度、爬升、预计时长、天气、日落时间、封闭信息
- 按 NPS “Ten Essentials” 思路准备：导航、补水、食物、保暖/防雨、头灯、急救、刀具/修理、防晒、应急火种/照明、额外衣物等
- 穿合适的鞋袜，不要用新鞋直接上山
- 山里温度、风和降雨变化可能快于城市
- 单人徒步或无信号区域要共享路线和预计返回时间
- 不要为了赶行程硬冲最后一个点，回撤时间要前置

#### 程序备注

- 默认 severity = important
- 若无信号 / 夜间 / 高海拔叠加，则升为 critical

### 5.2 高海拔 / 高反模块

#### 触发条件

- 睡眠海拔或活动海拔显著升高
- 目的地包含高海拔城市、山口、雪山、徒步营地

#### 核心提醒

- 高海拔环境的核心问题是缺氧；上升过快会增加高反风险
- 前 24–48 小时避免剧烈运动和过量饮酒
- 尽量采用缓升策略，必要时“白天上升、低处睡眠”
- 出现持续头痛、恶心、乏力、呼吸困难、意识异常等加重症状应尽快下撤并就医
- 高海拔同时伴随更强紫外线、寒冷和低湿度

#### 程序备注

- 默认 severity = important
- 若当天同时有长徒步 / 山顶住宿 / 飞抵高海拔，则升为 critical

### 5.3 自驾 / 长途驾驶模块

#### 触发条件

- 行程包含 car rental / drive / road trip / long transfer
- 目的地公共交通弱，需自己开车

#### 核心提醒

- 出发前确认驾照、IDP/翻译件、当地保险、取还车规则
- 提前了解当地交通方向、限速、停车、收费、公路救援、儿童座椅规则
- 系安全带，避免疲劳驾驶、酒驾、夜间陌生山路驾驶
- 山路、冰雪、乡村道路、野生动物穿越路段要额外谨慎
- 车辆事故、剐蹭、爆胎时，先保证人身安全，再按租车和保险流程留证

#### 程序备注

- 默认 severity = important
- 若叠加冰雪 / 山路 / 夜驾 / 多驾驶员，则升为 critical

### 5.4 水上活动 / 海边 / 游泳 / 浮潜 / 潜水模块

#### 触发条件

- 行程包含 beach / swim / snorkel / dive / boat / kayak / rafting / ferry

#### 核心提醒

- 尽量选择有救生员的区域，下水前看水流、风浪、潮汐、天气
- 不熟悉水况时不要单独下水，不要高估体能
- 酒后避免游泳、驾船、深水活动
- 潜水要遵守专业机构和教练规则，并注意潜水后的禁飞时间
- 冷水、离岸流、天气突变都可能快速升级风险

#### 程序备注

- 默认 severity = watch
- 若是深潜、远海、无救援设施或儿童同行，升为 important / critical

### 5.5 冬季 / 滑雪 / 冰雪路面模块

#### 触发条件

- 行程处于雪季 / 冰雪道路 / 滑雪场 / 冬季山地

#### 核心提醒

- 冬季风险不仅是冷，还包括滑倒、能见度低、冻伤、失温和交通延误
- 冰雪环境下帽子、手套、防滑鞋、保暖层、防水层要齐
- 滑雪/雪板/雪地活动优先考虑头盔与附加保险
- 冰面、结冰步道、未开放雪道不要擅自进入
- 自驾时要确认雪胎、链条、道路开放情况

#### 程序备注

- 默认 severity = important

### 5.6 高温 / 热浪 / 沙漠 / 强紫外线模块

#### 触发条件

- 高温季、沙漠、海边暴晒、长时间户外步行、城市热岛

#### 核心提醒

- 提前补水，避免等口渴再喝
- 把高强度户外活动尽量排到早晚
- 使用防晒、遮阳帽、太阳镜和透气衣物
- 热痉挛、热衰竭、意识模糊、停止出汗等都是危险信号
- 雪地和高海拔同样有强紫外线

#### 程序备注

- 默认 severity = watch
- 遇正式热浪、沙漠长徒步、老人儿童同行时升为 important

### 5.7 虫媒 / 蚊虫 / 蜱虫模块

#### 触发条件

- 热带、雨林、湿地、乡村、露营、徒步、高草地
- 目的地有相关健康提示

#### 核心提醒

- 使用合规驱虫剂，穿长袖长裤，住宿尽量选有纱窗/空调或蚊帐的地方
- 先涂防晒，再涂驱虫剂
- 蜱虫风险区域尽量避免高草和灌木，活动后检查皮肤与衣物
- 旅后出现发热、皮疹、关节痛等症状时要告知医生旅行史

#### 程序备注

- 默认 severity = watch
- 若目的地存在明确虫媒病健康通知，则升为 important

### 5.8 动物接触 / 野生动物模块

#### 触发条件

- 野生动物园、寺庙猴群、狗猫密集地区、洞穴、农场、野外露营

#### 核心提醒

- 不要摸、喂、逗、追动物，即便看起来温顺也不例外
- 被咬、抓、舔到破损皮肤后尽快就医评估是否需要狂犬病暴露后处置
- 不要徒手接触蝙蝠、猴子、流浪狗、野生哺乳动物
- 儿童需要额外提醒，避免因好奇靠近动物

#### 程序备注

- 默认 severity = important

### 5.9 大型活动 / 节庆 / 人群聚集模块

#### 触发条件

- 演唱会、体育赛事、烟花大会、宗教活动、节庆、跨年、樱花高峰等

#### 核心提醒

- 提前确认入场规则、实名规则、安检限制、集合点和散场交通
- 高峰时段信号差、拥挤、交通绕行和临时封路常见
- 同行人约好失联后的汇合点
- 贵重物品从简，注意踩踏和推挤风险
- 根据官方健康或安全提示调整行程

#### 程序备注

- 默认 severity = watch
- 若为国际大型活动、健康通知期或深夜散场，则升为 important

### 5.10 远郊 / 无信号 / 医疗资源弱模块

#### 触发条件

- 远离城市、离岛、深山、沙漠、边境、长线公路

#### 核心提醒

- 事先下载离线地图和预订信息
- 告知他人当天路线和预计返回时间
- 补水、食物、保暖、应急电源、急救包冗余要更高
- 了解最近的加油、补给、医院、报警与救援方式
- 天气突变、车辆故障、迷路时要优先保命而不是赶进度

#### 程序备注

- 默认 severity = important

### 5.11 夜生活 / 饮酒 / 深夜回程模块

#### 触发条件

- 酒吧、夜店、演唱会散场、深夜餐酒、凌晨航班/返程

#### 核心提醒

- 控制饮酒量，饮品不离手，不接受来源不明的酒水
- 规划好正规回程交通和备用方案
- 不单独走偏僻路线，不与陌生人拼车或临时换场
- 重要证件与主支付工具不要全带出门

#### 程序备注

- 独行女性、陌生城市、语言障碍场景下优先级提高

### 5.12 画像触发模块

#### 儿童同行

- 药物、补水、防晒、防蚊、推车/背带、儿童安全座椅、走失预案

#### 老人同行

- 慢病药、休息频率、台阶与步行距离、医疗可及性、低温/高温额外防护

#### 孕期

- 飞行与活动限制、医疗点位、保险、药物、强烈刺激活动避让

#### 独行

- 共享定位、夜间回程、紧急联系人、现金冗余、避免过度饮酒

## 6. 触发器设计

建议把触发器拆成 5 类。

### 6.1 `destination_trigger`

由国家 / 地区 / 城市决定：

- 法律与限制
- 健康风险
- 旅行警示
- 自然环境

### 6.2 `activity_trigger`

由活动标签决定：

- hiking
- altitude
- driving
- swimming
- diving
- skiing
- mass_event
- nightlife

### 6.3 `season_trigger`

由出行时间决定：

- 夏季高温
- 冬季冰雪
- 雨季 / 台风 / 飓风 / 山火窗口
- 花粉 / 过敏季

### 6.4 `profile_trigger`

由用户画像决定：

- child
- senior
- pregnancy
- solo
- chronic_meds

### 6.5 `fact_trigger`

由动态事实决定：

- 旅行警示升级
- 健康通知
- 极端天气
- 交通中断
- 活动组织方规则变更

---

## 7. 展示策略

### 总纲层

只展示：

- 最高优先级的 4–8 条总提醒
- 每条不超过一屏摘要

### 日页层

只展示：

- 与当天直接相关的注意事项
- 例如 Day 4 有登山，就显示徒步 / 高海拔模块摘要

### 条件页层

对于复杂活动，可追加“活动注意页”，但只在下列情况触发：

- 风险高
- 费用高
- 不熟悉度高
- 一旦失败会明显影响全程

---

## 8. 文案要求

### 8.1 不要写成恐吓文本

注意事项不是制造焦虑，而是帮助用户做正确准备。

### 8.2 一律写成“风险 + 行动”

不写：

- 注意高反
- 注意虫咬
- 注意别中暑

要写：

- 高海拔地区上升过快会增加高反风险，前 24–48 小时尽量避免剧烈运动和过量饮酒，如症状持续加重请尽快下撤并就医。
- 在有蚊虫风险的区域请使用合规驱虫剂、穿长袖长裤，并优先选择有纱窗或空调的住宿。
- 高温和强紫外线会显著提高脱水和晒伤风险，户外强度尽量安排在早晚，并提前补水与做好防晒。

### 8.3 能动态就不要假通用

例如：

- 不是所有海边都能安全下水
- 不是所有山都适合临时起意徒步
- 不是所有国家都接受原驾照
- 不是所有目的地都默认有稳定网络和医疗资源

---

## 9. 质量门控

### 必须通过的规则

1. 任何高风险活动都必须有对应动态模块。
2. 动态模块必须记录触发原因。
3. 同一风险不应在总纲和日页重复输出长文。
4. 来自动态事实的提示必须有来源快照和更新时间。
5. 若事实失效，自动降级为中性提示，避免继续输出过期结论。

---

## 10. 建议接入的信息源类型

### 官方健康与安全源

- CDC Travelers’ Health / CDC Yellow Book
- WHO travel health guidance
- 外交 / 使领馆旅行建议
- 国家公园、景区、气象、海事、道路官方渠道

### 活动与交通官方源

- 景区 / 活动 / 赛事 / 演唱会主办方
- 航空、铁路、公交、渡轮、缆车、租车公司

### 内部事实源

- 订单和预订信息
- 行程标签
- 气象 / 交通 / 健康通知快照

---

## 11. 一句话实现原则

> **旅行注意事项不再是“通用提醒拼盘”，而是按活动、目的地、季节、画像和动态事实触发的风险模块系统；用户只看到和自己真的相关的那部分。**

---

## 12. 参考信息源（用于程序接入与内容校验）

- CDC Travelers’ Health / CDC Yellow Book — food & water, road safety, heat/cold, adventure travel, high altitude, insects, animals, swimming & diving, travel health notices
- WHO — travel advice, ultraviolet radiation, road safety, travel and health
- Travel.State.Gov — laws abroad, travel advisories, driving and transportation, insurance, destination guidance
- National Park Service — Ten Essentials / hiking safety / water safety / wilderness basics
- GOV.UK — foreign travel advice / insurance / crisis abroad
