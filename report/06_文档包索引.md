# 最终攻略页型设计文档包索引 v2

## 这包文档是干什么的
这套文档只解决一件事：把“最终攻略如何按页设计、每一页放什么、哪些页必须独立、怎么让用户感到被认真定制”定成可以直接给前端 / 程序 AI 落地的方案。

它不讨论品牌文案，不讨论后台评分细节，也不讨论视觉稿像哪种风格。
它只讨论：
- 页型系统
- 页面组合框架
- 每种页型的用途
- 每种页型该吃哪些数据
- 页面内部的排版顺序
- 哪些地方必须做出设计感
- 实现顺序与依赖
- 验收与导出检查

## 文件结构
1. `01_页型总表与整本框架.md`
   - 整本攻略有哪些页
   - 不同天数如何组合
   - 固定页、章节页、日页、专题页的关系

2. `02_前置与总纲页型详细说明.md`
   - 封面
   - 目录
   - 偏好兑现页
   - 主要活动总表
   - 大路线总览
   - 酒店策略总览
   - 预约页
   - 出发前指引
   - 动态注意事项

3. `03_章节_每日_专题页型详细说明.md`
   - 城市圈章节页
   - 每日执行页
   - 主要活动页
   - 酒店详情页
   - 餐厅详情页
   - 拍摄主题页
   - 交通页
   - 顺路补充页
   - 调整 / Plan B 页

4. `04_页面数据协议与实现顺序.md`
   - 每种页型需要的 page view model
   - 与现有 payload 的映射
   - 哪些可以并行做
   - 哪些必须按顺序做

5. `05_设计感与重点突出规则_验收清单.md`
   - 怎么通过排版而不是堆字给情绪价值
   - 每页重点如何被看见
   - 哪些地方必须有“设计感”
   - PDF / H5 最终验收清单

## 推荐阅读顺序
1. 先看 `01`
2. 再看 `02`
3. 再看 `03`
4. 工程直接接 `04`
5. 设计 / 前端一起过 `05`

## 最重要的原则
- 不是一段文字一页，而是一类内容一种页型
- 一页只承担一种阅读任务
- 短行程靠逐日展开，长行程靠章节策展
- 主要活动 / 主要酒店 / 主要餐厅 / 主要拍摄主题必须有单独的高价值页
- 次要内容可以合并，但不能混乱
- 情绪价值主要靠：层次、取舍说明、亮点呈现、偏好兑现、视觉重点
