## 1. 产品哲学（[01_项目定位/项目是什么.md](docs/01_项目定位/项目是什么.md)）

付费旅行手账本（¥298 国内/¥348 国外），卖的不是信息是「**有人替我想过 + 敢对结果负责的确定感**」。用户付费背后四件事 AI 做不到：默会知识、伦理分流、情绪留白、关系编织——这就是产品对标点。三条不可动摇定位:① 卖策展不卖工具（一套主线 × 微调，不是从零生成）② 质量标准是「真专家会不会点头」不是「跑通不报错」③ 付费手账本 ≠ 攻略扩写（要"线索+画面感+本地视角"，不要信息罗列）。体验设计 9 原则中关键三条:**70/20/10 内容比例**（10% 惊喜是用户真正记住的部分）、**B 选项要真的"惊"**（用户能发给朋友炫耀）、**给线索不给结论**（让用户自己走到那里得出结论）。绝对红线:数据真实性（推断 ≠ 事实）、不打补丁绕问题、不硬编码业务数据、字段变更先改 SCHEMA、不假装修了。产品边界硬规:< 10% 特殊用户（二刷/奢华/已订酒店）走客服定制不进主产品。

## 2. 系统哲学（[03_数据契约/SCHEMA.md](docs/03_数据契约/SCHEMA.md) §设计宪法 + §第一性原理）

两条永久宪法:**第一条「能用自然语言文字的，绝不用字段」**——字段唯一意义是给系统硬判断/确定的等级标签；任何"看起来像信息但本质是描述/判断"的东西全部写自然语言段落。**第二条「内容归属对象，谁的事谁解决」**——景点相关（拍照/冷知识/避坑）归 entity.notes；餐厅相关归餐厅；装配规则（打分/A-B/升档）归对应装配 markdown；只有真正"模板设计才能解决的"（动线/peak-end/砍减）才进模板。字段扩展硬规三问:消费方是谁、能否用现有字段表达、是不是装配层的事——任一不通过即不加。四层文件格式分工:模板/景点用 JSON（渲染代码消费）、装配规则用 markdown（AI 像读 SOP 一样消费）。四层职责硬规:entity 写事实权威、slot.note 写动线即时判断（可引用事实但不复述）、整天 note 写骨架逻辑、curators_notes 写印刷级巧思——同一信息只在一层存在，错位就挪走。**字段变更必须先改 SCHEMA**，禁止反向。

## 3. 设计演进逻辑（DECISIONS D36/D40/D42/D43）

四次连续重构都在做同一件事:**字段越少，归属越清晰，AI 装配越自然**。**D36（4-22）字段大瘦身 + 装配层 markdown 化**——模板 30+→7+3，事实层 25+→8+7，景点附属内容（拍照/冷知识/小店）从模板挪到 entity，装配规则全部独立到 markdown，订下两条永久宪法。**D40（4-24）装配架构四层化**——新增 plans/ 预制方案层，废除 1400 行老花名册，模板字段再砍 11+ 个（label/description/curators_notes/min_days/打分三件套等），variant_label 改诗意化（不是必填，只在灵魂特别时写）。**D42（4-25）节奏字段迁方案层**——pace_type/pace_type_sub/time_sensitivity 四字段从模板弃用，改由装配引擎读三信号推断（plans 白名单 + 目录位置 + slots 第一段时间），时间约束改写 slot.note 自然语言。**D43（4-25）entity 大瘦身**——36+ 字段 → 21 key（系统 6 + note 5 + templates_meta 7 + 元数据 3），三名合并为 note.店名，所有营业/票价/交通改自然语言。演进精髓:从「字段表达一切」走向「自然语言 + 最小硬字段」。

## 4. 关西特定哲学（[japan/kansai/产品原则.md](japan/kansai/产品原则.md)）

精髓:用户翻开关西手账本任何一页，都该感觉是「**十个真懂关西的人帮我做的决策**」而不是攻略拼贴。节奏铁律:**慢-快-慢 / 京都中段沉浸 + 大阪前段或末段**——状态好时沉浸京都，末尾切换大阪；特种兵反向（大阪适合时差恢复）。70/20/10 关西落地:S 级是清水/岚山/伏见/金阁/道顿堀/大阪城/奈良鹿等；惊喜是黄昏渡月桥 15 分钟、清水子安塔背后视角、有马竹饰小径。A/B 推荐:A 是默认安心牌，B 是用户可能不知道但值得试，**A/B 必须体验角色不同**（A 回转寿司 vs B 路边关东煮 ✓；都是拉面 ✗）。模块拼装:**Sleep first, play second**——酒店决定活动半径。退化路径硬规:**人群 + 预算冲突时安全/适配性永远优先于预算升级**（亲子+高档不触发顶级餐厅，老人+特种兵密度必降级，情侣+经济宁少吃一顿保人群日晚餐）。餐饮/酒店/7 条化学反应章节已迁到装配 markdown 和 plans/写作规范.md，避免两份并存。
