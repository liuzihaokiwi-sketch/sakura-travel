# 产品价位体系 (Product Tiers)

## 概述
三大价位层级，共享同一套数据底座、评分引擎和杂志级渲染质量。差异化来自内容深度、个性化程度和地区/主题覆盖。

## 核心原则
- **渲染质量是基础线**：所有价位都是杂志级渲染，¥20 的排版必须让用户觉得"太值了"
- **引流款秀肌肉**：渲染品质本身就是转化驱动力
- **利润款卖省心**：个性化 + 餐厅酒店精选 + 避坑 = 选错一次的成本 > ¥128
- **高客单卖深度**：小众地区覆盖 + 高度主题化定制（茶道/动漫/温泉/滑雪…）

## 价位矩阵

### 引流款（¥19.9~29.9）
| 维度 | 规格 |
|---|---|
| 定位 | 秀肌肉、建立信任、引导转化 |
| 渲染 | 杂志级（这是肌肉本体） |
| 生成方式 | 预制模板 + AI 辅助润色 |
| 用户输入 | 不支持，直接选择预制路线 |
| 覆盖范围 | 日本主流城市/路线 |
| 景点 | ✅ S/A 级景点，详细信息 |
| 酒店 | 区域推荐 + 价格带，不给具体酒店名 |
| 餐饮 | ❌ 不包含 |
| 避坑指南 | ❌ 不包含 |
| 交通 | 基础交通方式 |
| 升级引导 | ✅ 关键位置自然露出 |
| 审核 | 自动审核 + 5%~15% 抽检 |
| 边际成本 | 趋近于零（预制内容） |

| 酒店 | ✅ 具体推荐 | ✅ 精选+理由 | ✅ 精选+对比+避坑 |

## 转化路径
```
免费预览 → ¥20 引流款 → ¥128 定制款 → ¥299+ 深度定制
            │                │                │
      "20块这质量？"    "选错一次>128"    "这种路线自己根本做不出来"
```

## 对数据层的要求
| 价位 | 数据要求 |
|---|---|
| 引流款 | 主流城市 S/A 级实体即可 |
| 利润款 | 主流城市全量实体 + editorial_boost 覆盖 |
| 高客单 | 小众地区实体数据 + 主题维度深度标签 + 专属 editorial_notes |

## 对工作流的要求
| 步骤 | 引流款 | 利润款 | 高客单 |
|---|---|---|---|
| 需求标准化 | 跳过（选模板） | 表单采集 | 表单+对话 |
| 候选召回 | 模板预制 | 动态召回 | 动态召回（扩展地区库） |
| 评分 | 预计算 | 实时计算 | 实时+主题权重加强 |
| 行程编排 | 模板装配 | 自动编排 | 自动+人工调优 |
| 餐厅推荐 | ❌ | ✅ | ✅+深度 |
| 酒店推荐 | 区域指南 | 具体推荐 | 精选+对比 |
| 审核 | 自动 | 自动+抽检 | 100%人工 |
| 渲染 | 杂志级 | 杂志级 | 杂志级 |

## 数据结构

### product_sku
| 字段 | 类型 | 说明 |
|---|---|---|
| sku_id | UUID | 主键 |
| tier | VARCHAR | standard / premium_69 / premium_128 / premium_199 / vip |
| theme | VARCHAR | general / tea / anime / onsen / ski / luxury / family… |
| region_scope | VARCHAR | mainstream / extended / niche |
| name | VARCHAR | 产品名称 |
| price_cny | DECIMAL | 价格 |
| features | JSONB | 功能清单 |
| workflow_config | JSONB | 工作流行为配置 |
| component_config | JSONB | 模板组件开关 |
| review_sample_rate | DECIMAL | 抽检比例 |
| is_active | BOOLEAN | 是否上架 |

### orders
