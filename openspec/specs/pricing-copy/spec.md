# Spec: pricing-copy

## 概述

定义价格页两张价格卡的完整文案规范，包括价格标注方式、价值描述、浮动说明口径和 CTA 文案。

## 价格标注规范

- 价格展示格式：`¥{price}` 大号字体
- 价格副标题：`{duration} 行程 · 参考价`，默认 duration = "7 天"
- 禁止出现："每天 ¥XX"、"¥XX/天"、"按天计费"等按天计价表达
- 禁止出现：复杂报价表、所有天数价格列表

## 价格浮动说明口径

**统一文案（两张卡底部各一行）：**
> 其他天数小幅浮动，制作前与你确认

**说明要求：**
- 不解释浮动计算方式
- 不写"联系客服"
- 隐含"付款前已知道确切价格"的保障感

## 页面小字（价格卡下方）

**最终采用版（最有转化感）：**
> 以上为 7 天行程参考价。天数不同，价格略有不同——提交信息后，我们 24 小时内微信确认价格再开始制作。你不会为一份没看到的东西付钱。

## 标准版价格卡规范

```
标题：标准行程定制
价格：¥248
副标题：7 天行程 · 参考价
权益列表：
  ✓ 完整 7 日行程 + 每日时间安排
  ✓ 餐厅推荐（含理由 + 避坑提醒）
  ✓ 交通说明 + 推荐交通卡
  ✓ PDF 正式交付 + 网页版随时查看
  ✓ 2 次免费微调
底部说明：其他天数小幅浮动，制作前与你确认
CTA：开始填写需求
CTA 样式：次要按钮（非填充，描边样式）
```

## 视觉优先级

- 移动端：纵向堆叠，深度版在下，有「推荐」角标
- 桌面端：并排，深度版有更强视觉权重（边框/阴影/角标）
- 两卡之间没有"OR"分隔，避免非此即彼的对立感
