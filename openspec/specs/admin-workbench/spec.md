## ADDED Requirements

### Requirement: 订单看板
`/admin` SHALL 展示三列看板：

| 待审核 📋 | 进行中 🔄 | 已交付 ✅ |
|---|---|---|
| 新生成的方案等待审核 | 正在审核/修改中 | 已发给用户的方案 |
| + 用户提交的精调请求 | | |

每个订单卡片显示：订单号、城市+天数+同行人、套餐类型（248/888）、旅行日期、创建时间/等待时长、状态标签（新订单/精调请求/紧急）

### Requirement: 单订单审核页
`/admin/order/[id]` SHALL 包含：

**布局**：桌面端左右分栏（左60% 右40%），移动端上下堆叠。

1. **顶部：订单信息条**
   - 订单号 | 城市天数 | 同行人 | 套餐类型 | 精调剩余次数 | 当前版本号
   - 状态下拉切换：待审核 → 审核通过 → 已发布

2. **左侧：方案预览**（60%）
   - 与用户看到的交付页**完全一致**的渲染
   - 只读模式，不可编辑
   - 用于快速浏览检查

3. **右侧：编辑面板**（40%，三个Tab）

   **Tab 1：结构化修改**（主操作区）
   - 选择天数：Day 1-7 标签
   - 替换景点：当前景点 → 下拉选备选（5个，比用户端多2个）
   - 替换餐厅：当前餐厅 → 下拉选备选（5个）
   - 替换酒店/住宿区域：当前推荐 → 下拉选备选
   - 调整时间安排：拖动时间轴节点
   - 删除/添加景点：从备选库选取

   **Tab 2：提示词补充**（辅助区，非必须）
   - 给审核人员用的 prompt 输入框
   - 用户永远看不到
   - 用于处理结构化按钮搞不定的特殊情况
   - 按钮：[重新生成此天] [重新生成全部]

   **Tab 3：历史版本**
   - 版本号、时间、操作人、变更摘要
   - 可以点击回退到任意版本
   - 每次操作自动保存版本

4. **底部：操作按钮栏**
   - [保存草稿] — 保存修改但不发布
   - [预览效果] — 新标签页打开用户视角预览
   - [发布给用户 ✓] — 状态切换为"已发布"

### Requirement: 5分钟审核流程
高频操作流程设计：
1. 打开订单 → 快速浏览方案（1分钟）
2. 检查关键点：行程节奏、餐厅营业状态、交通合理性、推荐理由是否有"AI味"（2分钟）
3. 如需调整：用结构化按钮替换1-2个点（1分钟）
4. 预览效果 → 点击"发布"（30秒）

**效率优化**：
- 系统自动标注可能有问题的内容（黄色高亮）：新开/已关闭的店、过长通勤、时间冲突
- 一键"看起来没问题"快捷按钮：跳过检查直接发布
- 键盘快捷键：`Cmd+S` 保存，`Cmd+Enter` 发布

### Requirement: 工作台与前台严格分离
- 内部工作台在 `/admin/*` 路由下，需要密码访问
- 用户前台不能看到任何后台信息（审核状态、内部备注、提示词）
- 不在同一个导航体系中
- 不同的视觉风格（工作台用简洁工具风，不用杂志风）
