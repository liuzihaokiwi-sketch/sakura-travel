## Why

当前产品前台存在致命转化问题：**从技术功能出发包装，而非从用户痛点出发设计**。用户看到"12维评分""11个爬虫""240+景点"时，感受到的是一个极客工具而不是"帮我解决旅行烦恼的服务"。

三个核心问题：
1. **没有转化漏斗**：用户从引流（小红书/樱花地图）到付费之间没有"免费尝鲜→信任建立→自然升级"的路径设计
2. **前台文案反人话**：讲功能不讲结果，讲技术不讲痛点，HubSpot 和 NN/g 的定价页最佳实践都强调"用户买到什么"比"你做了什么"更重要
3. **没有交付/修改闭环**：付费后怎么收、怎么交付、怎么改、怎么升级——全链路缺失

**MVP 优先级**：🔴 P0 — 这是从"有个网站"到"能收钱"的关键跨越，直接影响所有价位产品（免费→248→888）的转化率。

## What Changes

- **新增完整转化漏斗**：引流页 → 轻问卷 → 免费一日预览 → 价格页 → 正式问卷 → 生成中 → 交付页 → 修改页 → 升级页
- **重写全站文案**：从"功能导向"转向"痛点→结果导向"，弱化 AI 感，强化专业判断感
- **新增价格体系页面**：免费预览 + 248首发主推款 + 888锚点，三档清晰不犹豫
- **新增问卷系统**：轻问卷（免费用，4题快速）+ 正式问卷（付费后，深度定制）
- **新增交付页面**：杂志感 H5 行程展示，含升级入口和分享机制
- **新增结构化修改系统**：用户端按钮/下拉修改，内部端提示词纠偏，限次不限死
- **新增内部审核工作台**：5分钟完成一单审核的精简后台
- **统一品牌语气规范**：专业、有判断、有温度，不是 AI 工具也不是廉价模板

## Capabilities

### New Capabilities
- `product-positioning`: 产品核心定位 — 用户痛点映射、价值主张、品牌叙事、与普通攻略的差异化
- `conversion-funnel`: 转化漏斗设计 — 9步完整用户路径，每步页面目标/CTA/防流失策略
- `pricing-page`: 价格页与产品包装 — 三档定价、套餐命名、对比表、锚点策略
- `landing-page-copy`: 引流页文案 — Hero/痛点/方案/预览/信任/套餐/FAQ/CTA 八大模块
- `questionnaire-system`: 问卷系统 — 轻问卷（引流）+ 正式问卷（付费），结构化字段设计
- `delivery-page`: 交付页设计 — 杂志感行程展示、推荐理由写法、升级/复购/分享入口
- `modification-system`: 修改机制 — 用户端结构化修改 + 内部提示词纠偏 + 限次策略
- `admin-workbench`: 内部审核工作台 — 5分钟审核流程、高频操作、状态管理
- `brand-voice`: 品牌语气规范 — 一句话定位、CTA库、FAQ库、口号库、推荐理由模板

### Modified Capabilities
- `sakura-landing-page`: 首页文案和结构需要按新的痛点→结果导向重写
- `custom-service-page`: 定制服务页需要拆分为转化漏斗中的多个独立页面

## Impact

- **前端**：`web/app/` 下新增 6-8 个页面路由，现有 `/custom` 页面重构为漏斗入口
- **后端**：需新增 API — 问卷提交、订单创建、方案生成触发、修改请求、审核状态
- **数据库**：新增 orders/questionnaires/modifications/reviews 表
- **文案**：全站文案重写，需要品牌语气规范文档作为所有页面的写作指南
- **运营**：需要内部审核工作台（可以先用简单的 Admin 页面）
- **依赖**：支付接口（微信支付/支付宝，可后期接入，初期可手动确认）
