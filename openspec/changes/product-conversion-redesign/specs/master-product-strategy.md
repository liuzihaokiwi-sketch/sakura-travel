# 第一部分：最终产品定位

## 1. 这个产品卖的是什么

不是攻略，不是模板，不是 AI 工具。

**我们卖的是：一个替你做对旅行选择的专业规划服务。**

你花 3 天刷小红书，看了 200 篇攻略，最后还是不知道该怎么排。我们帮你把这 3 天的纠结，变成一份直接能用的方案——每一天怎么走、为什么这样走、哪些坑要避、哪些钱该省、哪些体验不能错过，全部帮你想好。

通常你会收到一份 30-40 页的完整行程方案。但重点不是页数，而是这些页帮你解决了什么：不用自己查交通、不用猜哪家餐厅靠谱、不用担心行程太赶走到崩溃、不用到了才发现该预约的没预约。

## 2. 用户最核心的 8 个痛点

| # | 痛点 | 用户内心OS |
|---|---|---|
| 1 | 信息过载 | "刷了三天小红书，收藏了200条，还是不知道怎么排" |
| 2 | 选酒店焦虑 | "位置、价格、评分，怎么选都怕踩雷" |
| 3 | 餐厅踩坑 | "排了两小时队，结果就那样" |
| 4 | 时效性盲区 | "现在去到底能不能看到樱花？" |
| 5 | 节奏失控 | "排太满累死，排太松又浪费" |
| 6 | 缺乏针对性 | "带孩子和蜜月完全不一样，普通攻略不分这个" |
| 7 | 交通迷路 | "JR、地铁、巴士、一日券，到底怎么组合最划算" |
| 8 | 准备遗漏 | "到了才发现要提前约、要提前买、要提前换" |

## 3. 解决方案如何逐一对应

| 痛点 | 我们的结果 |
|---|---|
| 信息过载 | 你不用刷了。告诉我们你的情况，直接给你一份"按你来排"的方案 |
| 选酒店焦虑 | 帮你选对住宿区域，告诉你为什么住这里更方便、更省钱 |
| 餐厅踩坑 | 推荐本地人真正在去的店，避开评分虚高的网红陷阱 |
| 时效性盲区 | 基于公开信息和旅行逻辑判断"现在值不值得去" |
| 节奏失控 | 按你的体力和偏好排节奏，不赶不浪费 |
| 缺乏针对性 | 根据同行人、预算、偏好做取舍，不是一套模板发给所有人 |
| 交通迷路 | 每天的交通怎么走、买什么券、步行多远，全部写清楚 |
| 准备遗漏 | 出发前的准备清单、预订提醒、安全须知，一次给齐 |

## 4. 为什么这不是普通攻略

普通攻略告诉你"东京有哪些景点"。我们告诉你"以你的情况，第三天下午去新宿御苑比浅草寺更值，因为那个时段人少、光线好、离你住的地方近"。

普通攻略是信息的堆砌。我们是信息的取舍——帮你从 1000 个选项里，挑出最适合你的 30 个，并且告诉你为什么。

## 5. 为什么"免费一天预览"是最合理的入口

- 用户没见过你的东西，不可能直接付 248
- 免费一天预览让用户零风险验证"这个服务的方法和风格适不适合我"
- 看到第一天的质量后，用户自然想看剩下六天
- 免费预览是信任建立的过程，不是赠品

## 6. 为什么前台必须减少选项

- 用户来的时候已经信息过载了，不能再给他一堆选择
- 选项越多，决策时间越长，流失率越高
- 三个层级足够：免费试 → 主推买 → 高端锚点
- 用户只需要做一个决定："要不要先免费看一天"

## 7. 为什么主推款必须成为唯一视觉中心

- 如果三个档位视觉权重一样，用户会纠结对比
- 248 是我们最想卖的，必须视觉最大、位置最中、信息最全
- 888 的作用是让 248 显得更值，不是让用户犹豫
- 免费的作用是降低门槛，不是分散注意力

# 第二部分：前台产品结构（最优解）

## 三个层级的角色

| 层级 | 角色 | 用户心理 |
|---|---|---|
| 免费一天预览 | **门槛消除器** | "免费看一天，没什么损失" |
| ¥248 首发 / 原价 ¥368 | **主推核心款** | "看了一天觉得不错，248 买完整版挺值" |
| ¥888 深度定制 | **价格锚点** | "888 太贵了，248 性价比真高" |

## 用户看到时的理解路径

```
"先免费试一天" → 觉得好 → "248 买完整的" → 看到 888 → "248 就够了，真划算"
```

## 为什么不增加更多档位

- 4 个档位 = 用户开始做减法 = 决策成本上升 = 流失
- 3 个是心理学最优：低/中/高，用户自然选中间
- 多一个档位就多一次"这个和那个差在哪"的困惑

## 为什么 248/368 是合理结构

- 368 是"正常价"，让 248 有折扣感
- 248 不是低价，而是"限时优惠价"，保持了产品调性
- "原价 368，首发 248"= 用户觉得占了便宜 + 有紧迫感

## 为什么 888 只适合做锚点

- 888 的核心作用是让 248 显得更值
- 如果 888 卖得太好，说明 248 的价值没传达到位
- 888 应该存在但不抢眼，让用户自己得出结论："248 就够了"

## 价格卡排版原则

```
┌──────────┐  ┌═══════════════════┐  ┌──────────┐
│ 免费预览  │  ║  ¥248 首发价      ║  │ ¥888     │
│          │  ║  ⭐ 最受欢迎       ║  │ 深度定制  │
│ 小卡片   │  ║  大卡片·有边框·   ║  │ 小卡片   │
│ 低调     │  ║  高亮·视觉中心    ║  │ 低调     │
└──────────┘  └═══════════════════┘  └──────────┘
```

- 248 卡片：最大、有品牌色边框、"最受欢迎"标签、删除线原价
- 免费卡片：简洁，一个 CTA 按钮
- 888 卡片：信息完整但视觉不抢眼，灰色调

## 如何让 248 显得值但不廉价

- 有删除线原价 368 做对比 → 不是便宜，是优惠
