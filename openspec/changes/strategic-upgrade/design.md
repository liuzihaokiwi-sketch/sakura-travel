## E. 我最建议优先做的 10 件事

> 按照"投入产出比 × 紧迫度"排序

### 1. 🔴 打通"问卷→生成→交付页"全流程（C1+C3）
**为什么是第一**：没有这个，一切都是纸上谈兵。不需要完美，只需要能跑通。
**具体做法**：quiz API → 触发 arq job → assemble_trip → 写入 itinerary 表 → /plan/[id] 从 API 读数据
**预计工作量**：2-3天

### 2. 🔴 补全大阪/京都路线模板+数据（C4+D10）
**为什么是第二**：东京+大阪+京都覆盖 80% 的中国赴日自由行需求（据 JTB 2025 统计，关西圈是仅次于东京的第二大目的地）。没有这两个城市，问卷选项严重受限。
**具体做法**：建 osaka_classic_3d/5d + kyoto_classic_2d/3d 模板 + 爬虫补实体
**预计工作量**：2-3天

### 3. 🔴 接通企微 webhook + 审核看板（C5+D4+D17）
**为什么是第三**：有了生成能力后，需要知道有单→处理→发出。这是运营闭环。
**具体做法**：quiz 提交时调企微 webhook → admin 看板对接真实数据 → 审核后一键发布
**预计工作量**：1-2天

### 4. 🟠 小红书内容矩阵启动（D26）
**为什么紧急**：内容需要时间发酵。今天发的帖子，2-4 周后才开始有稳定流量。冷启动最缺的不是产品，是流量。
**具体做法**：每天 1 篇攻略型笔记（东京/京都/大阪），CTA 引流到问卷
**预计工作量**：持续投入，每天30-60分钟

### 5. 🟠 免费预览"质量炸弹"打磨（D5）
**为什么重要**：Day 1 预览是免费→付费的唯一转化杠杆。这一天的方案质量直接决定 248 的转化率。
**具体做法**：为东京/京都/大阪各准备 1 套"标杆级"Day 1 方案（人工精修文案+图片）
**预计工作量**：1-2天

### 6. 🟠 推荐理由"证据化"（D14）
**为什么重要**：这是你和"小红书攻略""ChatGPT 直接问"最大的差异化点。你有真实评分数据，用出来。
**具体做法**：在推荐理由中嵌入 "Tabelog 3.82 分 / 1,240 条评价""距上一站步行 8 分钟"
**预计工作量**：0.5天

### 7. 🟡 退款 SOP + 订单状态完善（C2+D32）
**为什么现在做**：第一单成交前必须有退款预案，否则客诉来了会手忙脚乱。
**具体做法**：admin 后台加"确认收款""退款"按钮 + 退款话术模板
**预计工作量**：0.5天

### 8. 🟡 企微主号迁移+标签体系（D23）
**为什么现在做**：个微封号风险真实存在。企微免费、功能更强、可多人协作。
**具体做法**：注册企微 → 配置客服号 → 按用户状态打标签
**预计工作量**：0.5天（运营侧）

### 9. 🟡 方案可分享为长图（D20）
**为什么重要**：用户收到方案后最自然的动作是"发给同行人看"和"发朋友圈"。这是免费获客渠道。
**具体做法**：Playwright 截图已实现，交付页加一个"保存为图片"按钮
**预计工作量**：0.5天

### 10. 🟡 收集前 5 个用户的详细反馈（D12+C16）
**为什么重要**：冷启动阶段的用户反馈 = 黄金。每一条反馈都在帮你校准评分权重、文案风格、推荐质量。
**具体做法**：旅行后主动微信回访 + 结构化记录（满意度1-5 + 最喜欢/最不满的推荐 + 建议）
**预计工作量**：持续进行
