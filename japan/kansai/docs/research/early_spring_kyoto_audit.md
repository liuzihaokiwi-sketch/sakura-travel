# 早春京都模板补齐与反思审计

> 研究日期：2026-04-21
> 研究者：AI 策展（需复核）
> 目标：盘点现有 7 个早春京都模板是否够用，是否漏了景点/体验，是否需要合并或做人群变种。
> 范围：2 月中旬 – 3 月中旬（梅花主档期 + 3/14 酒祭、3/20 前十石舟不开航等关键切点）
> **声明**：本次研究 opencli 小红书桥未连（Browser Bridge not connected），数据源以 WebSearch 聚合的官方/博客/MK/京阪 2026 情报为主，小红书爆款数据为**定性估计**（待 opencli 恢复后复核）。标注 `[待小红书复核]`。

---

## 一、前置结论（先说判断）

**要新建：0 个独立模板**。
**要改现有模板：6 个需 patch（early_spring_notes / auto_append / variant）**。
**要归档/合并：1 个候选（岚山早春 vs 岚山 core 的关系需确认）**。
**不做人群变种**（按现有 memory `feedback_variant_naming` 不做情侣/亲子标签；audience_bonus 评分已经表达了差异）。

这个判断的核心逻辑：**早春京都的爆款集中度极高**——北野梅花祭（2/25）+ 城南宫枝垂梅 + 淀水路河津樱 + 夜间特别参拜 + 吉田神社节分祭（2/2-4）这 5 个点几乎垄断了早春京都的小红书流量，其他梅花名所都是"长尾"。长尾内容最合适的归宿是**挂在现有模板的 season_notes / 扩展 slot**，不是独立模板。早春窗口只有 4 周，模板多了反而稀释用户注意力。

## 二、候选景点逐一结论

### 2.1 梅宫大社（梅×猫，右京区松尾）

- **一句话定位**：日本最古老酒造神 + 14 只神社猫 + 500 株 35 品种梅（多品种梯次盛开，可延续到 3 月下旬）
- **数据支撑**：
  - NHK《岩合光昭の世界猫步》京都特辑取景地（全国级曝光）[1]
  - 梅花见顷 2 月中下旬 – 3 月中旬，3 月第一个周日"梅・产祭"[2]
  - 本社免费，神苑 ¥600；参拜时间 9:30-16:30
  - **猫只在清晨/傍晚出现**（白天都在睡觉），这是关键体验细节
  - 明星猫"ツキ"已于 2026 年 2 月离世 → 对"撸猫"期待值需要降调
  - `[待小红书复核]` 估计收藏 1k-5k 量级（小众但有明确标签）
- **可信度**：`cross_checked`（京都观光 Navi + Jalan + 多篇博客交叉）
- **结论：B**（加进 `kyo_kitano_ume_full` 的 extended_slot 或做北野-梅宫联动日的 P3 延长，不独立）
  - **原因**：(1) 梅宫离北野天满宫/金阁寺直线 6km 但巴士要 40 分钟，不顺路；(2) 单点 2 小时撑不起半日；(3) 酒造信仰+猫是独特，但不足以对抗北野 1500 株+梅花祭的量级；(4) 和岚山早春同处右京区，**更自然的挂法是岚山早春日 after-午餐的 P3 延长**（岚山 → 松尾大社 → 梅宫大社），可走"酒造+梅+猫"的侧线。

### 2.2 京都御苑梅林（上京区，京都御所西南）

- **一句话定位**：200 株梅 + 170 株专属梅林 + 1 月初开到 3 月中 + 完全免费 + 游客压力接近零
- **数据支撑**：
  - 国民公园协会 2026 年官方开花播报；早梅 1 月中开始，梅林 2 月下见顷 [3]
  - 内部多个分区（梅林/中山邸迹/黑木之梅/素心腊梅/桃林）分段开，**同时能看梅+腊梅+水仙+金缕梅**，关西别处做不到
  - 2026/3/25-29 御所春季特别公开
  - 完全免费，无门禁，24 小时可入（夜间不推荐）
  - `[待小红书复核]` 估计收藏 500-2k（日本人游客源多于中国游客）
- **可信度**：`verified`（官方国民公园协会）
- **结论：B**（加进 "市中心空窗日"的装配池 / 在 `kyo_tokubetsu_spring_night` 白天前置段提）
  - **原因**：(1) 独立性不足——景点本身"就是个大公园"，氛围好但故事薄；(2) 最大价值是"顺路 + 免费 + 完全不拥挤"，适合装配在**夜间特别参拜或其他东山系日的上午前置**（京都御苑 9:00-11:00 → 地铁到东山午餐 → 下午夜参）；(3) 可以在 `kyo_tokubetsu_spring_night` 增加 `pre_dusk_options`: 京都御苑梅林作为白天前置散步选项。

### 2.3 下鸭神社 光琳の梅（左京区）

### 2.5 大原野神社 / 正法寺（西京区大原野）

- **一句话定位**：西山麓"花の西之京" + 正法寺梅园 100 株 + 京都市全景 + 几乎没中国游客

### 2.8 節分祭（2/2-4 吉田神社追儺式 + 2/3 八坂豆まき + 蘆山寺鬼おどり）

- **一句话定位**：京都最大 800 摊+50 万人流+方相氏追鬼古式 + 仅 3 天窗口
- **数据支撑**：
  - 2026/2/2（一）前日祭 + 追儺式（18:00） / 2/3 当日祭 + 火炉祭（23:00） / 2/4 后日祭 [9]

  - **原因**：(1) 窗口只有 3 天，独立模板的成本/复用率极差；(2) 现有 7 个模板没有任何一个覆盖 2 月初的窗口——**淀水路河津樱最早 2 月中下旬，城南宫 2/18 起，节分祭 2/2-4 完全是"前档期"**；(3) **最佳方案：新建一个 `kyo_setsubun_special_date.json`（special_date pool）**，专门给 2/2-4 来京都的用户，覆盖吉田+八坂+蘆山寺路线；(4) 这是**唯一一个真正该"新建"的东西**，但它不是"full_day 模板"而是 `special_date` 类型，占用日期但不占用日装配配额。
  - **修正建议**：写成 `kyo_setsubun_special_night.json`（night_module），因为核心仪式都是 18:00 后的。或者 half_day_afternoon。

## 三、现有 7 个模板的整改建议

### 3.1 `kyo_arashiyama_spring_full` 岚山早春静美版

- **现状**：独立 full_day，评分 51，描述强调"樱花季前 + 梅尾 + 竹林独享窗口"
- **问题**：
  - (1) `japan/kansai/common/` 里**没有** `kyo_arashiyama_core_full`（实际有）——两个文件存在覆盖风险
  - (2) 检查后发现 core 模板依然存在，当前状态是"early_spring 版"和"common core 版"并存
- **建议**：**保留独立模板**，但在文档里写清楚：这不是和 common 二选一，装配时早春季节专用此版；common 版在非早春季节使用。
- **额外**：在 slots 里加**梅宫大社+松尾大社 P3 延长**（午餐后→松尾大社酒造→梅宫大社撸猫+梅），给体力好的用户。

### 3.2 `kyo_jonangu_ume_half` 城南宫枝垂梅半日

- **现状**：half_day_morning，评分 40，audience_bonus couple +5 friends +3 family -2
- **问题**：audience_bonus 已经表达了"出片/情侣向"倾向，不需要做人群变种。
- **建议**：**保留，patch 元数据**：
  - 加 `_season_dates: "2026/2/18-3/22（しだれ梅と椿まつり）"`
  - 加 `_fee: "神苑拝観 大人 ¥1000"`
  - 确认 auto_append 里有"枝垂梅+落椿共演"的叙事——这是城南宫**独一无二的双花卖点**，不能只写"梅"。
- **不做**：出片专属 variant（audience_bonus 已经做到）

### 3.3 `kyo_kinkaku_spring_full` 金阁寺北山线早春版

- **现状**：full_day，core = 金阁+龙安+北野天满宫
- **问题**：**和 `kyo_kitano_ume_full` 重叠**——两个模板都把北野天满宫作为核心，这是 common 策展原则里明确要规避的"一天内 critical_entities 重复"。
- **建议**：**二选一装配互斥**。在 `excludes_when` 里加"若已选 `kyo_kitano_ume_full` 则本模板不可选"，反之亦然。或者把 kinkaku 早春版的北野戏份降为 P3 可选（kinkaku → 龙安 → 仁和 → 北野 可选，非 core）。
- **不新建**：金阁"出片"变种（常识性，audience_bonus 已解决）

### 3.4 `kyo_kitano_ume_full` 北野梅苑日

- **现状**：full_day，core = 北野天满宫+平野神社（平野神社 3 月末樱花才是主角，早春梅只是过渡）
- **问题**：
  - (1) 平野神社早春只有"魁桜"零星开（按 `09_facts_to_collect.md` 的记录），作为 core 不成立
  - (2) **梅花祭 2/25** 在现有模板里看不到——这是 900 年传统 + 上七軒舞妓野点 ¥3000 + 1 月 25 日起限量发券，**必须进 season_notes 作为 special_date**
  - (3) 梅苑"花の庭"2/1-5/24 夜間 light up 蜷川実花 2026 特别 installation——**现代 × 传统**的绝佳叙事，现有模板没体现
- **建议**：
  - 把平野神社从 core 降为 P3
  - **把梅宫大社补进 P3 扩展**（梅宫离北野 5km，比平野更对味早春主题）
  - 新增 `special_date_20260225`：梅花祭+野点
  - 新增 `night_options`：梅苑夜間 light up + 蜷川実花（晚间 20:30 前）
  - 这样 kyo_kitano_ume_full 从"早春 full_day"升级为"早春梅主线日 + 夜间加成 + 2/25 特定日"，变成旗舰模板。

### 3.5 `kyo_tokubetsu_spring_night` 夜间特别参拜

- **现状**：night_module，season_window 写的是 early_spring 但描述里也包含樱花季（"早春/樱花季"）
- **问题**：
  - (1) 季节窗口不清晰——"早春"和"樱花季"是两个独立档期，应拆开；**早春版主打的是清水/高台/青莲的"梅+夜间"**（花灯路替代方案），**樱花季版主打夜樱**；
  - (2) 现有描述里混着两个季节，用户看不清
- **建议**：
