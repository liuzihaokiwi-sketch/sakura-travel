## 三层职责（D36 后）

| 层 | 文件 | 格式 | 消费方 | 关键约束 |
|---|---|---|---|---|
| 模板 | `templates/**/*.json` | JSON | 渲染代码 + AI 装配 | 只管动线（slots 三档 + 设计 note）；**不写打分/min_days/A-B/酒店升档** |
| 事实 | `facts/*.json` | JSON | 渲染 + 装配 | 景点/餐厅/酒店的客观属性；**附属内容（photo_spots/fun_facts/nearby_shops）跟着景点走** |
| 装配 | `assembly/*.md` | **Markdown** | AI 装配 | 所有"用户档位/天数/人群"相关规则；中文书写 |
| 事件 | `events/*` | 待迁 markdown | AI 装配 | 樱花/红叶/祭典窗口 |

## 常见任务入口

- **改模板** → `templates/{季节}/`，文件名 `{城市}_{动线}_{特征}_{时长}.json`
- **看 SOP** → [docs/04_操作SOP/模板写作.md](../../docs/04_操作SOP/模板写作.md)（顶部有 D36 覆盖段，先读）
- **看字段定稿** → [docs/03_数据契约/SCHEMA.md](../../docs/03_数据契约/SCHEMA.md)
- **改装配规则** → `assembly/{模板装配,餐厅装配,酒店装配}.md`
- **事实核查** → opencli + WebSearch
- **季节分档** → 7 档（common + early_spring + sakura + tsuyu + matsuri_peak_summer + summer_low + koyo + deep_winter）
- **v1 原版参考** → `_legacy/v1_content/`
