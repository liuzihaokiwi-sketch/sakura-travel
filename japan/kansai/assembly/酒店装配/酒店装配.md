## 一、装配原则

### 1.1 模板只给"区域"，酒店和几晚由装配挑

模板 hotel slot 只写 `hotel_area`。**模板不写具体酒店 entity，也不写 stay_nights**。

- 几晚 = 装配根据"用户在这区域要住几天"推断（连续 N 天住同区域 = 这家酒店住 N 晚）
- 哪家酒店 = 装配按 area + 用户档位 + 偏好修饰器从酒店池筛

### 1.2 系统直接推荐一家

**不给用户多家让他选**。

呈现：
- 推荐酒店 + 推荐理由（主要篇幅）
- 备选 1 家（小字，与推荐差异化）

### 1.3 住宿档位升级比例

按用户档 × 总晚数决定"全本档" vs "几晚升档"：

| 用户档 | 5-6 晚 | 7-8 晚 | 9-10 晚 | 11-13 晚 |
|---|---|---|---|---|
| 干净方便 | 全本档 | 全本档 | 全本档 | 全本档 |
| 品质舒适 | 全本档 | 1 晚升享受 | 1 晚升享受 | 1-2 晚升享受 |
| 享受型 | 1 晚升顶级/特色 | 1 晚升顶级/特色 | 1-2 晚升特色 | 2 晚升特色 |
| 顶级 | 1 晚最顶级 | 1 晚最顶级 | 1-2 晚最顶级 | 2 晚最顶级 |

**硬规**：
- 干净方便档不做升级（选便宜就尊重选择）
- 品质舒适以上自动安排特色住宿，升级费用已算入总价

### 1.4 酒店偏好修饰器

| 修饰器 | 倾向 |
|---|---|
| `quality_first` | 住得好，星级优先 |
| `resort_vibe` | 度假感，温泉 / 景观 |
| `value_for_money` | 性价比 |
| `balanced` | 系统综合判断（默认） |

### 1.5 关西酒店区域偏好

- **京都**：四条河原町（便利） / 京都站（交通） / 祇园（体验） / 岚山（一泊温泉）
- **大阪**：难波（美食 / 交通） / 心斋桥（购物） / 梅田（商务） / 新大阪（换城）
- **神户**：三宫（主要） / 北野（异人馆体验）
- **有马**：金泉町（最经典）
- **城崎**：温泉街中心

### 1.6 人群 + 预算冲突（硬规）

**安全 / 适配性永远优先于预算升级**。

- 亲子 + 高档 → 选**亲子友好**而非最贵（可能跳过设计感酒店选大型连锁套房）
- 老人 + 特种兵 → 优先**电梯 / 无障碍 / 楼层低**，体力约束优先
- 情侣 + 经济 → **氛围优先于省钱**，宁可少换地方也保证人群日酒店质量

### 1.7 换城与酒店连续性

- 换城日的 hotel_area 必然变化（京都→大阪），装配自动识别为"换酒店日"
- 同城连续 N 天，酒店尽量保持同一家（除非主动安排"前 X 晚市区 + 后 Y 晚温泉一泊"这种）
- 温泉一泊后必然换回市区——城市基地酒店不退房，行李寄存

### 1.8 "特色住宿"枚举

升档花在"特色"上而非"贵"上。关西特色住宿：

- 京都：町家民泊（祇园） / 翠岚（岚山温泉）/ 俵屋旅馆（顶级老铺）
- 有马：兵卫向阳閣 / 中の坊瑞苑（金泉露天）
- 城崎：西村屋本馆（蟹会席代表）
- 大阪：W Osaka（设计感顶流）/ 帝国饭店大阪（经典老牌）
- 神户：北野饭店（异人馆改造）
