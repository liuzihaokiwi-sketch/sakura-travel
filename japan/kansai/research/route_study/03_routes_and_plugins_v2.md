## 架构总览

```
用户输入（日期/天数/人群/notes）
       ↓
[系统硬筛] 选出候选主路线 + 可激活插件
       ↓
[3 档评级打分] 主路线和插件各自给档期/天数/人群评级
       ↓
[Opus 软匹配] 交叉 notes 和 vibe，从候选里挑最合适的一条主路线 + 若干插件
       ↓
[展开] 主路线骨架 + 挂载的插件 → 每天模板序列
```

### 两个核心对象

**主路线（Route）**：定义一趟旅程的骨架——哪几城、大致顺序、基础天数
**插件（Plugin）**：可挂载到主路线上的体验模块——半日 / 一日 / 2D1N 不等

### 3 档评级

每个主路线和插件都有三组评级：
- **档期**：12 个档期分别标 **推荐 / 一般 / 不推荐**
- **天数**：按天数段标 3 档
- **人群**：按 party 类型标 3 档

"不推荐" = 硬筛掉，根本不进候选
"一般" + "推荐" = 进候选池，交给 Opus 选

### 12 档期定义（沿用 01 沉淀的划分）

| 档期代号 | 时间 | 特征 |
|---|---|---|
| `early_spring_low` | 3/1–3/24 | 早春淡季，梅花 |

| `sakura_late` | 4/8–4/15 | 晚樱（吉野奥千本） |
| `early_summer_low` | 4/16–5/25 | 初夏淡季，新绿 |
| `tsuyu_ajisai` | 5/26–7/10 | 梅雨 + 绣球花 |
| `matsuri_peak` | 7/10–7/25 | 祭典盛夏 |
| `high_summer_low` | 7/26–8/25 | 盛夏淡季 + 花火 |
| `autumn_low` | 8/26–11/5 | 初秋淡季 |
| `momiji_early` | 11/5–11/15 | 早红叶 |

## 一、主路线（Main Routes）

### K01 · 双城经典

**核心承诺**：关西首次来访者的"安全默认"——双城分住 + 奈良日归是所有攻略站最大公约数。

**最小骨架**：京都 → 大阪，中途奈良日归（从京都出发 1h）

**评级**：

档期：
- 推荐：`early_summer_low` / `autumn_low` / `early_spring_low`
- 一般：其他所有档期
- 不推荐：无（全年能跑，但花季/红叶季会被 K20/K22 抢）

天数：
- 推荐：5–7 天
- 一般：8 天
- 不推荐：4 天以下 / 9 天以上（会退化成 K02 或加插件）

人群：
- 推荐：classic / family_fun / 首次来访
- 一般：romantic / 带老人
- 不推荐：带幼儿（2 岁以下）

---

### K02 · 双城深度（不加奈良）

**核心承诺**：二回目用户 / 时间紧张者 / 对奈良不感兴趣的人——把京都和大阪每个区域都做透。

**最小骨架**：京都 3–4（南东山+岚山+金阁北山+大原或鞍马）→ 大阪 2–3（市井核心+USJ/海游馆）

**评级**：

档期：
- 推荐：`early_summer_low` / `autumn_low` / `early_spring_low`
- 一般：`tsuyu_ajisai`（梅雨不算坏）
- 不推荐：`sakura_peak` / `momiji_peak`（浪费档期）

天数：
- 推荐：5–7 天
- 一般：8–9 天
- 不推荐：10+ 天（该加插件了）

人群：
- 推荐：classic / romantic / photogenic / 二回目
- 一般：family_fun（没奈良娃会少一个乐子）
- 不推荐：首次来访的亲子（奈良喂鹿几乎必吃）

---

### K20 · 樱花追花双城

**核心承诺**：利用樱花 7–10 天递进延长观赏窗口——醍醐寺早樱 → 哲学之道 + 圆山盛开 → 岚山晚樱 → 吉野山奥千本最后。

**最小骨架**：京都 3（多点分时赏樱）+ 奈良 1（公园樱花）+ 大阪 2（大阪城西之丸、造币局通抜）

**评级**：

档期：
- 推荐：`sakura_peak`
- 一般：`sakura_late`（只剩岚山 + 吉野奥千本可看）
- 不推荐：所有其他档期（这条路线在窗口外无意义）

天数：
- 推荐：6–9 天
- 一般：5 天（紧凑）/ 10 天（建议升级挂 P_yoshino）
- 不推荐：4 天以下

人群：
- 推荐：classic / romantic / photogenic / 闺蜜
- 一般：family_fun（人海对小孩不友好）
- 不推荐：带老人（早起抢位 + 人海）

---

### K22 · 红叶夜枫双城

**核心承诺**：日间山水 + 夜间灯火——东福寺 / 高雄三寺白天，永观堂 / 清水寺夜间特别拝観。

**最小骨架**：京都 3–4（东福寺+永观堂夜枫 / 高雄日 / 岚山红叶 / 大原或琉璃光院）+ 大阪 2（箕面红叶 + 市区）

**评级**：

档期：
- 推荐：`momiji_peak`
- 一般：`momiji_early`（贵船可看，其他还没红）/ `early_winter` 前段（岚山晚红叶 12/初还在）
- 不推荐：所有其他档期

天数：
- 推荐：6–8 天
- 一般：5 天（紧凑）/ 9+ 天（可加插件）
- 不推荐：4 天以下

人群：
- 推荐：classic / romantic / photogenic
- 一般：family_fun（夜枫到 21 点小孩会困）
- 不推荐：带老人（11/18–11/26 人潮挤压，体力吃不消）

---

### K23 · 红叶错峰冷门

**核心承诺**：给二回目游客 / 不愿挤人潮——圆光寺 / 曼殊院 / 光悦寺 / 毘沙门堂等冷门赏枫点 + 高雄三寺深度。

**最小骨架**：京都 3–4（修学院+一乘寺+山科+洛北鞍马贵船）+ 大阪 2

**评级**：

档期：
- 推荐：`momiji_peak` / `momiji_early`
- 一般：`early_winter` 前段
- 不推荐：其他所有档期

天数：
- 推荐：7–9 天
- 一般：6 天 / 10 天
- 不推荐：5 天以下（冷门点地理分散）

人群：
- 推荐：classic / photogenic / 二回目
- 一般：romantic
- 不推荐：首次来访（永观堂不去会遗憾）/ 带老人（冷门点徒步多）/ 带幼儿

---

### K26 · 城崎松叶蟹冬季

**核心承诺**：11 月中–3 月初是城崎的独立主角窗口——7 外汤巡 + 浴衣夜步 + 松叶蟹怀石。**这个档期城崎改写整条骨架**，不是常规+插件。

**最小骨架**：京都 2–3（轻度）→ 城崎 2D1N → 大阪 2（收尾）

**评级**：

档期：
- 推荐：`early_winter` / `deep_winter`
- 一般：`momiji_peak` 后段（11 月中后期松叶蟹刚启动）
- 不推荐：其他所有档期（蟹不在季，温泉街夏天体验差）

天数：
- 推荐：7–10 天
- 一般：6 天（紧凑，京都只剩 1.5 天）
- 不推荐：5 天以下

人群：
- 推荐：romantic / 闺蜜 / 美食家
- 一般：classic
- 不推荐：带幼儿（城崎以旅馆文化为主，不适合孩子跑动）

---

### K43 · 白滨海边温泉

**核心承诺**：关西唯一海边温泉——白良滨白沙滩 + 面海温泉 + 当地海鲜。和内陆温泉（有马/城崎）完全不同的度假感。

**最小骨架**：京都 2–3 → 大阪 2 → 白滨 2D1N–3D2N → 返程

**评级**：

档期：
- 推荐：`early_summer_low` / `high_summer_low`（沙滩 + 海浪）
- 一般：`autumn_low`（海边秋季适合散步）/ `deep_winter`（蟹季）
- 不推荐：`sakura_peak` / `momiji_peak`（浪费更强档期） / `tsuyu_ajisai`（梅雨沙滩废）

天数：
- 推荐：8–11 天
- 一般：7 天（紧凑）
- 不推荐：6 天以下（往返成本高）

人群：
- 推荐：romantic / family_fun
- 一般：classic
- 不推荐：photogenic（白滨没京都/大阪那么出片）

---

### K44 · 亲子标配

**核心承诺**：带娃行程的**系统性调整**——避开体力型景点、加入水族馆/乐园、降换城频率。不是"K01 的亲子版"，是独立结构。

**最小骨架**：京都 2.5（铁道博物馆+水族馆+任天堂博物馆）+ 奈良 0.5（喂鹿）+ 大阪 3.5–4（USJ+海游馆+天保山+梅田摩天轮）

**评级**：

档期：
- 推荐：`early_summer_low` / `autumn_low` / `early_spring_low`
- 一般：`sakura_peak`（虽然樱花好看但人潮对娃不友好）/ `momiji_early`
- 不推荐：`high_summer_low`（7/15–8/20 高温）/ `momiji_peak`（人潮挤压）/ `deep_winter`（娃怕冷）

天数：
- 推荐：5–7 天
- 一般：8 天
- 不推荐：4 天以下 / 9+ 天（娃撑不住长线）

人群：
- 推荐：family_fun（本路线专属）
- 一般：无
- 不推荐：非亲子（这条不是为他们设计）

---

### K42 · 吉野 + 高野精神线

**核心承诺**：两座世界遗产山（吉野山 + 高野山）+ 可选熊野古道——关西最深度的精神文化路线。

**最小骨架**：京都 2–3 → 吉野山 1 → 高野山 1N2D → 大阪 1–2（→ 可衔接熊野古道 1 日）

**评级**：

档期：
- 推荐：`sakura_peak`（吉野山樱季）/ `momiji_peak`（高野山红叶）/ `autumn_low`（徒步舒服）
- 一般：`early_spring_low` / `early_summer_low`
- 不推荐：`tsuyu_ajisai`（梅雨）/ `high_summer_low`（高温）/ `deep_winter`（高野山大雪）

天数：
- 推荐：10–13 天
- 一般：9 天（紧凑）
- 不推荐：8 天以下（塞不进）

人群：
- 推荐：classic / 深度文化 / photogenic
- 一般：romantic（精神型蜜月）
- 不推荐：family_fun / 带老人 / 带幼儿（高野山海拔 + 吉野徒步）

## 二、插件（Plugins）

插件不是独立路线，是**可挂载到主路线上的体验模块**。插件有自己的"激活条件 + 适配评级"。

## 三、匹配流程

### Step 1：硬筛主路线

用户输入 → 按档期/天数/人群**过滤掉"不推荐"**的主路线 → 剩下候选 N 条（通常 2–5 条）

### Step 2：硬筛插件

同样方式过滤插件 → 得到可激活插件池

### Step 3：Opus 软匹配

Opus 读候选主路线 + 可激活插件 + 用户 notes，输出：
- 选一条主路线（如果候选 ≥2 条）
- 挂载 N 个合适插件（受互斥 + 天数约束）
- 写人话解释"为什么这条路线 + 这几个插件"

### Step 4：展开

选定的主路线骨架 + 挂载插件 → 按天展开成具体的 day template 序列 → 交给后续装配（餐厅、酒店、具体景点时段）
