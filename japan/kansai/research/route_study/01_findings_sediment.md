## 一、5–7 天关西路线：真实存在的主流组合

### 5 天

- **类型 A**：京都 3 + 奈良 1 日归（多源一致，japan-guide / Funliday / 换乘案内の案内君）
- **类型 B**：京都 2.5 + 大阪 1.5（乐吃购系列推荐）

**关键事实**：
- "京都 3 + 大阪 2" 在各源中是新手首选
- 城市顺序 95% 是 "先京都后大阪"，理由是 KIX→京都 HARUKA 75min 直达、最后大阪住一晚方便返程
- 唯一例外：学生低预算方案可能"大阪进出"（航班价差）

### 6 天

- **主流**：京都 3 晚 + 大阪 3 晚 + 中插奈良一日 或 姬路半日（japan-guide、Klook、乐吃购、KKDAY 一致）
- **次流**：京都 3 + 大阪 2 + 神户 0.5–1（有马温泉或港口夜景）
- "3+3 分住" 是被评论区反复模仿的分法

### 7 天（最主力区间）

四类组合都有代表作：
- **A. 双城主线**：京都 3 + 大阪 3 + 其他 1（奈良 or 姬路）—— 10+ 源覆盖
- **B. 奈良独立成城**：京都 3 + 奈良 1 + 大阪 2.5
- **C. 高野山宗教版**：京都 2.5 + 高野山 1 + 大阪 2.5 + 其他 1
- **D. 京都 + 神户线**：京都 3 + 大阪 2 + 神户 1

**7 天里"先京都后大阪"是绝对共识**（机场路由 + 情绪递进：古都→现代收尾）。

## 二、8–13 天 + 延伸城市

### 8–10 天主流

- 官方（japan-guide）10 天标配：京都 3 + 奈良 1 + **高野山 1（宿坊）** + 大阪 2
- 即：**10 天加的第四城多数是高野山，不是神户**——这和一般印象相反
- 次流：京都 + 大阪 + 奈良 + 神户（传统版）
- 新兴：京都/大阪基地 + 有马一日或 2D1N

### 11–13 天出现"家族分化"

- **温泉家族**：基础 + 有马 + 城崎（或白滨）
- **文化朝圣家族**：京都深度 + 高野山 + 熊野古道
- **海岸度假家族**：基础 + 白滨（带孩子友好）
- **大满贯家族**：13 天塞全，读者评价多为"太赶"

| 城市 | 核心承诺 | 最少天数 | 关键时段 | 谁适合 |

### 机场共识

- 95%+ 攻略用 KIX 进出
- 伊丹（ITM）只适合国内联程
- 神户机场（UKB）游客极少
- 大阪南线（和歌山、高野山、白滨）仍以 KIX 为锚

## 三、季节变体

### 樱花季（3 月底–4 月中）

**专属设计核心**：利用吉野山海拔递进延长花期（京都早 / 奈良中 / 吉野晚 7–10 天）

**时间窗口**（精确到周）：
| 地点 | 预计开花 | 满开 |
|---|---|---|
| 京都市区（清水寺/圆山/哲学之道） | 3/27 前后 | 3/30–4/3 |
| 奈良公园 | 3/28 前后 | 4/1–4/5 |
| 岚山 | 3/30 前后 | 4/3–4/7 |
| 吉野山（下→中→上→奥） | 4/2 前后 | 4/7–4/15（递进） |
| 背割堤 | 3/30 前后 | 4/2–4/7 |

**樱花季应避开的地点**：清水寺日间（人海）、哲学之道中段；**替代**：龙安寺、半木之道、原谷苑

### 红叶季（11 月中–12 月初）

**专属设计核心**：**夜间特别参拜**大幅上升权重，路线要重分配（日间山水 + 夜间灯火）

**时间窗口**：
| 地点 | 最佳 | 高峰人潮 |
|---|---|---|
| 贵船神社 | 11/初–11/中 | — |
| 永观堂夜枫 | 11/中 | 11/18–11/26 |
| 东福寺 | 11/中 | 11/18–11/26 |
| 高雄三寺 | 11/中–11/下 | — |
| 岚山 | 11/下–12/初 | 11/22–12/2 |
| 琉璃光院（仅秋季开放） | 11/中–11/下 | 11/18–11/26 |

**冷门替代**：圆光寺、曼殊院、光悦寺（知乎私藏清单）

### 盛夏（7–8 月）

**核心是祭典，不是避暑**：
- **祇园祭山鉾巡行**：7/17 （前祭）、7/24（后祭）—— 需提前 5–6h 占位
- **宵山**：7/14–7/16（前夜，氛围最浓）
- **天神祭**：7/25（大阪，陆渡御 + 船渡御 + 花火）
- **淀川花火**：8 月第 1 个周六
- **纳凉床**：鸭川、贵船（仅 5–9 月运营）

**应避开**：伏见稻荷正午（户外长楼梯 + 高温）、岚山竹林正午

### 冬季（12 月中–2 月）

**关西冬季的核心不是京都，是温泉**：
- **城崎温泉 11/中–3/初 松叶蟹季**——这是城崎独立成为主角的唯一窗口
- **有马温泉** 金泉疗效冬季最佳
- 京都景点权重下降，除非赶上初雪金阁

### 梅雨（6 月）

**不是淡季，是独家季**：
- **大阪府民之森**：25000 株绣球花，6/中–7/中
- **京都三千院 / 贵船**：绣球 + 纳凉床组合

## 四、美学与体验设计维度

### 1. 时间轴的精细切割

- **朝观光 vs 夜观光**（京都官方网站有专题）：清水寺 6:00 vs 17:00 vs 21:00 是三种体验
- **五食时制**：朝食→昼食→三时の菓子→晚食→夜食（そうだ京都行こう的设计）
- **终电后京都**：00:34 深夜伏见稻荷是独立的"精神空间"
- **樱花/红叶微五阶段**：蕾绽→五分咲き→満開→花吹雪→葉桜（每段体验截然不同）

### 2. 感官设计（95% 游客只用视觉）

- **SOUND TRIP**（京都官方）：三千院用"声明 + 鸟鸣 + 水声 + 音無の滝"做听觉优先的路线
- **香りでめぐる京都**：寺院檀香 / 炭火 / 雨后苔
- **五感で感じる涼**：夏季不是"避暑"而是"涼の身体化"（风铃 + 冰咖啡 + 庭水）
- **鸭川音环境**："適度なノイズ"的留白品质基础

### 3. 人的维度（临时归属感）

- **女将文化**：旅馆女将协会（京都 みやこ女将の会）不是"接待"，是"文化纳入"
- **职人遇见**：Black Tomato 与京都陶艺家建立长期合作，让客人在工作中遇见
- **いけず文化体验**："この先いけずな京町家"沉浸式活动把文化差异变成游戏

### 4. 留白与未完成

- **ぼーっと場所**：鸭川三条–五条 Delta、京都御苑 24h 草地
- **町家的"余白"美学**：有意保留空房间让居住者参与意义创造
- **未完成感设计**：女将故意"下次再介绍你那个"，制造 ongoing relationship

### 5. 深层美学

- **枯山水的无**：龙安寺 15 石永远看不全
- **借景**：修学院离宫三重借景（近 茶室 / 中 池 / 远 比叡山）
- **物哀**：专项"落樱 / 落叶"路线，目击消失的过程
- **幽玄**：高台寺夜间拝観的"言葉にならない美しさ"
- **72 候**：把春夏秋冬细化为 36 个微季节

### 6. 收尾设计

- **伏见酒蔵小路**：最后一餐在杜氏家听日本酒故事而不是在机场 MUJI
- **Black Tomato Feeling Engine**：旅程末尾回归开始时的感觉
- **女将の餞別**：离别时约定下一次的具体季节 + 具体地点

## 五、我们当前 70 条模板的共性缺口（搜索后看清的）

**按重要性排序**：

1. **时间轴粒度粗**：模板只切朝/昼/夜三段，缺"黎明 45min / 终电后 / 黄昏 17min" 这类微时刻模板
2. **感官单一**：几乎全部是视觉主导，没有"以声音/气味/触感为主线"的模板
3. **"引き算"模板缺失**：没有纯留白型（2h 什么都不做）、没有"意图迷路"型
4. **人的遭遇设计薄**：和女将 / 职人 / 本地人的互动几乎没在模板里结构化
5. **季节粒度粗**：只有 sakura / koyo / summer / winter，缺 "樱花五阶段"、"红叶早中晚期"、"72 候"
6. **没有档期独立主角**：城崎松叶蟹只是 `onsen` 标签，但它其实在 11/中–3/初 是独立主角线路
7. **收尾设计缺失**：所有 departure_* 模板都是"购物 + 返程"，没有"最后一餐/最后画面"的仪式化设计
8. **没有复购层**：所有模板都是"首次来"视角，没有"二回目 / 三回目"的深度变奏

## 六、本次搜索的元反思

### 搜索假设的错误
我和 GPT 最初的假设是"找关西 7 天的冠军路线"，搜出来才发现：
- 即便在同一个"7 天 × 4 月"条件下，也有 A/B/C/D 四种都被读者认可
- 评论区"照抄了"的路线也不止一条
- 作者自己也经常说"另一种走法也行"

### 真相
**关西路线不存在单一冠军**。优秀路线是**匹配条件（档期 × 天数 × 人群 × 特殊诉求）下的若干合格成品的平权集合**。

### 设计后果
- 不做"选第一名"，做"筛合格集合"
- 候选集合可以有 1–5 条（条件越独特候选越少）
- 用户下次复购自然指向集合里另一条（复购价值天然存在）
