# 酒店区域策展 md 写作规范

> 适用 `assembly/hotels/{kyoto,osaka,other}/*.md` 所有条目。
> 每区域 md = 区域选择叙事（头部）+ 城市酒店轨 + 特色住宿轨 + 条目列表。
> 建于 2026-04-24。

---

## 〇、区域 md 必须有的五段结构

```markdown
# 区域名 酒店策展

## 住这里的人是谁
（第一次来 / 多刷 / 情侣 / 亲子 / 商务；一句话画像 + 典型行程）

## 区域优势
（交通便利度 / 动线覆盖 / 氛围 / 餐饮配套，≤4 点）

## 不该住这里的人
（硬不合适人群 + 原因，避免装配错推）

## 城市酒店（便利型）
### 经济档 / 中档 / 高档 / Showcase
（条目列表）

## 特色住宿（体验型）
### 温泉旅馆 / 町家 / 宿坊
（条目列表，按需出现，无就不写该小节）
```

## 一、事实性字段硬规

### 1.1 价格

- 必须 OTA 真实价格（一休 / 楽天 / 携程），不估算
- 写范围：`¥18,000-¥35,000/晚`
- 含餐/含税标清：`含一泊二食` / `素泊单价`
- 旺季倍率 >2x 要标：`旺季（11 月红叶）翻 2 倍`

### 1.2 位置

- 最近站 + 步行分钟（查 Google Maps 实测，不推断）
- 爬坡 / 绕路 / 末班车早 → 写进 `access_friction`
- 行李友好度（大件行李能否直推）

### 1.3 类型

4 类明确写 `city_hotel` / `ryokan_onsen` / `machiya` / `shukubo` 之一。

### 1.4 档位

`economy` / `mid` / `high` / `showcase` 之一，按 [准入标准.md §一](准入标准.md) 价位表。

### 1.5 MICHELIN Keys

有则写：`MICHELIN Keys：One / Two / Three（202X 年版）`

### 1.6 设施（用自然语言，不列清单）

温泉大浴场 / 私汤 / Spa / 礼宾 / 早餐评价 / 健身房——**按需写**，不是每家凑齐。

## 二、B 视图（引用段）写作约束

### 2.1 禁止项

- 禁止写 OTA / 官网没的比较（"全京都最好的町家"无背书不写）
- 禁止营销话术（"奢华体验"、"完美一晚"）
- 禁止推断价格（"大概 ¥20,000" 查不到就标待核实）

### 2.2 可写项

- 具体设计亮点（檜浴槽 / 町家改造 / 庭院景观房）
- 服务细节（女将接客 / 手写欢迎卡 / 浴衣款式）
- 适合场景（蜜月 / 商务长住 / 温泉深度）
- 用户动作（何时订 / 如何挑房 / 提前通知过敏需求）
- 路线衔接（到 X 景点步行 N 分钟 / 步行可达 Y 车站）

### 2.3 字数

150-300 字。最多 1 个加粗。

## 三、完成度标记

```markdown
<!-- depth: skeleton | verified | full -->
```

| 标记 | 含义 |
|---|---|
| `skeleton` | 旧简版，只有类型/档位/价格/位置 |
| `verified` | 已核营业/价格/Keys，字段栈完整 |
| `full` | verified + B 视图引用段 + 区域叙事可引用 |

## 四、纪律

### 4.1 开工前

- 读 `japan/kansai/当前状态.md` 底部「进行中」段
- 追加一行「我在改 hotels/{城市}/{file}.md」

### 4.2 核实时间盒

- 每条 5-8 分钟封顶（比餐厅略长，因价格多时段）
- 查不到价格标「待核实」不编

### 4.3 SCHEMA 变更

- 想加字段先停，去讨论扩 SCHEMA，不私自加

### 4.4 数据真实性

- 价格 / Keys 年份 / 地址 / 品牌归属必须有源
- 含餐状态 / 旺季倍率 / 房型描述必须 OTA 当前显示
- 保持「真实日本旅行顾问点头」标准

## 五、中文硬规（用户可见字段一律中文）

所有用户可见字段一律中文。日文仅允许：
- 酒店名 / 房型名 / 站名首次出现用「中文（日文）」格式，后续直接中文
- 独立字段 `name_ja` / `name_en`（数据字段，不占文案空间）

### 字段级对照

✅ 正确：
```
类型：温泉旅馆（体验型）
所在：有马温泉金泉町，神户三宫站 + 电铁 30 分钟
价格：¥40,000-¥80,000/晚（含一泊二食，旺季翻 2 倍）
特点：金泉露天风吕 + 会席料理，女将传统接客
B 视图：一进门玄关女将鞠躬接客，榻榻米走廊尽头是金泉露天风吕
```

❌ 错误：
```
类型：温泉旅館（体験型）
所在：有馬温泉金の泉町、神戸三ノ宮駅+電鉄 30 分
价格：¥40,000-¥80,000/泊（一泊二食付、ハイシーズン 2 倍）
特点：金泉露天風呂 + 会席料理、女将の伝統的なおもてなし
B 视图：玄関で女将さんのお辞儀、畳の廊下の先に金泉露天風呂
```

### 专有名词保留白名单（无需翻译）

- 温泉 / 旅馆 / 露天风吕 / 会席料理 / 町家 / 宿坊
- 一泊二食 / 素泊 / 女将 / 朝食
- 一休 / 楽天 / MICHELIN Keys

### 自检

**写完每段读一眼：中国游客第一眼看懂吗？看不懂且没括号补说明 = 改。**

## 六、不编造数据（事实必有源）

所有价格 / Keys 年份 / 地址 / 品牌归属 / 含餐状态 / 房型描述**必须有可追溯的源**。查不到 → 模糊化 / 标「待核实」 / 不写，三选一，**绝不编**。

### 字段级对照

✅ 正确：
```
价格：¥40,000-¥80,000/晚（一休查得到）
Keys：Two Keys（2025 版）（米其林官网可验）
品牌：虹夕诺雅集团（星野 Resort 官网有）
特点：金泉露天风吕 + 一泊二食（OTA 页面列着）
```

❌ 错误：
```
价格：约 ¥50,000/晚（未核 OTA）
Keys：Three Keys（官网是 Two）
品牌：顶级老铺家族经营（不明确归属）
特点：70 年历史（创业年未查）
```

### 查不到的三种处理

1. **模糊化**：「江户时代老铺」替具体年份 / 「高档温泉旅馆」替未查品牌
2. **标「待核实」**：价格 / Keys 年份这类必核项
3. **不写**：品牌归属 / 总经理履历查不到直接省略

### B 视图禁推断

- ❌ "全关西最好的温泉旅馆"（无背书）
- ❌ "比 X 旅馆更好"（跨家比较无源）
- ❌ "房间面积 X 平米"（官网无明示不写具体数字）
- ✅ 可写：现场可见物理特征（玄关女将 / 榻榻米走廊 / 露天风吕位置）

### 自检一句话

**每处数字 / 归属 / 比较句都能指出源吗？指不出 = 改。**

---

## 七、第一性原理（够用即可，不硬凑）

所有字段 / 子标题 / 字数**只是参考上限**，不是下限对齐。判准是「用户按 md 选酒店能不能做决定」，不是「字段凑齐没」。

### 应用

- 有 1 项实打实的亮点就写 1 项，不为凑字数编
- 纯功能商务酒店 B 视图 80 字够了就 80 字，不强写到 200 字
- showcase 级酒店该写详细就写 300 字，不因"区域平均 150 字"砍
- 简单酒店 curators_notes 1 条 or 不写，绝不为"每家 3 条"硬凑

### 自检一句话

**这段信息对用户决策有用吗？没用 = 砍，不因凑字数留。**
