## 一、关西是什么

**关西 = 京都 + 大阪 + 奈良 + 神户 + 周边**——日本最经典的旅行圈,也是中国游客赴日的入门圈。

**地理**:本州岛西部,覆盖 2 府 4 县(京都府、大阪府、兵库县、奈良县、滋贺县、三重县、和歌山县的一部分)。

**核心构成**(按产品权重):
| 圈内位置 | 代表城市 | 产品权重 |
|---|---|---|
| 主骨架(住宿链) | **京都 + 大阪** | 主战场,80% 内容在这两城 |
| 日归插件(5 个) | 奈良 / 神户 / 姬路 / 宇治 / 彦根 | 5+ 天行程 1-2 天挂上 |
| 住宿插件(8 个) | 有马 / 城崎 / 白滨 / 高野山 / 吉野山 / 伊势 / 熊野 / 丹后 | 9+ 天行程加 1-2 晚 |

**特殊地位**:关西机场(KIX)是中国直飞日本的最重要门户之一。南方用户赴日 70%+ 第一次选关西。

## 二、为什么先做关西

**选择优先级的 3 个理由**:

1. **市场最大**:关西是中国游客赴日入门圈,搜索量/游记量/市场需求最大
2. **覆盖面最广**:关西 = 寺庙文化(京都) + 都市消费(大阪) + 自然(有马/高野) + 岛屿(淡路) + 温泉(城崎) —— 能覆盖 90% 日本体验维度
3. **作者最熟**:作者本人在关西研究最深,有原始判断力

**所以关西是**:
- 整个产品的**样板间**——关西做成什么样,其他圈参考
- 架构和 SOP 的**试验田**——4 层架构、7 季节目录、Opus 装配等先在关西跑通
- 内容的**标杆**——v2 的所有模板/事实/研究都以关西为最高标准

## 三、关西的目标用户

**主要用户画像**:
- **年龄**:20-40 岁
- **地域**:中国一二线城市,尤其南方(深圳/广州/上海/杭州)
- **经验**:大多数是**第一次来日本**或**第二次但想换深度**
- **同行方式**:情侣、闺蜜、亲子、默认(朋友/独自)
- **预算**:¥10000-20000 人均(经济档);¥20000-35000(中档);¥35000+(高档)
- **痛点**:
  - 不会日语,怕踩坑(餐厅选错/预约失败/电车走错)
  - 想"既不累又玩得深"(小红书级别但不要特种兵)
  - 出片需求强(情侣/闺蜜档尤其)
  - 怕"像旅行团一样打卡"

**不做**(走客服定制):
- 第二次以上深度游(要小众,模板是大众广谱)
- 奢华游(¥3000+/晚酒店)
- 已订酒店(酒店反向约束动线)

## 四、关西产品的"慢 - 快 - 慢"节奏

关西的产品精髓不在景点列表,在**节奏编排**:

```
大阪(前段):热闹、能量、城市感 — 容错率高,适合时差恢复
京都(中段):安静、深度、文化沉浸 — 需要专注力和体力高峰
神户/收尾(后段):放松、味觉、余韵 — 体力下降后的温柔结尾
```

**铁律**:
- **先京都后大阪**是最优顺序(状态好时沉浸京都,末尾切换大阪漫游)
- **奈良永远作为"桥"**,不独立安排(奈良→大阪 JR 直通 50 分钟,顺路)
- **姬路+神户一天组合**(铁路顺路,各半天)
- **宇治不能和姬路/奈良同一天**(宇治要慢,和大景点组合变走马观花)
- **京都超过 5 天必须切日归模式**(第 4-5 天安排奈良/宇治/嵯峨保持新鲜感)

以上是**关西策展的结构性洞察**,不是规则,但所有关西模板都应该符合。完整洞察见 `research/insights.md`。
