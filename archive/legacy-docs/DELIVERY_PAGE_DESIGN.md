> 档位：248档（含2次精调） / 888档（含5次精调）
> 更新时间: 2026-03-21

## A28. 交付页信息架构（模块顺序 + 每模块展示内容）

> 设计原则：杂志感、大图、暖色、移动端优先、不露出AI

### 模块顺序（自上而下）

| 序号 | 模块名 | 核心展示内容 | 备注 |
|---|---|---|---|
| M1 | **封面大图** | 目的地氛围大图 + 行程标题 + 出行日期 + 人数 + "为你定制" 标签 | 全屏沉浸，无导航栏 |
| M2 | **行程总览卡片** | 天数 · 城市 · 节奏标签（悠闲/紧凑）· 主题标签（美食/文化/亲子…） | 一屏看完核心信息 |
| M3 | **每日行程时间轴** | 每天 4-6 个时间块，景点卡片含大图+一句话推荐理由+开放时间+门票+步行距离 | 核心模块，占比最大 |
| M4 | **餐厅精选** | 每日午/晚餐推荐，含 Tabelog 评分、人均、菜系、一句话点评 | 248档含；点评像专业判断 |
| M5 | **酒店推荐** | 酒店卡片：大图+价位+区域+推荐理由+预订链接 | 248档为精选列表，888档含比价 |
| M6 | **交通指南** | 每日交通方式+时间+交通卡/通票建议+省钱方案 | 详细到"几号线几号出口" |
| M7 | **出行前准备** | 签证·行李·换汇·通讯·APP 清单 | 实用工具型内容 |
| M8 | **避坑指南** | 常见坑点+应对方案，按场景分类 | 248档基础版，888档深度版 |
| M9 | **拍照攻略** | 最佳机位+时间+构图参考（888档专属或248加价项） | 888档标配 |
| M10 | **预算总览** | 总花费估算+每日分项明细（门票/餐饮/交通） | 图表可视化 |
| M11 | **精调入口** | "对行程有想法？来一次精调" 按钮 + 剩余精调次数 | 始终浮动在底部 |
| M12 | **升级提示（仅248）** | PlanB 888档升级引导卡片 | 仅248档展示 |

## A29. 默认展示 vs 折叠模块划分

> 原则：首屏必须"哇"，往下滑有节奏感，工具型内容折叠不打断阅读流

### 默认展开模块

| 模块 | 理由 |
|---|---|
| M1 封面大图 | 第一视觉冲击 |
| M2 行程总览 | 30秒了解全貌 |
| M3 每日行程时间轴 | 核心价值，必须完整展开 |
| M4 餐厅精选 | 高感知价值，"有餐厅推荐"是购买动机之一 |
| M5 酒店推荐 | 同上 |
| M11 精调入口 | 底部浮动按钮，始终可见 |

### 默认折叠模块（点击展开）

| 模块 | 折叠态展示 | 理由 |
|---|---|---|
| M6 交通指南 | 显示摘要"本次行程推荐购买 Suica + 地铁3日券，预计省¥340" | 工具型，按需查看 |
| M7 出行前准备 | 显示"签证·行李·换汇·通讯 4大板块已就绪 →" | 出行前再看 |
| M8 避坑指南 | 显示"已整理12条避坑提醒 →" | 补充内容 |
| M9 拍照攻略 | 显示"已标注18个最佳拍摄点 →" | 888档默认展开，248档折叠 |
| M10 预算总览 | 显示"预计总花费 ¥XXXX/人 →" | 敏感信息，让用户主动查看 |
| M12 升级提示 | 卡片形式嵌在M8后面 | 不喧宾夺主 |

### 折叠交互规范

- 折叠态：单行摘要 + 右侧箭头 `›`
- 展开动画：高度过渡 300ms ease-out
- 展开后不自动收起
- 每个折叠区顶部有锚点，目录导航可跳转

## A30. PlanB 设计逻辑 + 交付页内升级入口（888）

> 核心思路：248用户看到交付物后，自然产生"要是还能XX就好了"的念头，升级入口恰好出现

### PlanB 设计逻辑

#### 什么是 PlanB
- 248档交付物本身已经"够用"，但在关键场景留出**感知缺口**
- PlanB 不是"你的方案不好"，而是"你可以更好"
- 升级话术永远正向："解锁更多可能"，不说"你的版本缺了XXX"

#### 感知缺口设计（248 → 888 的升级触发点）

| 触发场景 | 248档展示 | 888档增量 | 用户心理 |
|---|---|---|---|
| 酒店推荐 | 精选3-5家 + 推荐理由 | +多平台比价 + 节省金额 | "能帮我比价省钱？" |
| 机票信息 | 无 | 多平台比价 + 最优方案 | "机票也能帮我省？" |
| 精调次数 | 2次 | 5次 | "改了2次还想再调..." |
| 拍照攻略 | 折叠/无 | 每景点拍摄点+机位+时间 | "好想知道怎么拍好看" |
| 行前答疑 | 无 | 1次集中答疑 | "出发前还有问题想问" |
| 省钱报告 | 无 | 详细对比+预计节省金额 | "花888能省5000？" |

#### 升级定价逻辑
- 248 → 888 升级价：**¥640**（补差价）
- 已用精调次数不清零，升级后额度变为5次（已用次数保留）
- 升级后48h内交付888版增量内容

### 交付页内升级入口（3处）

#### 入口1：酒店模块底部（最自然）
```
┌─────────────────────────────────────┐
│  💡 想知道这几家酒店哪个平台订最便宜？ │
│                                     │
│  升级到深度定制版，帮你多平台比价，     │
│  平均能省 ¥2000+                     │
│                                     │
│  [查看深度定制版能帮你做什么 →]        │  ← 不写价格，先看价值
└─────────────────────────────────────┘
```

#### 入口2：精调次数用完时（刚需触发）
```
┌─────────────────────────────────────┐
│  你的2次精调已用完                     │
│                                     │
│  升级深度定制版，获得额外3次精调机会，   │
│  还能解锁机票比价+拍照攻略              │
│                                     │
│  [了解升级方案 →]                     │
└─────────────────────────────────────┘
```

#### 入口3：页面底部升级卡片（M12 模块）
```
┌─────────────────────────────────────┐
│  🎯 你的行程已经很棒了！               │
│  还可以更进一步 ——                    │
│                                     │
│  ✅ 机票酒店多平台比价（平均省¥3000+）  │
│  ✅ 每个景点的拍照机位+最佳时间         │
│  ✅ 5次精调机会（还剩3次可用）          │
│  ✅ 出发前1对1答疑                    │
│                                     │

## A31. 交付页视觉风格规范

> 关键词：杂志感 · 大图 · 暖色 · 移动端优先 · 不露AI

### 色彩系统

| 角色 | 色值 | 用途 |
|---|---|---|
| **主色-暖橙** | `#F2994A` | CTA按钮、重点标签、精调入口 |
| **辅色-樱粉** | `#F5C6C6` | 卡片背景点缀、季节氛围 |
| **辅色-抹茶绿** | `#6B9E78` | 交通/省钱相关标签 |
| **深底色** | `#2D2926` | 封面标题文字、正文标题 |
| **正文色** | `#4A4A4A` | 正文/描述 |
| **浅底色** | `#FFF8F3` | 页面背景（暖白，非纯白） |
| **卡片底色** | `#FFFFFF` | 内容卡片 |
| **分割线** | `#F0EBE6` | 模块间分隔 |

### 字体规范

| 层级 | 字号（移动端） | 字重 | 行高 | 用途 |
|---|---|---|---|---|
| H1 | 28px | Bold | 1.3 | 封面标题 |
| H2 | 22px | Bold | 1.4 | 模块标题（"Day 1 · 东京初印象"） |
| H3 | 18px | SemiBold | 1.4 | 景点名/餐厅名 |
| Body | 15px | Regular | 1.7 | 正文描述/推荐理由 |
| Caption | 13px | Regular | 1.5 | 开放时间/门票/地址 |
| Tag | 12px | Medium | 1.2 | 标签（"Tabelog 3.8"） |

- 中文字体栈：`"Noto Sans SC", "PingFang SC", "Hiragino Sans GB", sans-serif`
- 日文回退：`"Noto Sans JP", "Hiragino Kaku Gothic Pro"`

### 图片规范

| 场景 | 比例 | 最小宽度 | 处理 |
|---|---|---|---|
| 封面大图 | 16:9 → 全屏 | 1080px | 底部渐变遮罩叠加标题 |

## A32. 用户端可修改项清单（结构化交互）

> 核心原则：用户不能自由输入 prompt，只能通过按钮/选项进行结构化修改
> 每次精调 = 选择修改项 → 提交 → 48h内收到更新版

### 可修改项总表

| 序号 | 修改类型 | 交互方式 | 修改粒度 | 示例 |
|---|---|---|---|---|
| 1 | **换景点** | 点击景点卡片 → "换一个" → 弹出3个备选 | 单个景点 | Day2下午的景点从"明治神宫"换成"代代木公园" |
| 2 | **换餐厅** | 点击餐厅卡片 → "换一家" → 弹出3个备选（同区域/同菜系） | 单个餐厅 | Day3午餐从寿司换成拉面 |
| 3 | **调节奏** | 选择"更悠闲"/"更紧凑"/"当前刚好" | 全局或单日 | "Day4太赶了，想轻松点" |
| 4 | **调整天数分配** | 选择"东京多待1天"/"京都多待1天" | 城市间天数 | "想在京都多待一天" |
| 5 | **换酒店区域** | 选择新区域（下拉菜单） | 住宿区域 | 从"新宿"换到"浅草" |
| 6 | **调预算倾向** | 拖动滑块"省钱 ←→ 体验优先" | 全局 | 提升酒店预算，降低餐厅预算 |
| 7 | **换主题偏好** | 多选标签（文化/美食/购物/自然/拍照/亲子） | 全局 | 增加"拍照"权重 |
| 8 | **跳过某天** | 点击某天 → "这天我有自己安排" | 单日 | Day5自由活动 |

### 修改交互流程

```
用户点击 [精调一下] 浮动按钮
  ↓
弹出精调面板（底部抽屉）
  ↓
选择修改类型（上述8种，卡片式展示）
  ↓
根据类型展示对应交互：
  · 换景点 → 选择哪天哪个景点 → 展示3个替代选项（含图+一句话理由）
  · 换餐厅 → 选择哪天哪餐 → 展示3个替代选项
  · 调节奏 → 3档滑块 + 可选"只调某天"
  · 其他 → 对应结构化选项
  ↓
用户确认修改项
  ↓
显示"已收到你的精调需求，预计24h内更新"
  ↓
精调计数 -1，显示剩余次数
```

### 每种修改的备选项展示规范

#### 换景点/换餐厅的备选卡片
```
┌──────────────────────────┐
│  [缩略图]                 │
│  ✦ 代代木公园              │
│  "比明治神宫更适合野餐发呆， │
│   春天有大片樱花"           │  ← 推荐理由 = 和原景点的差异化
│  🚶 步行距离不变            │  ← 标注对行程的影响
│  [选这个]                  │
└──────────────────────────┘
```

#### 调节奏的交互
```
Day 4 · 当前安排 5 个景点

  😌 悠闲     🚶 适中     🏃 紧凑
  (3个景点)   (4个景点)    (6个景点)
     ○          ●            ○      ← 滑块选择

  [只调这一天]  [全部天数都调]
```

### 不可操作项（无修改入口）

- 出发/返回日期（已锁定）
- 出发城市（已锁定）
- 行程天数（需要联系客服调整）
- 同行人数变更（涉及价格变动）

## A33. 不可修改项 + 修改次数限制 + 文案包装

> 原则："2次精调"而非"限2次"；正向表达，让用户觉得是权益而非限制

### 不可修改项

| 项目 | 不可修改原因 | 前台展示方式 |
|---|---|---|
| 出发/返回日期 | 影响机票/酒店/季节因素，牵一发动全身 | 不展示修改入口，显示为确认信息 |
| 出发城市 | 影响机票方案 | 同上 |
| 行程总天数 | 影响整体路线架构 | 底部小字"天数调整请联系定制师" |
| 同行人数 | 影响酒店房型和预算 | 同上 |
| 整体目的地（如从日本改韩国） | 完全不同的产品 | 不展示 |

### 精调次数机制

| 档位 | 精调次数 | 每次精调范围 |
|---|---|---|
| 248档 | **2次精调** | 每次可改1-3个修改项 |
| 888档 | **5次精调** | 每次可改1-5个修改项 |

#### 单次精调的修改项数量限制
- 248档：每次精调可提交 **1-3个修改项**
- 888档：每次精调可提交 **1-5个修改项**
- 超出视为"大改"，引导升级或加价（+¥69/次）

### 文案包装规范

#### ❌ 限制性表达 vs ✅ 权益性表达

| ❌ 不要这样说 | ✅ 要这样说 |
|---|---|
| "限2次修改" | "含2次专属精调" |
| "修改次数已用完" | "你的2次精调已全部使用" |
| "不可修改" | "如需调整请联系定制师" |
| "超出修改次数" | "解锁更多精调机会" |
| "修改限制" | "精调额度" |
| "还剩1次" | "还有1次精调机会" |
| "最后1次" | "珍贵的最后一次精调，想好了再用哦" |

#### 精调次数展示文案（按状态）

**状态1：未使用**
> 🎯 你有 **2次专属精调** 机会，可以换景点、换餐厅、调节奏
> 看完行程后，随时来调

**状态2：用了1次**
> ✅ 第1次精调已完成 · 还有 **1次精调** 机会
> 这次想调哪里？

**状态3：全部用完（248档）**
> ✅ 你的2次精调已全部使用
> 行程已经根据你的偏好优化过了！
> 如果还想继续调整 → [了解升级方案] / [单次加购 ¥69]

**状态4：全部用完（888档）**
> ✅ 你的5次精调已全部使用
> 这份行程已经打磨得非常好了！
> 如果还有想法 → [联系定制师]

### 大改 vs 精调的边界

| 类型 | 定义 | 计入精调？ | 处理方式 |
|---|---|---|---|
| **精调** | 在现有框架内微调（换1-3个景点/餐厅、调节奏） | ✅ 计入 | 结构化提交 → 48h更新 |
| **大改** | 改变路线架构（如增删城市、改天数、改出行方式） | ❌ 不计入 | 引导联系客服/加价¥69 |
| **纠错** | 信息错误（营业时间过期、景点已关闭） | ❌ 不计入 | 免费修正，不扣精调次数 |

## A34. 内部审核工作台模块设计（目标5分钟完成一单）

> 核心原则：结构化切换为主，提示词补充为辅。工作台与用户前台严格分离。
> 设计目标：一个熟练操作员 5 分钟完成一单审核 + 精调处理

### 工作台总览

#### 界面布局（桌面端三栏）

```
┌──────────┬───────────────────────────┬──────────────┐
│  订单队列  │       行程编辑区           │   操作面板    │
│  (左栏)   │       (中栏·主区)          │   (右栏)     │

## A35. 修改入口文案（5条）

> 原则：温暖、像朋友在帮你、不施压、暗示专业性

### 文案1：浮动按钮（常驻底部）

> **精调一下** ✨
> 想换个景点或调调节奏？

*设计说明：最简洁，像微信聊天的语气，✨增加温度感*

### 文案2：行程末尾引导

> **看完了？觉得哪里还可以更好？**
> 你有 {n} 次精调机会，可以换景点、换餐厅、或者让某天更悠闲一些。
> 不着急，想好了再来 👉 [开始精调]

*设计说明：不施压，"不着急"降低决策焦虑*

### 文案3：景点卡片上的微交互

> 🔄 **不太喜欢这个？换一个试试**
> 点击查看 3 个同区域备选

*设计说明：直接出现在每个景点卡片右上角，最直觉的入口*

### 文案4：精调完成后的确认页

> ✅ **已收到你的精调需求！**
> 我们的定制师会在 24 小时内更新你的行程。
> 更新后会通过微信通知你，记得来看看新版本哦 😊

*设计说明："定制师"而非"系统"，维持专人感；24h设预期*

### 文案5：精调次数即将用完的温馨提示

> 🎯 **这是你的最后一次精调机会**
> 建议把想调的地方一次说完哦！
> 可以同时换景点、换餐厅、调节奏——最多 3 项，让我们帮你一次搞定。

*设计说明：暗示"用好这次"，引导一次性提多个需求避免浪费*
