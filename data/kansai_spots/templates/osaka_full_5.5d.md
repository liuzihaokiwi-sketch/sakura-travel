# 大阪模板

> 架构：2.5天核心基座 + 可插拔模块
> 核心基座：Day 1（到达）+ Day 2（精华日）+ 最后半天（切走）
> 可插拔模块：USJ（+1天）/ 人群日（+1天）/ 季节活动日（+1天）
> 系统从基座往上拼模块，不是从满版往下删

> 用户只做3个显式选择：
> 1. USJ要不要
> 2. 人群类型（情侣/闺蜜/亲子）
> 3. 预算档位（经济/中档/高档）
> 其余全部系统定，包括餐厅、酒店、店铺、活动安排

## 模块拼装规则

核心基座（2.5天）固定不动，模块按条件拼上去：

```
核心基座 2.5天：到达日 + 精华日 + 最后半天
  + USJ模块（用户选了USJ）→ 3.5天
  + 人群日模块（天数够4.5天+）→ 4.5天
  + 季节活动模块（日期匹配且需要独立一天）→ 5.5天
```

| 用户大阪天数 | 核心基座 | +模块 |
|-------------|---------|-------|
| **2.5天** | 到达+精华+切走 | 无 |
| **3.5天** | 同上 | +USJ |
| **4.5天** | 同上 | +USJ +人群日 |
| **5.5天** | 同上 | +USJ +人群日 +季节活动日 |

不需要独立一天的季节活动（花火、光のルネサンス、箕面红叶）作为overlay加到现有天的晚上/上午，不占天数。
需要独立一天的（天神祭、岸和田花车祭、大相扑、樱花季造幣局+大阪城）才拼为独立模块。

## 用户决策（仅3个，其余系统定）

1. **USJ**：要去 / 不去
2. **人群类型**：情侣 / 闺蜜好友 / 亲子 / 其他
3. **预算档位**：经济 / 中档 / 高档

以下由系统自动决定，不问用户：
- 酒店：系统根据天数+预算+人群选最匹配的一家
- 餐厅：系统推荐A（安全牌），B（惊喜牌）作为备选
- 晚间体验：系统根据人群推荐（情侣/闺蜜→爵士酒吧，其他→心斋桥逛街）
- 店铺：系统根据兴趣标签推荐
- 季节活动：系统根据日期自动加入
