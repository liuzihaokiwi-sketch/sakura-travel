# 精品店采集 -- 品类特有规则

> 前置阅读：00_playbook.md
> 更新：2026-04-02 基于关西实战

---

## 核心定位

手账本的"惊喜层"，不是骨架。游客不会专门为一家店规划行程，而是**逛到某个区域时顺便逛**。

所以按**区域**分文件，不按类型分。AI编排行程时知道用户今天逛东山，就查该区域有什么宝藏店。

## 不要做什么

1. **不要用餐厅/酒店的思路做精品店** -- 没有Tabelog，没有OTA评分，没有统一排名源
2. **不要只搜小红书** -- 小红书覆盖热门但偏，日文专业站覆盖深（古着店VINTY有全京都49家索引）
3. **不要收连锁** -- 3COINS/Loft/无印良品不算"宝藏"，连锁店AI自己知道
4. **不要给grade打分** -- 精品店用 must_visit / recommended / worth_checking 三档就够

## 补充来源

- 中文（小红书搜索）: "京都必逛 古着 杂货 宝藏店铺" -- 标题含店名线索
- 中文（知乎/豆瓣）: "京都购物""大阪中古店"
- 台湾/香港: ELLE HK中古店特集、乐吃购购物专题
- 奈良/神户要单独搜 -- 大词搜京都大阪为主，小城市会被淹没

## 文件组织

按区域分，不按类型分：
```
shops/
  {city}_{area}.json     核心逛街区域各一个文件
  markets.json           市集/蚤の市单独（跨城市，按日期查询）
```

关西为例：kyoto_kawaramachi / osaka_amemura / osaka_nakazaki_horie / kobe / nara / markets

## 精品店特有教训

1. **日文专业站是金矿** -- VINTY/MEGURU/thisismedia/recoya这些，一个站就能出几十家带详情的数据
2. **区域集中度很高** -- 古着店集中在3-4个街区（河原町/美国村/中崎町/堀江），分文件时自然会不均匀
3. **市集要记日期** -- 弘法市每月21日、天神市每月25日、百万遍每月15日，AI排行程需要知道
4. **奈良有独特的店** -- 中川政七商店(1716年)/吉田蚊帳/春鹿酒蔵，不搜就完全漏掉
5. **商店街是体验不是店** -- 锦市场/新京极/Orange Street本身就是POI，不只是"有店"
