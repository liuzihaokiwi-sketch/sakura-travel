# Spec: pdf-delivery-ux

## 概述

定义 PDF 为正式交付主形态的策略、水印设计规范，以及三处 PDF 说明文案（价格页/成功页/交付页）的内容规范。

## 交付形态定位

| 形态 | 定位 | 场景 |
|------|------|------|
| PDF | **正式交付主形态** | 离线查看、发给同行人、打印留存 |
| H5 网页版 | 辅助补充 | 手机随时查阅、行程更新后同步 |

**为什么 PDF 是主形态：**
- 用户心理锚点："我有一份专属文件"比"我有个链接"分量更重
- 飞机/地铁/信号差场景下可查看
- 发给同行家人/朋友不需要注册账号
- 有清晰的版本感（用户知道"这是最终版"）

## 水印设计规范

**水印策略：轻水印，角落式，不覆盖正文**

| 要素 | 规范 |
|------|------|
| 位置 | 每页页脚中央 |
| 颜色 | 灰色，`#999999`，透明度 60% |
| 字号 | 8pt，比正文小很多 |
| 内容 | `日本定制攻略 · 专属制作 · 仅供本人非商业使用 · 订单 #XXXX` |
| 是否对角线满铺 | ❌ 禁止，破坏阅读体验 |
| 是否包含手机号 | 可选：仅后 4 位，不暴露完整号码 |

**水印定性原则：**
- 定性为"个性化标识"，不是"防盗版警告"
- 语气是"这是你的专属版本"，不是"禁止转发"
- 用户在购买前已知情，不是悄悄加的

## 三处 PDF 说明文案规范

### 1. 价格页小字（最简洁，不打断决策）

**位置：** 价格卡下方，页面小字的第二句

**文案：**
> 行程以 PDF 形式交付，方便离线查看。PDF 附带轻水印（订单号 + 仅供本人使用），不影响正常阅读。

**要求：**
- 一句话说完，不展开
- 先说对用户有利的（"方便离线"），再说水印
- 不用"版权保护"、"防止扩散"等防御性语言

---

### 2. 提交成功页（完整说明，用户注意力充足）

**位置：** 提交成功后的说明区块，紧接在"接下来的步骤"列表下方

**文案：**
> **关于 PDF 交付：**
>
> 行程制作完成后，PDF 正式版会发送到你的微信。
>
> - 可以离线查看，手机、平板、电脑均可打开
> - 方便直接发给同行的家人或朋友
> - 附带轻水印（订单编号），作为你的专属版本标识，不影响阅读
> - 仅供本人非商业使用

**要求：**
- 用列表而不是段落，扫读友好
- "专属版本标识"是正向定性，不是限制

---

### 3. 交付页（操作指引式，用户已拿到 PDF）

**位置：** 交付页 PDF 下载/查看区块下方

**文案：**
> **关于这份 PDF**
>
> - 可以直接发给同行的家人或朋友，无需对方注册
> - 建议出发前在手机本地保存一份，不依赖网络
> - 附带轻水印（订单号 + 仅供本人使用），不影响阅读内容
> - 如需调整行程，直接微信告知，我们更新后重新发送

**要求：**
- 操作指引优先，说明约束在后
- 最后一条（可微调）降低用户"拿到了就结束了"的焦虑

## 用户接受水印的心理路径

1. **知情在前**：付款前页面已明确说明有水印 → 无惊吓感
2. **轻量不打扰**：水印在页脚角落，阅读时几乎感知不到
3. **正向定性**：文案说"专属版本标识"而不是"防盗水印" → 从限制变成身份感
4. **实际使用无障碍**：可以发给同行人，可以打印，可以截图 → 水印不影响任何正常使用场景
