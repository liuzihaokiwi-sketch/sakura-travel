# 用户表单与方案确认页设计（最终版）

> 付费后填写。4屏轻表单 → 方案确认页 → 预算确认页。
> 核心原则：用户付费买的是"专家替你决策"，不是旅行编辑器。

---

## 完整流程

```
屏1：什么时候去
屏2：旅行感觉 + 几位出行
屏3：节奏
屏4：特殊情况（可跳过）
→ 系统生成方案
→ 方案确认页（行程 + 可加体验 + 纠偏）
→ 预算确认页（吃住档位 + 加购 + 总费用）
→ 确认下单，开始制作手账本
```

- ○ 约会感 — 浪漫·夜景·有情调的餐厅·安静的两人时光
- ○ 出片感 — 好看的地方·咖啡甜品·古着·逛逛拍拍
- ○ 亲子感 — 孩子玩得开心最重要·轻松不赶

## 第二部分：方案确认页

> 整个产品体验的核心价值页。用户看到"有人真的帮我排好了"。

### 区域1：条件回显 + 标题

**你的 7 天关西之旅**

这是我们先帮你排好的第一版主线。
按这些条件为你安排：4月3日–9日 ｜ 2位成人 ｜ 约会感 ｜ 平衡节奏

### 区域2：关键决策解释（2-3条）

💡 大阪在前京都在后 — 先感受城市能量，后半段慢下来
💡 奈良放最后一天 — 回程顺路，节奏最轻松
💡 这版没有USJ — 按你的节奏先保主线完整，想加可以一键加入

### 区域3：每日主安排（画面感文案）

> 这是第一版主线，确认后手账本会继续补齐每天的餐厅推荐、店铺、咖啡厅、拍照点和实用小技巧。
> 下一步你可以再选择吃住的舒适度和预算，不会改变这个行程安排。

### 区域4：你可能还想加的体验

> 系统只展示当前方案里没有的、对这个用户最相关的**3个**。点"查看更多"展开其余候选。

展示逻辑（内部，只用风格+人数+天数，不用预算）：
- 方案没有USJ且不是悠闲节奏 → 展示USJ
- 方案没有温泉且天数够 → 展示温泉
- 方案没有神户且天数够 → 展示神户
- 方案没有奈良且天数够 → 展示奈良
- 方案没有岚山且有京都 → 展示岚山
- 以此类推
- **预算不参与此处逻辑**，预算只在后一步影响吃住档位

卡片文案：
```
🎢 一天交给主题乐园 — 环球影城USJ
♨️ 加一晚温泉旅馆 — 泡完汤整个人都不一样
🥩 留一顿神户牛 — 港城夜景配铁板烧
🏯 去看日本最美的城堡 — 姬路城
🍵 抹茶小城散半天 — 宇治
⛰️ 住进山上寺庙 — 高野山
♨️ 浴衣逛温泉街 — 城崎温泉
🏖 太平洋边泡温泉 — 白浜
```

每个卡片一键加入。点击后系统自动重排方案。

- 想多留一点大阪

**其他想法：**

## 第三部分：预算确认页

> 方案确定后，选吃住档位，系统根据天数和方案内容算出总费用。
> 页首说明：**你的路线已经确定，这里选择的是这趟旅行想住得多舒服、吃得多讲究。不会改变前面的行程安排。**

### 吃饭

**档位（每人每顿均价）**

○ 街头地道　　约 ￥40-60　　拉面·定食·乌冬·咖喱
○ 本地好店　　约 ￥80-150　 居酒屋·名物老店·特色料理
○ 好好享受　　约 ￥200-350　割烹·寿司吧台·铁板烧
○ 顶级体验　　约 ￥500+　　 怀石·omakase·米其林

> 每个档位都会安排升级体验和平价美食，不是每顿都一样

**吃饭更看重什么？**

○ 味道好最重要 — 哪怕环境一般、要排队也值得
○ 体验舒服最重要 — 不排队、环境好、吃得从容

> 不选 = 系统按A/B混搭，安全牌偏舒服、惊喜牌偏味道

### 住宿

**档位（每间每晚均价）**

○ 干净方便　　约 ￥250-450　 商务酒店·交通便利
○ 品质舒适　　约 ￥500-900　 品质酒店·可能有大浴场
○ 享受型　　　约 ￥1,000-1,800　温泉·景观·设计感
○ 顶级　　　　约 ￥2,000+　　 顶奢酒店·高端旅馆

> 中档以上会安排特色住宿（温泉或景观），已含在总价中

**住宿更看重什么？（可多选）**

□ 住得稳 — 干净、服务好、品牌靠谱不踩雷
□ 住出体验 — 温泉、景观、设计感，住的本身就是旅行的一部分
□ 住得方便 — 车站近、位置好、搬行李不折腾

> 不选 = 帮我平衡

### 舒适加购（可选）

□ 行李寄送　　　　约 ￥100-150/次
　换城市不用拖行李，酒店到酒店次日送达

□ 偶尔打车　　　　约 ￥300-500/全程
　远离地铁或晚归时打车

### 费用预估示例

> 以"7天·本地好店·品质舒适"为例（每人）：

```
💰 7天预估（每人，不含机票购物）

餐饮 14顿　　　　￥1,400
住宿 6晚　　　　 ￥2,100（2人分摊后）
交通　　　　　　 ￥500
门票·活动　　　　￥350

合计约 ￥4,350 / 人
```

> 选择不同档位，数字实时变化

### 预算计算逻辑（内部，不展示给用户）

#### 餐饮升降档混搭比例

系统不是14顿都按均价算，而是按brief.md的餐饮策略混搭：

| 用户选的档位 | 5-6天（10-12顿） | 7-8天（14-16顿） | 9-10天（18-20顿） | 11-13天（22-26顿） |
|------------|-----------------|-----------------|------------------|-------------------|
| 街头地道 | 8顿本档+2顿升中档 | 10顿本档+2顿升中档+2顿平价 | 13顿本档+3顿升中档+2顿平价 | 16顿本档+4顿升中档+2顿平价 |
| 本地好店 | 7顿本档+1顿升享受+2顿平价 | 10顿本档+2顿升享受+2顿平价 | 13顿本档+2顿升享受+3顿平价 | 17顿本档+3顿升享受+3顿平价 |
| 好好享受 | 7顿本档+1顿升顶级+2顿中档 | 10顿本档+1顿升顶级+3顿中档 | 13顿本档+2顿升顶级+3顿中档 | 17顿本档+2顿升顶级+4顿中档 |
| 顶级体验 | 7顿本档+1顿showcase+2顿享受 | 9顿本档+1顿showcase+2顿享受+2顿中档 | 12顿本档+2顿showcase+3顿享受+3顿中档 | 15顿本档+2顿showcase+4顿享受+4顿中档 |

> showcase = 全程最高光餐（每城最多1顿，全程最多2顿）

#### 住宿升级比例

| 用户选的档位 | 5-6晚 | 7-8晚 | 9-10晚 | 11-13晚 |
|------------|-------|-------|--------|---------|
| 干净方便 | 全部本档 | 全部本档 | 全部本档 | 全部本档 |
| 品质舒适 | 全部本档 | 1晚升享受型 | 1晚升享受型 | 1-2晚升享受型 |
| 享受型 | 1晚升顶级或特色旅馆 | 1晚升顶级或特色旅馆 | 1-2晚升特色旅馆 | 2晚升特色旅馆 |
| 顶级 | 1晚最顶级体验 | 1晚最顶级体验 | 1-2晚最顶级体验 | 2晚最顶级体验 |

> 干净方便档不做升级（用户选便宜就尊重选择）
> 品质舒适以上自动安排特色住宿，升级费用已算入总价

## 第四部分：客服跟进项

> 以下不在表单/预算页展示，而是用户确认方案后客服主动发送。
> 根据用户方案中包含的内容，自动触发对应的跟进消息。

### 方案中有USJ → 客服发送

USJ门票方案对比（普通票/快速通关/VIP），含当日人流预测和推荐组合

### 方案中有和服体验 → 客服发送

京都和服店推荐（3家不同风格价位），含预约链接

### 方案中有高端餐厅 → 客服发送

需要提前预约的餐厅清单，客服可代订或提供预约指导

### 方案中有嵯峨野小火车 → 客服发送

抢票时间和技巧，旺季客服可代抢

### 方案中有宿坊/温泉旅馆 → 客服发送

推荐的房型对比和注意事项（宿坊规矩、温泉礼仪等）

### 方案中有交通卡选择 → 客服发送

关西周游卡 vs ICOCA vs JR Pass 的个性化计算和建议

## 系统逻辑（内部）

### 默认方案生成规则

| 风格 × 条件 | 默认方案特征 |
|------------|------------|
| 经典版 | 最精华的安排，均衡覆盖核心景点 |
| 约会感 | 含海游馆夜景+梅田蓝天大厦，晚间偏安静浪漫 |
| 出片感 | 含岚山+美国村，咖啡和拍照时间多 |
| 亲子感 | 含USJ+海游馆，不含深夜活动，节奏偏松 |
| 有老人 | 自动降低节奏，减少步行量大的景点 |
| 有儿童 | 自动加入亲子友好景点 |
| 7天+ | 自动加奈良 |
| 9天+ | 自动加神户或温泉 |
| 春季3-4月 | 樱花线路自动升级 |
| 夏季7月 | 天神祭自动加入 |

### 不做的事

- ✕ 前置问体验偏好（跟确认页重复）
- ✕ 前置问预算（放在方案确认后，预算只影响吃住档位不影响路线）
- ✕ 可加体验展示超过3个（多的放"查看更多"）
- ✕ 天数加减器
- ✕ 每天可取消按钮
- ✕ 多方案让用户选
- ✕ 城市选择页
- ✕ USJ/和服等复杂项放在表单里（客服跟进）
- ✕ 用关系标签代替风格标签（不用"闺蜜"，用"出片感"）
- ✕ 经典版放最后或描述成"什么都有一点"（放第一，是自信的默认选择）
