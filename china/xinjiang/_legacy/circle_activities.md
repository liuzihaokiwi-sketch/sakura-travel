# 北疆伊犁赛里木湖圈主要活动列表 v1

## 0. 文档用途

这份文档面向**上游行程规划程序 / AI 规划器**，用于先定义「北疆伊犁赛里木湖圈」内哪些内容应该被视为**主要活动（major activity）**，哪些只应当作为**次要活动（secondary activity）**或**补位活动（filler activity）**。

目标不是把赛里木湖、伊宁、喀拉峻、独库这些名字平铺出来，而是建立一个更稳的上游决策池：
- 先选线路圈
- 再从圈内选主要活动
- 再据此定住法、车程与节奏
- 最后才填停靠点、拍照位、餐饮和备胎方案

---

## 1. 线路圈定义

### 1.1 本文中的「北疆伊犁赛里木湖圈」定义

这里的「北疆伊犁赛里木湖圈」不是单城市，而是一个**以伊宁 / 博乐为核心承接点，围绕赛里木湖、伊犁河谷、草原景观与条件性公路线（独库 / 伊昭）组织的区域线路圈**，默认包含：
- 伊宁
- 赛里木湖
- 果子沟（路过型）
- 喀拉峻 / 特克斯（择优）
- 那拉提 / 独库北段（条件）
- 伊昭公路（强季节条件）
- 琼库什台（条件）

适合 **6–9 天** 建模。  
4–5 天优先只做“伊宁 + 赛湖 + 1 个草原核心”；7 天左右才适合稳定加入独库或喀拉峻 / 那拉提双节点；9 天以上再逐步开放更完整的深草原和公路线组合。

### 1.2 为什么它可以被视为一个线路圈

程序层面，这不是城市圈，而是**强自驾 / 包车导向的高景观线路圈**。  
对 20–40 岁客群而言，它的价值不是“景点密度”，而是**风景窗口、车程平衡、天气判断、住宿位置与情绪强度**。

## 2. 主要活动定义标准

一个内容要进入 `major_activity_pool`，默认至少满足以下条件中的 3 条：
1. 值得占用半天 / 一整天 / 一晚住宿
2. 会明显影响住法 / 车程 / 节奏
3. 有明确天气窗 / 花期窗 / 通车窗
4. 用户会把它作为“来新疆的理由之一”
5. 即使删掉其他点，这个活动本身仍然成立
6. 围绕它能自然挂接 1–3 个次要活动

## 7. 程序落地建议

### 7.1 建议的数据结构

```json
{
  "city_circle_id": "xj_yili_sayram_circle_v1",
  "major_activity_pool": [
    {
      "activity_id": "xj_sayram_ring_lake",
      "title": "赛里木湖环湖核心线",
      "level": "S",
      "default_duration": "full_day",
      "seasonality": ["summer", "autumn", "flower_season"],
      "profile_fit": ["first_timer", "couple", "friends", "photo", "self_drive"],
      "trip_role": "anchor",
      "can_drive_hotel": true,
      "time_window_strength": "high",
      "reservation_pressure": "low",
      "secondary_attach_capacity": 2,
      "default_selected": true
    }
  ]
}
```

### 7.2 选择逻辑建议
程序不要问“这个地方有多火”，而要问：
- 它会不会改变车程 / 住法 / 通行窗口？
- 它有没有强天气窗 / 花期窗 / 通车窗？
- 它删掉后，这趟新疆线是否失去一个核心层次？
- 它是否能自然挂接 1–3 个次要活动？
