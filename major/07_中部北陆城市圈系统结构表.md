
# 07｜中部北陆城市圈系统结构表

面向：程序 AI / 城市圈建模 / 前端配置层  
目标：把“中部北陆城市圈”固化成可排程、可编辑、可扩展的数据结构。  
默认版本：`chubu_hokuriku_circle_v1`

---

## 1. 圈定义

```ts
interface CityCircle {
  circle_id: string;
  name: string;
  region: string;
  core_bases: BaseNode[];
  daytrip_nodes: DaytripNode[];
  major_activity_clusters: MajorActivityCluster[];
  secondary_activity_clusters: SecondaryActivityCluster[];
  default_hotel_strategies: HotelStrategyPreset[];
  recommended_day_range: [number, number];
  expansion_rules: ExpansionRule[];
}
```

### 1.1 建议圈边界

| 字段 | 值 |
|---|---|
| `circle_id` | `chubu_hokuriku_circle_v1` |
| `name` | 中部北陆城市圈 |
| `region` | Chubu / Hokuriku |
| `core_bases` | 名古屋 / 高山 / 金泽 |
| `daytrip_nodes` | 白川乡 / 飞驒古川 / 郡上八幡(条件型) |
| `optional_expansion_nodes` | 上高地 / 松本 / 富山 |
| `recommended_day_range` | `5~9` 天 |
| `hard_cap_without_expansion` | `8` 天 |

### 1.2 默认策略

| 天数 | 默认住法 | 默认核心 |
|---|---|---|
| 4–5 天 | 双基点（名古屋 + 高山 or 金泽） | 不默认把三城都塞满 |
| 6–7 天 | 双基点或三基点 | 名古屋 + 高山 + 金泽 + 白川乡 |
| 8–9 天 | 三基点条件开放 | 可加入上高地/松本/富山中的 1 个 |

## 6. 风险标签系统（给活动簇）

### 6.1 字段

```ts
interface ActivityRiskTags {
  precheck_risk: RiskLevel;
  live_disruption_risk: RiskLevel;
  execution_friction: RiskLevel;
  user_condition_risk: RiskLevel;
  known_schedule_dependency: boolean;
  day_of_disruption_possible: boolean;
  backup_same_corridor_required: boolean;
}
```

### 6.2 分类原则

- 巴士库存、特急/新干线库存、旅馆售罄，属于前置可规避风险。  
- 山区天气、降雪、道路管制、能见度变化，属于实时扰动风险。  
- `shirakawago_worldheritage / mountain_nature_escape` 默认应带 fallback。

## 8. 中部北陆圈默认生成模板

### 4–5 天

- 必选主活动：`高山古街 or 金泽文化`
- 名古屋主要承担入口/出口与缓冲
- 不默认把 `名古屋 + 高山 + 金泽 + 白川乡` 全部压满

### 6–7 天

- 默认 4–5 个主活动簇
- `高山 / 金泽 / 白川乡` 应形成一条连续文化线
- 名古屋更多承担入口与收尾，而非最大内容占比

### 8–9 天

- 可开放 `山区自然线`
- 可加入一晚目的地旅馆
- 不建议再叠加过多远端扩展

## 10. 来源锚点（供程序维护）

- JNTO 已把 Nagoya–Takayama–Kanazawa 与 Shirakawa-go 作为成熟线路组合来呈现。
- 这条线更适合“文化山城连续线”建模，而不是把每个城市完全独立处理。
