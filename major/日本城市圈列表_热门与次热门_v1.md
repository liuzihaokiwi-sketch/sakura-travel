# 日本城市圈列表（热门 / 次热门）v1

> 说明：这不是日本官方行政区划，而是为了**行程规划与程序建模**而定义的“城市圈 / 旅行圈”。
>
> 这里的“热门 / 次热门”也不是官方排名，而是我基于以下因素给出的**建模优先级**：
> 1. 首次访日用户常见需求强度
> 2. 官方经典线路与官方交通覆盖是否成熟
> 3. 是否适合做成“1 个基点 + 多个顺路节点”的稳定骨架
> 4. 是否容易在 4–8 天内形成高完成度行程

---

## 一、热门城市圈（建议第一批优先建模）

### 1. 东京城市圈
- `circle_id`: `tokyo_metropolitan_circle`
- **核心基点**：东京
- **常见扩展节点**：横滨、镰仓 / 江之岛、箱根、河口湖、日光、轻井泽
- **建议天数**：4–8 天
- **适合用户**：首次赴日、城市经典、购物、美食、夜景、动漫、轻日归、樱花 / 红叶用户
- **建模理由**：东京本身内容密度极高，且官方与铁路系统都把周边大量目的地视为可从东京辐射的经典短途 / 日归线路，适合做“东京驻点 + 1–3 个扩展日”的强骨架。
- **程序备注**：
  - 3–5 天默认只做东京核心 + 0–1 个近郊日归
  - 6–8 天可加入 1–2 个强日归节点
  - 河口湖 / 箱根 / 日光不要同一趟全部塞满，优先按季节与用户偏好选 1–2 个

### 2. 关西经典城市圈
- `circle_id`: `kansai_classic_circle`
- **核心基点**：京都 / 大阪（双基点或单基点）
- **常见扩展节点**：奈良、神户、有马温泉、宇治、滋贺（大津）
- **建议天数**：5–8 天
- **适合用户**：寺社古都、美食、城市夜生活、首次关西、情侣、亲子
- **建模理由**：京都、奈良、大阪本就是官方经典线路；JR-West 的 Kansai Area Pass 也把大阪、京都、神户、奈良等作为同一可高频联动区域，极适合做“京都 / 大阪双驻点 + 奈良 / 神户日归”的稳定主干。
- **程序备注**：
  - 4–6 天以京都+大阪+奈良为主
  - 7–8 天再加神户 / 有马温泉 / 宇治
  - 低于 5 天不要自动强塞京都大阪双深度 + 神户 + 奈良全量

### 3. 北九州城市圈
- `circle_id`: `north_kyushu_circle`
- **核心基点**：福冈（博多）
- **常见扩展节点**：由布院、别府、长崎、熊本、太宰府、佐贺
- **建议天数**：5–8 天
- **适合用户**：温泉、美食、轻松多城、铁路旅行、第二次赴日
- **建模理由**：JR Kyushu 的 Northern Kyushu Rail Pass 覆盖福冈、长崎、熊本、大分等核心区域；官方经典路线也常把福冈、长崎、熊本、大分放在一个北九州旅行池里，适合做“福冈入口 + 温泉 / 历史 / 火山城市扩展”的多样化产品。
- **程序备注**：
  - 5 天以内优先“福冈 + 由布院 / 别府”或“福冈 + 长崎”二选一
  - 6–8 天可做福冈 + 长崎 + 熊本 / 由布院-别府
  - 不建议短天数同时硬塞长崎、熊本、由布院、别府全量

### 4. 冲绳本岛城市圈
- `circle_id`: `okinawa_main_island_circle`
- **核心基点**：那霸 / 恩纳（可双基点）
- **常见扩展节点**：国际通、首里城公园、万座毛、美丽海水族馆、古宇利岛、南部海景点、庆良间诸岛近海活动
- **建议天数**：4–7 天
- **适合用户**：海岛度假、亲子、情侣、自驾、浮潜 / 海景
- **建模理由**：官方经典冲绳行程本身就主要围绕冲绳本岛展开，从那霸文化入口到北部海景和海洋馆，结构非常清楚，适合独立建成“本岛圈”。
- **程序备注**：
  - 不把离岛（石垣 / 宫古）默认并入本岛圈
  - 无自驾用户应降低北部密集排布
  - 海边酒店本身可升级为主要活动

### 5. 北海道核心城市圈（札幌基点）
- `circle_id`: `hokkaido_sapporo_core_circle`
- **核心基点**：札幌
- **常见扩展节点**：小樽、登别、富良野、美瑛、旭川、二世古（按季节）
- **建议天数**：5–8 天
- **适合用户**：雪景、避暑、自然、美食、温泉、亲子、冬季滑雪用户
- **建模理由**：JR 北海道官方就有 Sapporo–Noboribetsu Area Pass 与 Sapporo–Furano Area Pass，两条线都以札幌为核心；JNTO 也常把札幌、小樽、登别、富良野 / 美瑛作为高频组合。说明“札幌为中心，按季节派生子圈”是非常适合程序建模的。
- **程序备注**：
  - 冬季优先：札幌 + 小樽 + 登别 / 二世古
  - 夏季优先：札幌 + 小樽 + 富良野 / 美瑛 / 旭川
  - 不建议一次行程把冬季雪场逻辑和夏季花田逻辑混写

### 6. 中部北陆城市圈（名古屋—高山—白川乡—金泽）
- `circle_id`: `chubu_hokuriku_classic_circle`
- **核心基点**：名古屋 / 金泽（单基点或线性移动）
- **常见扩展节点**：高山、白川乡、飞驒古川、富山
- **建议天数**：5–7 天
- **适合用户**：古镇、山景、温泉、少人一点的经典日本、摄影
- **建模理由**：JNTO 直接把 Nagoya–Takayama–Kanazawa 做成经典线路，并把名古屋描述为前往高山、白川乡等地的重要入口；这条线非常适合“线性移动圈”而不是“单城深挖”。
- **程序备注**：
  - 这是少数适合默认线性推进的热门圈
  - 4–5 天以高山 + 白川乡 + 金泽为主
  - 6–7 天再补名古屋或飞驒古川

## 二、次热门城市圈（建议第二批建模）

### 7. 广岛濑户内城市圈
- `circle_id`: `hiroshima_setouchi_circle`
- **核心基点**：广岛 / 冈山（可双基点）
- **常见扩展节点**：宫岛、尾道、仓敷、岛波海道、福山 / 鞆之浦
- **建议天数**：4–7 天
- **适合用户**：海景、慢节奏、骑行、历史、人文、美食
- **建模理由**：广岛与宫岛是官方强锚点，尾道与岛波海道又天然形成体验差异；再加上仓敷，能组成非常完整的濑户内海线路，但整体热度仍低于东京 / 关西 / 冲绳一级。
- **程序备注**：
  - 可拆成“广岛-宫岛轻圈”和“广岛-尾道-仓敷扩展圈”两层
  - 岛波海道不应默认塞给所有人，应受体力 / 骑行兴趣触发

### 8. 仙台东北入门城市圈
- `circle_id`: `sendai_tohoku_starter_circle`
- **核心基点**：仙台
- **常见扩展节点**：松岛、秋保温泉、山寺、藏王、福岛北部
- **建议天数**：4–6 天
- **适合用户**：避开东京大阪人流、温泉、秋色、雪景、轻文化线
- **建模理由**：JNTO 有非常明确的 Sendai + Matsushima + Akiu Onsen 线路；JR East 也把东北区域作为一整块可高效到达的目的地。对第一次想尝试“非黄金路线”的用户，仙台圈是最容易落地的东北入口。
- **程序备注**：
  - 第一版不用把整个东北六县都塞进一个圈
  - 先把“仙台基点 + 松岛 / 秋保 / 山寺”做稳
  - 重雪季再开放藏王树冰逻辑

### 9. 南九州火山温泉城市圈
- `circle_id`: `south_kyushu_volcano_circle`
- **核心基点**：鹿儿岛
- **常见扩展节点**：樱岛、指宿、雾岛
- **建议天数**：4–6 天
- **适合用户**：火山地貌、温泉、慢旅行、第二次赴日
- **建模理由**：鹿儿岛本身就有樱岛强锚点，指宿的砂蒸温泉又是高度差异化体验；相比北九州，它更“目的地型”，热度稍低，但辨识度很强。
- **程序备注**：
  - 适合做“自然 / 火山 / 温泉”标签强触发
  - 和北九州不要默认自动混圈

### 10. 长野高原城市圈
- `circle_id`: `nagano_highland_circle`
- **核心基点**：轻井泽 / 松本（按玩法分基点）
- **常见扩展节点**：上高地、白丝瀑布、旧轻井泽、安昙野、浅间山周边
- **建议天数**：4–6 天
- **适合用户**：避暑、高原度假、自然摄影、轻徒步、情侣
- **建模理由**：轻井泽从东京搭新干线很快可达，松本又是上高地 / 日本阿尔卑斯的入口。这类高原自然圈很适合做“东京替代扩展圈”或单独的避暑圈，但大众热度仍低于东京 / 关西主圈。
- **程序备注**：
  - 轻井泽偏休闲，松本 / 上高地偏自然徒步，不要粗暴并成同一天
  - 上高地需受季节和体力条件触发

### 11. 南北海道城市圈
- `circle_id`: `south_hokkaido_hakodate_circle`
- **核心基点**：函馆
- **常见扩展节点**：大沼公园、松前、江差
- **建议天数**：3–5 天
- **适合用户**：夜景、海港、历史街区、海鲜、北海道首尾衔接
- **建模理由**：JNTO 明确把函馆视为探索南北海道的良好起点，适合作为北海道入境 / 离境的轻圈玩法。它更适合与东北或北海道主线衔接，而不是当作全国第一优先大圈。
- **程序备注**：
  - 适合作为“北海道补充圈 / 接驳圈”
  - 不建议第一次北海道用户只玩函馆而不看札幌系内容，除非用户偏好非常明确

### 12. 纪伊半岛朝圣疗愈圈
- `circle_id`: `kii_peninsula_spiritual_circle`
- **核心基点**：高野山 / 纪伊田边（按切入点而定）
- **常见扩展节点**：熊野古道、熊野本宫、那智、白滨
- **建议天数**：4–7 天
- **适合用户**：徒步、朝圣、温泉、深度人文、非大众路线
- **建模理由**：高野山—熊野古道—熊野三山的世界遗产与精神属性极强，识别度很高，但因为交通和体力门槛更高，不适合作为第一批大众圈，而适合做“高意图用户触发型城市圈”。
- **程序备注**：
  - 必须受徒步 / 朝圣 / 深度人文偏好触发
  - 必须接入动态注意事项：山路、天气、行李转运、步行时长

## 三、程序上建议怎么使用这份城市圈表

### 1. 第一批默认只开放这 6 个热门圈
- 东京城市圈
- 关西经典城市圈
- 北九州城市圈
- 冲绳本岛城市圈
- 北海道核心城市圈
- 中部北陆城市圈

### 2. 次热门圈只在以下条件下触发
- 用户明确表达“去过东京 / 关西了，想换点别的”
- 用户明确偏好自然 / 骑行 / 徒步 / 温泉 / 非大众路线
- 用户旅行时长 ≥ 5 天，且接受更高一点的交通复杂度

### 3. 7 天以内的默认规则
- 默认 1 个城市圈
- 不自动混 2 个完整城市圈
- 若做扩展，只允许“主圈 + 1 个轻扩展子圈 / 强日归节点”

### 4. 后续可以继续拆的子圈
- 东京城市圈 → 东京核心圈 / 东京近郊海岸圈 / 东京富士温泉圈
- 关西经典圈 → 京都文化圈 / 大阪娱乐圈 / 神户温泉夜景圈
- 北海道核心圈 → 冬季雪景圈 / 夏季花田圈
- 北九州圈 → 福冈都市圈 / 由布院别府温泉圈 / 长崎历史圈
