# 京都城市圈主要活动列表 v1

## 0. 文档用途

这份文档不是给终端游客直接看，而是给**上游行程规划程序 / AI 规划器**使用，用于先定义「京都城市圈」内哪些内容应该被视为**主要活动（major activity）**，哪些只应当作为**次要活动（secondary activity）**或**补位活动（filler activity）**。

目标不是列出所有热门景点，而是建立一个更稳定的上游决策池：

- 先选城市圈
- 再从城市圈内选主要活动
- 再据此定路线与酒店
- 最后才填次要活动、餐厅、购物和可砍项

---

## 1. 城市圈定义

### 1.1 本文中的「京都城市圈」定义

这里的「京都城市圈」不是狭义的京都市，而是一个**京都为文化中心、可通过关西交通网高效串联的经典关西游玩圈**，默认包含：

- 京都市核心区
- 宇治
- 奈良市区
- 大阪市区
- 神户市区 / 有马温泉

这个定义适合 5–9 天的经典关西旅行建模。  
如果行程只有 3–4 天，程序应优先只深挖京都 + 1 个外扩节点；  
如果行程 7 天左右，才适合稳定装入 4–5 个主要活动；  
如果 9 天以上，才逐步开放更完整的大阪 / 神户扩展。

### 1.2 为什么它可以被视为一个城市圈

程序层面，不按行政区，而按**可组合旅行体验**和**交通可达性**定义城市圈更稳。  
官方资料本身也把京都、奈良、大阪视作可连成一条经典 Kansai 线路；JR-West 的 Kansai Area Pass 也把 Kyoto / Osaka / Kobe / Nara 作为同一有效区域。

---

## 2. 主要活动定义标准

一个内容要进入 `major_activity_pool`，默认至少满足以下条件中的 3 条：

1. **值得专门占用半天 / 一整天 / 一晚住宿**
2. **会明显影响酒店驻点或跨区路线**
3. **有明确时间窗**（清晨、日落、季节窗口、夜景、预约场）
4. **用户会把它作为“来这一圈的理由之一”**
5. **即使删掉其他点，这个活动本身仍然成立**
6. **围绕它能自然挂接 1–3 个次要活动**

凡是不满足这套条件、只是“顺路去看一下更好”的内容，默认不进入 major pool。

---

## 3. 主要活动层级

为避免所有热门内容都挤进 major pool，本文把主要活动分为 3 层：

### S 级：默认强主活动
- 对大多数第一次来京都城市圈的用户都成立
- 常常直接占据半天至一天
- 很容易成为整趟旅行的骨架

### A 级：条件性主活动
- 对特定用户画像非常强
- 对另一部分用户则更适合作为次要活动
- 程序应根据表单画像触发

### B 级：体验型主活动
- 通常不是经典必打卡，但一旦命中偏好，会强烈影响酒店、预算和排程
- 更适合情侣、亲子、奢华、季节限定等特殊路线

## 4. 京都城市圈默认强主活动（S 级）

### S1. 东山祇园经典线
- `activity_id`: `kyo_higashiyama_gion_classic`
- `核心区域`: 清水寺 / 二年坂三年坂 / 高台寺 / 八坂神社 / 祇园
- `推荐时长`: 0.75–1 天
- `推荐季节`: 全年，樱花 / 红叶更强
- `适合画像`: 首次来关西、情侣、摄影、传统文化、轻奢美学
- `通常是否为主活动`: **是，且优先级极高**

#### 为什么它是 major
这是最典型的“京都感”承载区。  
它不是一个单点，而是一整条步行氛围线：寺社、坡道、町家、街景、夜色、和服体验都集中在这条线上。  
程序上，它是京都圈里最适合被当作**整日核心主轴**的活动簇之一。

#### 典型锚点
- 清水寺本体
- 产宁坂 / 二年坂 / 三年坂街景
- 八坂神社
- 祇园傍晚街区散步
- 可叠加：高台寺、圆山公园、花见小路、将军冢夜景（条件）

#### 常见挂接次要活动
- 和服体验
- 茶屋 / 抹茶甜点
- 传统町家午餐 / 晚餐
- 小型手作 / 清水烧 / 文创购买
- 傍晚鸭川或先斗町收尾

#### 常见降级方式
如果天数紧，可压缩成“清水寺 + 坂道街区 + 祇园傍晚”半天主活动。

---

### S2. 岚山嵯峨野自然遗产线
- `activity_id`: `kyo_arashiyama_sagano`
- `核心区域`: 岚山 / 竹林 / 天龙寺 / 渡月桥 / 嵯峨野
- `推荐时长`: 0.75–1 天
- `推荐季节`: 春、秋最强；夏季清晨也好
- `适合画像`: 首次来关西、情侣、摄影、自然、轻徒步
- `通常是否为主活动`: **是**

#### 为什么它是 major
岚山不是“竹林一个点”，而是完整的西京都体验面：  
自然、寺院、河景、桥景、观光铁路 / 船、咖啡与散步都能围绕它展开。  
程序上，它足够支撑独立的一天，且与东山线风格明显不同，能形成体验错落。

#### 典型锚点
- 竹林小径
- 天龙寺
- 渡月桥
- 岚山公园 / 河岸散步
- 可叠加：嵯峨野小火车、保津川下り、常寂光寺、野宫神社

#### 常见挂接次要活动
- 甜品 / 咖啡
- 竹制 / 和风纪念品
- 轻徒步
- 河边人像拍摄
- 足汤 / 小型温泉休息

#### 常见降级方式
如果天气不好或体力有限，程序应收缩成“天龙寺 + 竹林 + 渡月桥”的半天型 major。

---

### S3. 奈良古都与鹿线
- `activity_id`: `nara_park_todaiji_deer`
- `核心区域`: 奈良公园 / 东大寺 / 春日大社 / 奈良町
- `推荐时长`: 0.75–1 天
- `推荐季节`: 全年
- `适合画像`: 首次关西、亲子、情侣、轻松慢行、文化历史
- `通常是否为主活动`: **是**

#### 为什么它是 major
奈良不是“去喂一下鹿”这么简单。  
它是一个非常完整的古都日归主活动：大佛、鹿、神社、古都氛围、低压慢节奏都集中在一个可步行片区里。  
与京都相比，它更松弛、更开阔，也更适合做节奏缓冲日。

#### 典型锚点
- 奈良公园
- 东大寺大佛殿
- 春日大社
- 奈良町散步

#### 常见挂接次要活动
- 和果子 / 柿叶寿司
- 奈良町老屋咖啡馆
- 若草山（条件）
- 奈良国立博物馆（雨天条件）

#### 常见降级方式
如果只是顺路浅尝，程序可压缩成“奈良公园 + 东大寺”的半天 major；  
但对第一次去奈良的用户，一般不建议再压得更碎。

---

### S4. 宇治抹茶文化线
- `activity_id`: `uji_matcha_byodoin`
- `核心区域`: 平等院 / 宇治川 / 表参道 / 茶店体验
- `推荐时长`: 0.5–0.75 天
- `推荐季节`: 全年
- `适合画像`: 抹茶偏好、文化、慢节奏、情侣、轻文艺
- `通常是否为主活动`: **是，但更偏半天型主活动**

#### 为什么它是 major
宇治虽然量级不如东山或奈良，但它是**体验非常完整的单主题区域**：  
平等院、宇治桥、抹茶、老街和河景共同构成“能专门来一次”的内容簇。  
它很适合和京都南线、伏见线或回程轻松日做组合。

#### 典型锚点
- 平等院凤凰堂
- 宇治桥与宇治川
- 抹茶甜品 / 茶屋 / 茶体验
- 表参道与纪念品街

#### 常见挂接次要活动
- 源氏物语相关点
- 茶具购买
- 轻散步拍照
- 小型和菓子店

#### 常见降级方式
如果行程极紧，宇治可以降为 secondary；  
但只要用户表单里有“抹茶”“慢节奏”“想拍安静古风街区”，它就应升回 major。

---

### S5. 伏见稻荷时间窗线
- `activity_id`: `kyo_fushimi_inari_timewindow`
- `核心区域`: 伏见稻荷大社 / 稻荷山参道 / 千本鸟居
- `推荐时长`: 0.5 天
- `推荐季节`: 全年；清晨、傍晚、夜色更强
- `适合画像`: 首次来京都、摄影、早起型、神社控
- `通常是否为主活动`: **是，但属于时间窗驱动型主活动**

#### 为什么它是 major
伏见稻荷的关键不是“点位多”，而是**时间窗极强**。  
清晨和白天、白天和夜色，体验差异非常大。  
它会改变整天起止方式，所以程序上不能把它简单塞成普通路过点。

#### 典型锚点
- 千本鸟居
- 稻荷山前段步道
- 清晨空场拍摄
- 日暮蓝调 / 夜色参道（条件）

#### 常见挂接次要活动
- 东福寺（秋季尤其强）
- 宇治
- 京都站周边补给
- 伏见酒藏（画像命中时）

#### 常见降级方式
对不早起、怕爬坡、只求打卡的用户，它可以被降为 secondary；  
但在摄影 / 首次京都用户里，默认保留 major 身份。

## 5. 京都城市圈条件性主活动（A 级）

### A1. 大阪 USJ 整日主题乐园线
- `activity_id`: `osa_usj_themepark`
- `核心区域`: Universal Studios Japan
- `推荐时长`: 1 天
- `适合画像`: 亲子、IP 乐园、情侣、特种兵、第一次关西但强娱乐需求
- `通常是否为主活动`: **命中画像时一定是**

#### 为什么它是 major
它会直接吃掉一整天，并改变整个城市圈的重心、体力、票务策略和住宿安排。  
一旦选入，几乎必然是最强主活动之一。

#### 程序规则
- 若表单命中「亲子 / 主题乐园 / 任天堂 / 哈利波特 / 强娱乐」，优先升为 S
- 若用户明确偏文化 / 慢旅行 / 预算克制，则默认不入选

---

### A2. 大阪 Minami 美食夜生活线
- `activity_id`: `osa_minami_food_night`
- `核心区域`: 道顿堀 / 心斋桥 / 难波 / 法善寺横丁
- `推荐时长`: 0.5–0.75 天
- `适合画像`: 美食、夜生活、朋友同行、第一次关西想要城市反差
- `通常是否为主活动`: **条件成立时是**

#### 为什么它不是默认对所有人都强
它很热闹、很有大阪辨识度，但对一部分用户来说，它更像“夜间强化体验”，而不是整趟旅行的文化主轴。  
所以程序不应默认把它硬塞进每一条京都圈路线里。

#### 何时升级为 major
- 用户明确想要大阪街头感 / 夜生活 / 居酒屋 / 小吃线
- 旅行同行人偏年轻或朋友局
- 京都寺社比例过高，需要强反差体验

---

### A3. 金阁寺—龙安寺—仁和寺西北寺社线
- `activity_id`: `kyo_northwest_temple_line`
- `核心区域`: 金阁寺 / 龙安寺 / 仁和寺
- `推荐时长`: 0.5–0.75 天
- `适合画像`: 世界遗产偏好、寺社控、二刷京都
- `通常是否为主活动`: **条件成立时是**

#### 为什么是条件性 major
这条线历史价值很强，也足够独立成日程块；  
但对很多第一次京都且更偏“氛围 + 街景 + 拍照”的用户来说，它的优先级往往会落在东山 / 岚山之后。  
因此更适合作为条件升级，而不是一刀切默认入选。

---

### A4. 贵船—鞍马山线
- `activity_id`: `kyo_kibune_kurama_mountain`
- `核心区域`: 贵船神社 / 贵船川床（季节）/ 鞍马寺 / 山路
- `推荐时长`: 0.75 天
- `推荐季节`: 夏季纳凉、秋季山色最强
- `适合画像`: 想避暑、轻徒步、山中神社、二刷京都、想逃离人群
- `通常是否为主活动`: **季节或画像命中时是**

#### 为什么它不能默认给所有人
它更偏山线和交通换乘，不是人人都爱；  
但一旦用户喜欢自然、避暑、轻徒步，它就会从普通备选直接升级成非常强的主活动。

---

### A5. 神户港夜景 / 北野异人馆 / 都市洋风线
- `activity_id`: `kobe_harbor_cityscape`
- `核心区域`: 北野 / Harborland / Meriken Park / 夜景
- `推荐时长`: 0.5–0.75 天
- `适合画像`: 情侣、夜景控、都市海港感、轻奢
- `通常是否为主活动`: **条件成立时是**

#### 为什么是条件性 major
神户很好看，但对一般第一次关西 5–7 天用户来说，未必比奈良 / 宇治 / 伏见更优先。  
它更像“风格替换件”：当用户厌烦寺社、喜欢海港和洋风时，神户非常有价值。

---

### A6. 大阪湾家庭娱乐线
- `activity_id`: `osa_bay_family_fun`
- `核心区域`: 海游馆 / 天保山 / 湾区娱乐
- `推荐时长`: 0.5–0.75 天
- `适合画像`: 亲子、雨天、轻松派、动物海洋馆偏好
- `通常是否为主活动`: **对亲子/雨天路线可升级**

#### 为什么它默认不是所有人的 main pick
它很适合家庭和天气不稳定时的替代，但对情侣或传统文化优先的用户，常常不如其他活动重要。

## 6. 京都城市圈体验型主活动（B 级）

### B1. 有马温泉住一晚 / 日归温泉线
- `activity_id`: `arima_onsen_ryokan`
- `核心区域`: 有马温泉温泉街 / 金之汤 / 银之汤 / 旅馆
- `推荐时长`: 日归 0.5 天；住宿 1 天 1 夜
- `适合画像`: 情侣、奢华、慢旅行、疗愈、纪念日
- `通常是否为主活动`: **一旦选住宿，必定是强主活动**

#### 为什么它极重要
这类活动不是靠景点密度，而是靠**住一晚本身改变整趟旅程节奏**。  
一旦选它，程序必须重新思考酒店切换、预算分配和前后两天体力安排。

#### 程序规则
- 选“温泉酒店 / 纪念日 / 想慢一点 / 预算较高”时优先升级
- 若只是补一个泡汤点，可降为 secondary

---

### B2. 京都高价旅宿 / 町家 / 怀石主导线
- `activity_id`: `kyo_luxury_stay_kaiseki`
- `核心区域`: 京都町家旅宿 / 高级旅馆 / 怀石晚餐 / 私密体验
- `推荐时长`: 1 晚 + 半天
- `适合画像`: 纪念日、蜜月、预算高、重体验而非多打卡
- `通常是否为主活动`: **命中画像时是**

#### 为什么它应被视为 major
很多高客单用户并不是为了“多去几个点”，而是为了**住得好、吃得好、氛围好**。  
这类用户的 major 活动，不应只来自景点，还应来自酒店和餐饮体验本身。

---

### B3. 嵯峨野观光小火车 / 保津川下り线
- `activity_id`: `kyo_sagano_train_hozu_river`
- `核心区域`: 嵯峨野观光铁路 / 保津川下り / 岚山联动
- `推荐时长`: 0.5 天
- `适合画像`: 季节景观、喜欢交通体验、第一次岚山但想更完整
- `通常是否为主活动`: **季节命中时可升级**

#### 为什么是体验型 major
它本身足够成立，但更像“岚山增强包”，通常不单独压过东山 / 奈良 / 宇治。  
更适合程序在命中季节、空位、交通体验偏好时升级。

---

## 7. 默认不应作为 major 的内容

以下内容默认更适合做 `secondary_activity` 或 `filler_activity`，除非画像特别命中：

- 锦市场
- 先斗町 / 鸭川夜散步
- 京都塔 / 京都站周边
- 大阪城单点参观
- 心斋桥购物本身
- 神户中华街单点
- 花见小路短暂停留
- 单个咖啡馆 / 单个甜品店
- 单个商场 / 单个百货
- 单个展望台

### 原因
这些内容通常满足以下特征之一：
- 本身不能单独支撑半天以上
- 删除后不影响整趟旅程骨架
- 更适合作为顺路增强，而非路线决定因素

## 8. 程序默认选取建议

### 8.1 5–6 天，第一次去京都城市圈
建议从以下池子里选 3–4 个 main majors：

- 东山祇园经典线
- 岚山嵯峨野线
- 奈良古都与鹿线
- 宇治抹茶文化线 或 伏见稻荷时间窗线
- 若画像很强，可替换为 USJ 或 有马温泉

### 8.2 7 天经典平衡型
建议选 4–5 个 main majors：

- 东山祇园经典线
- 岚山嵯峨野线
- 奈良古都与鹿线
- 宇治抹茶文化线
- 伏见稻荷时间窗线 / 有马温泉 / 大阪夜生活 三选一

### 8.3 7–8 天情侣轻奢型
建议选 4–5 个 main majors：

- 东山祇园经典线
- 岚山嵯峨野线
- 宇治抹茶文化线
- 有马温泉住一晚
- 神户海港夜景 或 京都高价旅宿 / 怀石主导线

### 8.4 7–8 天亲子娱乐型
建议选 4–5 个 main majors：

### 9.2 选择逻辑建议
程序不要问“这个景点热不热门”，而要问：

- 这个活动能不能单独成立？
- 能不能带出路线差异？
- 有没有强时间窗？
- 会不会改变酒店策略？
- 能不能挂接 1–3 个次要活动？
- 删掉它，整趟旅行会不会失去一个独立体验维度？

只要答案大多是“会”，它就更接近 major。
