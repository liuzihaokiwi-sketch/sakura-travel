# 广佛顺珠广府圈主要活动列表 v1

## 0. 文档用途

这份文档面向**上游行程规划程序 / AI 规划器**，用于先定义「广佛顺珠广府圈」内哪些内容应该被视为**主要活动（major activity）**，哪些只适合作为**次要活动（secondary activity）**或**补位活动（filler activity）**。

目标不是把“广州塔、陈家祠、顺德吃饭、珠海海边”简单并列，而是先建立一个更稳定的上游决策池：
- 先选城市圈
- 再从圈内选主要活动
- 再据此定路线与酒店
- 最后才填美食、夜景、购物和可砍项

---

## 1. 城市圈定义

### 1.1 本文中的「广佛顺珠广府圈」定义

这里的「广佛顺珠广府圈」不是单一行政城市，而是一个**以广州为交通中心、串联佛山 / 顺德 / 珠海的广府文化与美食休闲圈**，默认包含：
- 广州市区核心区
- 佛山禅城 / 南海文化区
- 顺德美食与园林水乡区
- 珠海情侣路 / 横琴 / 海滨休闲区

适合 **5–7 天** 建模。  
3–4 天优先只深挖广州 + 顺德 or 佛山；6 天左右才适合稳定装入珠海海滨线；7 天以上再逐步开放更完整的顺德深玩 + 珠海一晚度假。

### 1.2 为什么它可以被视为一个城市圈

程序层面，不按行政区，而按**高铁 / 城际可达性、广府文化连续性与“吃住逛”互补体验**定义城市圈更稳。  
对 20–40 岁客群，这个圈的核心价值不是“景点密度”，而是**好吃、好住、节奏松弛、兼顾城市与海边、适合情侣 / 闺蜜 / 家庭**。

## 2. 主要活动定义标准

一个内容要进入 `major_activity_pool`，默认至少满足以下条件中的 3 条：
1. 值得专门占用半天 / 一整天 / 一晚住宿
2. 会明显影响酒店驻点或跨区路线
3. 有明确时间窗（早茶、夜景、海边日落、演出 / 开园）
4. 用户会把它作为“来这一圈的理由之一”
5. 即使删掉其他点，这个活动本身仍然成立
6. 围绕它能自然挂接 1–3 个次要活动

## 4. 默认强主活动（S 级）

### S1. 广州老城广府生活线
- `activity_id`: `gzs_old_canton_life`
- `核心区域`: 永庆坊 / 上下九 / 沙面 / 北京路（择区拼接）
- `推荐时长`: 0.75–1 天
- `适合画像`: 首次广府圈、情侣、闺蜜、citywalk、烟火气
- `通常是否为主活动`: **是，且优先级极高**

### S2. 广州塔—珠江夜景都市线
- `activity_id`: `gzs_pearl_river_night`

- `推荐时长`: 0.5–0.75 天
- `适合画像`: 情侣、年轻朋友局、首次广州、夜景偏好
- `通常是否为主活动`: **是，但属于时间窗型主活动**

### S3. 顺德世界美食线
- `activity_id`: `shunde_food_journey`
- `核心区域`: 清晖园 / 华盖路 / 金榜 / 渔村 / 伦教（择优）
- `推荐时长`: 1 天
- `适合画像`: 美食、情侣、朋友局、家庭、周末放松
- `通常是否为主活动`: **是**

### S4. 佛山岭南文化功夫线
- `activity_id`: `foshan_lingnan_kungfu`
- `核心区域`: 祖庙 / 岭南天地 / 南风古灶（择优）
- `推荐时长`: 0.75–1 天
- `适合画像`: 文化、建筑、亲子、首次广府圈
- `通常是否为主活动`: **是**

### S5. 珠海海滨松弛线

- `核心区域`: 情侣路 / 日月贝 / 海滨公园 / 香炉湾（择优）

- `适合画像`: 情侣、松弛、轻度假、亲子
- `通常是否为主活动`: **是，但更偏松弛型主活动**

## 7. 程序落地建议

### 7.1 建议的数据结构

```json
{
  "city_circle_id": "guangfu_gz_fs_sd_zh_circle_v1",
  "major_activity_pool": [
    {
      "activity_id": "gzs_old_canton_life",
      "title": "广州老城广府生活线",
      "level": "S",
      "default_duration": "full_day",
      "seasonality": ["all_year"],
      "profile_fit": ["first_timer", "couple", "citywalk", "food", "photo"],
      "trip_role": "anchor",
      "can_drive_hotel": true,
      "time_window_strength": "medium",
      "reservation_pressure": "low",
      "secondary_attach_capacity": 3,
      "default_selected": true
    }
  ]
}
```

### 7.2 选择逻辑建议
程序不要问“哪个城市名气更大”，而要问：
- 这条线能不能独立成立？
- 它会不会改变住宿与跨城策略？
- 它是否显著拉开这趟旅行的气质？
- 它能不能自然挂接吃 / 逛 / 夜景 / 海边等次要活动？
