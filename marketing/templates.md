## 小红书图文模板

### 模板 A：美食 Top N

**适用场景**: 按区域/菜系/价位推荐多家餐厅

**标题公式**:
- "{城市}必吃{N}家{菜系}｜本地人推荐"
- "{区域}{菜系}TOP{N}｜从{价格低}到{价格高}全覆盖"
- "去{N}次{城市}总结的{菜系}清单｜收藏不亏"

**正文结构**:
```
开头（1-2句）: 筛选标准说明（为什么推这几家）
---
每家餐厅（重复N次）:
  店名 + 一句话定位
  推荐菜品 1-2 道
  人均价格
  地址/最近车站
  Tips（预约方式、排队情况、营业时间注意）
---
结尾: 总结对比 + 引导收藏
```

**标签建议**: #{城市}美食 #{菜系} #{城市}旅行 #日本美食 #旅行攻略

**字数参考**: 800-1200字

### 模板 B：一日游路线

**适用场景**: 单日完整行程规划

**标题公式**:
- "{区域}一日游｜从早到晚完美路线"
- "{城市}第一次去这样走就对了"
- "本地人带你走{区域}｜避开人潮的路线"

**正文结构**:
```
开头: 路线概述（适合谁、总花费、交通方式）
---
时间线:
  09:00 第一站 — 景点名 + 为什么来 + 停留时间
  11:00 第二站 — ...
  12:30 午餐 — 餐厅推荐（从数据库选S/A级）
  14:00 第三站 — ...
  ...
  18:00 晚餐 — ...
---
结尾: 交通Tips + 省钱建议 + 引导收藏
```

**标签建议**: #{区域}一日游 #{城市}旅行 #日本攻略 #行程规划

**字数参考**: 1000-1500字

### 模板 C：对比型

**适用场景**: 同类产品/体验的横向对比

**标题公式**:
- "{A} vs {B}｜到底选哪个？"
- "实测{N}家{品类}｜帮你避坑"
- "{同一条街}{N}家{品类}逐家对比"

**正文结构**:
```
开头: 对比背景（为什么要比、怎么比的）
---
对比维度表格:
  | 维度 | A | B | C |
  |------|---|---|---|
  | 价格 |   |   |   |
  | 口味 |   |   |   |
  | 环境 |   |   |   |
  | 排队 |   |   |   |
---
每家详细点评（2-3句）
---
结尾: 总结推荐（不同需求选不同）
```

**标签建议**: #{品类}测评 #{城市}美食 #避坑指南

**字数参考**: 800-1000字

### 模板 D：冷知识/避坑

**适用场景**: 实用信息、常见误区

**标题公式**:
- "去{城市}前必须知道的{N}件事"
- "{城市}旅行最容易踩的{N}个坑"
- "99%的人不知道的{城市}冷知识"

**正文结构**:
```
开头: 引入（我踩过的坑 / 被问最多的问题）
---
逐条列举:
  1. 知识点/坑点 — 详细说明 + 正确做法
  2. ...
  （每条 2-4 句，简洁有力）
---
结尾: 总结 + 引导互动（你还踩过什么坑？）
```

**标签建议**: #{城市}攻略 #避坑 #旅行干货 #冷知识

**字数参考**: 600-1000字

## 抖音脚本模板

### 模板 E：图文轮播

**适用场景**: 将小红书图文内容改编为抖音轮播

**脚本结构**:
```
画面1 (封面): 标题文字 + 吸引画面
画面2-N: 每张图配 1-2 行文字说明
末尾画面: 总结 + 关注引导
```

**配音文案**:
```
开头钩子（3秒内）: 一句话抓注意力
  例: "京都这5家拉面，本地人排队也要吃"
正文: 跟随画面节奏，每张图 5-8 秒配音
结尾: 引导关注/评论
```

**时长参考**: 30-60秒

## 模板使用说明

1. 模板是起点，不是限制。实际写作时根据内容灵活调整
2. 数据来源：餐厅/酒店/景点信息从项目数据库提取，确保准确
3. 每篇内容发布前核实关键事实（价格、营业时间、地址）
4. 标签根据平台热度实时调整，以上仅为参考
