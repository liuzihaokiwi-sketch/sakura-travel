# 第一部分:说明书

## 这份文档是什么

做关西模板/调研时,AI 会用 opencli 搜小红书,看到一堆爆款笔记。
**这些笔记本身就是情报**——用户愿意收藏 / 点赞 / 评论的是什么,反映了真实市场需求。

但做模板时顺便看一眼就划走了,这些情报没沉淀下来。这份文档就是**把遇到的爆款笔记和爆款规律记下来**,后续做内容时能直接参考结构和选题。

## 和"素材库"的区别

| 素材库 | 爆款参考(这份) |
|---|---|
| **我们自己挖到的**小众素材 | **别人已经发的**爆款内容 |
| 是产品护城河的来源 | 是结构/视角/套路的参考 |
| 直接用来起选题 | 用来学怎么起选题 |

两份文档互补:素材库给你"写什么",爆款参考给你"怎么写"。

## 写的时机

**场景 1:opencli 搜小红书时**
- 用 `node dist/main.js xiaohongshu search "关键词" --limit 10 --format md`
- 看到 collects(收藏)明显高于 likes(点赞)的,或 likes > 500 的 → 值得记录
- 特别是 collects/likes 比值高的(说明"真有用",不只是"好看")

**场景 2:WebSearch 发现小红书/抖音/知乎高赞内容**
- 类似规则:看数据,看结构

**场景 3:刷到的他人游记/攻略超出预期**
- 本地杂志深度文章、up 主视频中的选题

## 每条爆款的标准格式

```markdown
### {爆款标题(原标题或简化版)}
**平台**: 小红书 / 抖音 / YouTube / 其他
**作者 + 日期**: {作者昵称 - 发布日期}
**链接**: {URL}
**数据**: 点赞 X / 收藏 Y / 评论 Z
**主题**: {用素材库相同的主题词表}
**爆款结构**: {点明内容的组织方式:3 段式 / 排行榜 / 对比 / 故事流 / ...}
**选题视角**: {作者从哪个切入点抓住读者}
**可借鉴**: {我们可以怎么用这个结构/视角}
**不抄的部分**: {哪里是常规/平庸的,不要学}
```

**字段解释**:
- **数据**:收藏 > 点赞 更能说明"真有用";点赞 > 收藏 更说明"好看/有趣"
- **爆款结构**:这是核心——**结构可以复用,选题要原创**
- **可借鉴 vs 不抄**:防止抄到常规化陷阱

## 京都主题

### 4刷红叶季，这12个才是我心中的京都最佳
**平台**: 小红书
**作者 + 日期**: 一葱 2025-10-28
**链接**: https://www.xiaohongshu.com/search_result/6900615d000000000400315c
**数据**: 点赞 3 / 收藏 1703 / 评论 91
**主题**: 季节限定
**爆款结构**:
1. 人设建立("4刷红叶季")→ 立刻建立信任感
2. 三层分类(王道必去 / 岚山秘境 / 本地私藏)→ 满足不同读者需求
3. 每个景点一句核心评价,不废话
4. 照片全是真实走过的现场感

**选题视角**: "我比你多去了几次"的信任感 + 三层分类让读者自己对号入座
**可借鉴**:
- 人设先行(几刷/几次实战),再给结论 — 比"TOP10推荐"可信
- 三层结构(王道/秘境/私藏)是通用框架,可迁移到任何城市任何季节
- 每景点只给一句最核心的东西,不贪多

**不抄的部分**: 对预约景点(祐斋亭)说"价格虽贵"但没给具体价位指引 — 我们要补上

### 2年去了4次红叶季，时间怎么选，数据说话！
**平台**: 小红书
**作者 + 日期**: 一葱 2025-08-23
**链接**: https://www.xiaohongshu.com/search_result/68a9a2b0000000001b0378bb
**数据**: 点赞 5 / 收藏 834 / 评论 461
**主题**: 季节限定 / 行程路线
**爆款结构**:
1. 痛点切入("红叶真的太难预测了")
2. 核心规律一句话("山区先红,市区后红")
3. 具体建议(11月末主攻山区,12月初主攻市区)
4. 反直觉洞察("宁早不晚",初期层次感比全红期更美)

**选题视角**: 解决"红叶季什么时候去最好"这个高频但没人用数据回答的问题
**可借鉴**:
- 痛点("每年猜玄学")比推荐开头更抓人
- 一个核心规律讲透,不堆景点列表
- 461条评论说明这是高互动话题 — 用户在此选题上非常焦虑,有强信息需求

**不抄的部分**: 只讲时间不讲具体景点分配 — 我们可以把"时间"和"景点分类"结合做成更完整的指南

### [示例] 2025 祇园祭终极攻略 | 千年京都夏祭
**平台**: 小红书
**作者 + 日期**: 和之旅 2025-05-04
**链接**: https://www.xiaohongshu.com/search_result/681770a10000000003038b69
**数据**: 点赞 85 / 收藏 111 / 评论 3
**主题**: 祭典/季节限定
**爆款结构**:
1. 先给爆点(必看亮点:辻回し、长刀鉾稚儿、山鉾清单)
2. 日程表(7/1-7/24 完整)
3. 实用信息(交通/注意事项)
4. 彩蛋(拍"山鉾转弯"的最佳站位)

**选题视角**: **不写"祭典介绍",直接写"终极攻略"** — 把"这个祭典该怎么看"当成卖点,读者直接收藏留用
**可借鉴**:
- "先爆点后日程"的顺序(比"从头介绍"抓人)
- 最后一条"彩蛋"形式(拍照站位这种硬通货,用户收藏率飙升)
- 用 emoji 分段(🎪 📅 🚇 ⚠️)提升可读性

**不抄的部分**:
- "日本三大祭之首"这种百科式开头(太常规)
- "后祭比前祭人少"这个洞察它讲了但**没讲透**——我们的产品要讲出"人流减半 + 町家屏風祭质量不降"的具体细节

## 大阪主题

### 大阪｜最适合散步的地方🚶🏻(中之岛·北浜)
**平台**: 小红书
**作者 + 日期**: 小南北 2024-05-23
**链接**: https://www.xiaohongshu.com/search_result/664f5e9f0000000015009927
**数据**: 点赞 2 / 收藏 6044 / 评论 92
**主题**: 大阪小众视角
**爆款结构**:
1. 定性开头("大阪最适合散步的地方")→ 直接给结论
2. 清单格式:公共建筑(编号1-5)+咖啡甜品(编号6-10)
3. 每个地方:名称+地址+一句特色描述

**选题视角**: "大阪不只是道顿堀"——给已经知道大阪的人提供另一面,满足"我想看不一样的大阪"需求
**可借鉴**:
- 标题直接给结论(最适合散步),不卖关子
- 建筑+咖啡双清单覆盖不同兴趣的读者
- 具体地址增加信任感

**不抄的部分**: 缺少动线设计(哪个先走哪个后走);缺少人群标签(适合谁)

### 在大阪花的最值的一笔钱 高空落日飞车🌅
**平台**: 小红书
**作者 + 日期**: 小小只yiyi儿 2025-08-24
**链接**: https://www.xiaohongshu.com/search_result/68aa67f7000000001d036404
**数据**: 点赞 466 / 收藏 3万 / 评论 828
**主题**: 大阪小众视角 / 出片机位
**爆款结构**:
1. 标题给悬念("花的最值的一笔钱")→ 读者想知道是哪里
2. 靠高质量日落/夜景图片驱动(点赞少但收藏3万 = 纯靠内容实用价值)

**选题视角**: 找到一个"没人告诉你的"大阪夜景点——比所有人都知道的梅田Sky Building多一分"发现感"
**可借鉴**:
- 收藏/点赞比值超高(3万/466)= 纯实用价值驱动,不是好看
- "本地家庭才去的地方"这个框架非常吸引人
- 标题"最值的一笔钱"触发"我也想试试"的冲动

**不抄的部分**: 内容太简短没有具体攻略(交通/价格/时间);我们要补充这些才能真正有用

### 大阪天神祭｜花火＆祭典 完全攻略
**平台**: 小红书
**作者 + 日期**: 卡皮巴拉in京都 2025-07-03
**链接**: https://www.xiaohongshu.com/search_result/686624ab000000002201d4e6
**数据**: 点赞 9 / 收藏 1416 / 评论 146
**主题**: 祭典/季节限定
**爆款结构**:
1. 三大看点先列(陆渡御/船渡御/花火)→ 快速建立期待
2. 具体时间线和观看位置地图
3. 隐藏彩蛋:大阪締め参与方式(击掌仪式)→ 让读者感觉"学到了别人不知道的"

# 给后续 AI 的一句话
