# marketing 目录说明

> 小红书/抖音运营内容 + 产品调研/做模板时沉淀的素材库。
> 小雅(+ 她那边的 AI)做内容时从这里取材。

## 结构

```
marketing/
├── strategy.md            整体运营策略(账号矩阵思路、内容类型库)
├── templates.md           内容模板(小红书爆款结构等)
├── accounts.md            账号矩阵(哪些号、各自定位)
├── competitor_notes.md    竞品观察
├── open_decisions.md      待定决策
├── design_refs/           设计参考(字体/配色/排版)
│
├── japan/
│   └── kansai/
│       ├── 素材库.md          ← ⭐ 核心,做模板/调研时沉淀的素材(说明书+大表)
│       ├── 爆款参考.md        ← ⭐ 核心,他人爆款分析(结构+选题视角)
│       ├── xiaohongshu/       实际发出的小红书内容
│       └── douyin/            实际发出的抖音内容
├── china/                 (预留)
└── europe/                (预留)
```

## 两类核心文件

### 素材库 vs 爆款参考

| | 素材库 | 爆款参考 |
|---|---|---|
| **来源** | 我们自己挖到的 | 他人已发的爆款 |
| **性质** | 产品护城河 | 结构/视角参考 |
| **用法** | 直接起选题 | 学怎么起选题 |

**两份互补**:素材库给你"写什么",爆款参考给你"怎么写"。

### 谁在什么时候往里填

- **AI 做模板时** → 挖到小众视角 / 10% 惊喜 / 本地冷知识 → 素材库
- **AI 调研时** → opencli 搜到高收藏爆款 → 爆款参考;发现本地视角/痛点 → 素材库
- **AI 采集数据时** → 发现可发小红书的冷知识/避坑点 → 素材库
- **小雅做内容时** → 从素材库取材 + 从爆款参考学结构 → 发布后回填效果

## 横向联动机制

这个系统**不是靠自觉**,是写进了 SOP:

- `docs/04_操作SOP/模板写作.md` 顶部有 "🔗 横向联动" 提醒
- `docs/04_操作SOP/数据采集.md` 顶部有同样提醒
- `docs/04_操作SOP/研究方法.md` 顶部有同样提醒

AI 做任何这些工作时,读 SOP 就会看到"顺手填素材库"的提醒 → 创作过程中实时沉淀,不等做完才想起。

## 给后续 AI/小雅

**做内容时**:
1. 先看 `strategy.md` + `templates.md` 知道整体方向
2. 去对应地区(如 `japan/kansai/`)看素材库和爆款参考
3. 取 3-5 条素材 + 参考 1-2 个爆款结构 → 起稿

**发布后**:
- 回填素材库第三部分"效果反馈"(哪条真爆了,哪条效果一般)
- 积累 1-2 个月可以做"关西内容规律总结"
