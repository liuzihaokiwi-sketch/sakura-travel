# 06｜冲绳本岛城市圈系统结构表

面向：程序 AI / 城市圈建模 / 前端配置层  
目标：把“冲绳本岛城市圈”固化成可排程、可编辑、可扩展的数据结构。  
默认版本：`okinawa_main_island_circle_v1`

---

## 1. 圈定义

```ts
interface CityCircle {
  circle_id: string;
  name: string;
  region: string;
  core_bases: BaseNode[];
  daytrip_nodes: DaytripNode[];
  major_activity_clusters: MajorActivityCluster[];
  secondary_activity_clusters: SecondaryActivityCluster[];
  default_hotel_strategies: HotelStrategyPreset[];
  recommended_day_range: [number, number];
  expansion_rules: ExpansionRule[];
}
```

### 1.1 建议圈边界

| 字段 | 值 |
|---|---|
| `circle_id` | `okinawa_main_island_circle_v1` |
| `name` | 冲绳本岛城市圈 |
| `region` | Okinawa |
| `core_bases` | 那霸 / 本部半岛(条件型) / 北部海岸(条件型) |
| `daytrip_nodes` | 首里 / 美国村 / 古宇利岛 / 美丽海水族馆 / 座间味或渡嘉敷(条件型) |
| `optional_expansion_nodes` | 山原 / 恩纳 / Kerama 过夜 |
| `recommended_day_range` | `4~8` 天 |
| `hard_cap_without_expansion` | `7` 天 |

### 1.2 默认策略

| 天数 | 默认住法 | 默认核心 |
|---|---|---|
| 3–4 天 | 单基点（那霸）或双基点条件开放 | 那霸 + 中北部 1 天游 |
| 5–6 天 | 双基点（那霸 + 北部海岸/本部） | 城市历史 + 海景/海洋馆 + 海岛活动择优 |
| 7–8 天 | 双基点 / 三基点条件开放 | 可加入 Kerama 或北部自然扩展 |

## 6. 风险标签系统（给活动簇）

### 6.1 字段

```ts
interface ActivityRiskTags {
  precheck_risk: RiskLevel;
  live_disruption_risk: RiskLevel;
  execution_friction: RiskLevel;
  user_condition_risk: RiskLevel;
  known_schedule_dependency: boolean;
  day_of_disruption_possible: boolean;
  backup_same_corridor_required: boolean;
}
```

### 6.2 分类原则

- 船班时刻、海上活动名额、景点休馆、租车库存，属于前置可规避风险。  
- 海况、风浪、台风、强降雨导致的停航或视野变化，属于实时扰动风险。  
- `kerama_marine_day / yambaru_nature / seasonal_whale_watch` 默认必须带 fallback。

## 8. 冲绳圈默认生成模板

### 4–5 天

- 必选主活动：`那霸琉球城市 / 北部海岸+美丽海`
- 画像命中后再加入：`古宇利 or 美国村 or 目的地海景酒店`
- 不默认加入：`Kerama + 山原双重户外高风险`

### 6–7 天

- 默认 4–5 个主活动簇
- 那霸与北部应各有承接
- `Kerama / 山原 / 贵价海景酒店` 三者择二

### 8 天

- 可开放 `Kerama 过夜`
- 可把北部海岸拆成 2 天
- 台风季增强实时风控权重

## 10. 来源锚点（供程序维护）

- JNTO 口径支持把那霸作为本岛入口，再向首里、北部海岸、美丽海、Kerama、山原延展。
- Kerama 与北部自然活动对海况/天气敏感，适合单独建高实时扰动风险标签。
