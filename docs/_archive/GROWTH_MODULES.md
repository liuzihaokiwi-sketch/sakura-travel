# 增长模块设计

> 最后更新: 2026-03-28
>
> 独立于主链的流量获取模块。可单独启动、单独修改，不影响攻略生成系统。

---

## 设计原则

1. **和主链零耦合** — 单独部署，坏了不影响付费用户
2. **复用已有数据** — 实体库、季节活动、城市圈知识包都是现成的
3. **每个模块都是流量漏斗** — 吸引 → 展示价值 → 引导下单

---

## 一、营销内容自动生成

系统已有实体数据 + 城市圈知识 + 季节信息，可以自动批量生成营销素材。

### 内容类型

| 类型 | 格式 | 示例 | 发布渠道 |
|------|------|------|---------|
| 城市攻略要点 | 图文 | "关西7天怎么玩？这5个安排决定了你的体验" | 小红书/公众号 |
| 美食排行 | 图文 | "京都本地人也排队的5家拉面" | 小红书 |
| 预算指南 | 图文 | "7天关西旅行到底花多少钱？逐天拆给你看" | 小红书/抖音 |
| 季节专题 | 图文/视频脚本 | "2026京都赏樱最佳时间+路线" | 全渠道 |
| 避雷指南 | 图文 | "去大阪千万别踩的5个坑" | 小红书 |
| 对比类 | 图文 | "住河原町 vs 京都站前，到底哪个方便？" | 小红书 |
| 冷知识/趣味 | 短视频脚本 | "日本便利店能做的10件你不知道的事" | 抖音 |
| 行程展示 | 视频脚本 | "我帮客户做的关西7天手账长什么样" | 抖音 |

### 生成流程

```
选题库（人工维护/AI建议）
  → 从实体库+知识包提取素材
  → AI 生成文案（小红书风格/抖音脚本风格）
  → 配图指引（用哪些插画/照片）
  → 输出：可直接发布的图文 / 视频脚本
```

### 技术实现

```
app/domains/content_marketing/       # 独立模块
  topic_pool.py                      # 选题库管理
  generators/
    xiaohongshu_post.py              # 小红书图文生成
    douyin_script.py                 # 抖音视频脚本生成
    wechat_article.py                # 公众号长文生成
  templates/
    city_guide.py                    # "X天Y城市攻略"模板
    food_ranking.py                  # "必吃N家"模板
    budget_breakdown.py              # "花多少钱"模板
    seasonal_special.py              # 季节专题模板
    avoid_traps.py                   # 避雷模板
    comparison.py                    # 对比类模板
```

### 选题自动建议

系统可以根据以下信号自动建议选题：
- **季节驱动**：樱花季前1个月自动生成赏樱内容
- **热搜驱动**：接入小红书/抖音热搜词，匹配已有数据的话题
- **数据驱动**：哪个城市圈的实体数据最丰富，就优先生成该圈内容
- **转化驱动**：哪类内容带来的下单最多，加大同类生产

### 每周产出目标（建议）

| 渠道 | 频率 | 内容 |
|------|------|------|
| 小红书 | 3-5篇/周 | 图文为主 |
| 抖音 | 2-3条/周 | 视频脚本（你拍摄） |
| 公众号 | 1篇/周 | 深度长文 |

## 二、独立工具页（流量磁铁）

高搜索量的实用工具页，吸引自然流量，页面底部引导到付费产品。

### 工具列表

| 工具 | 路径 | 数据来源 | 搜索量预估 |
|------|------|---------|-----------|
| **樱花开放预报** | `/tools/sakura` | 季节活动表 + 历年数据 | 极高（每年3-4月） |
| **红叶见顷预报** | `/tools/koyo` | 季节活动表 + 历年数据 | 高（每年10-11月） |
| **旅行预算计算器** | `/tools/budget` | 实体价格数据 | 中高 |
| **行李清单生成器** | `/tools/packing` | 城市圈知识包 + 季节 | 中 |
| **签证材料清单** | `/tools/visa` | 静态知识 | 中 |
| **交通卡选择器** | `/tools/transport-pass` | "你的行程适合买哪张交通卡" | 中 |
| **餐厅预约指南** | `/tools/booking-guide` | "日本餐厅怎么预约，一篇搞懂" | 中 |
| **汇率+小费计算** | `/tools/currency` | API 实时汇率 | 低但实用 |

### 每个工具页的漏斗结构

```
用户搜索"京都樱花什么时候开"
  → 进入 /tools/sakura
  → 看到樱花预报 + 赏樱路线建议
  → 页面底部："想要一份完整的赏樱行程？7天定制手账 ¥198"
  → 点击 → 进入下单页
```

### 技术实现

```
web/app/tools/                       # Next.js 页面，独立路由
  sakura/page.tsx                    # 樱花预报
  koyo/page.tsx                      # 红叶预报
  budget/page.tsx                    # 预算计算器
  packing/page.tsx                   # 行李清单
  visa/page.tsx                      # 签证材料
  transport-pass/page.tsx            # 交通卡选择
  booking-guide/page.tsx             # 预约指南
  currency/page.tsx                  # 汇率计算

web/app/tools/layout.tsx             # 工具页通用布局（含底部引导栏）
```

### SEO 策略

每个工具页需要：
- 中文 title + description 优化
- 结构化数据（Schema.org）
- 页面速度优化（SSG 静态生成）
- 内链到其他工具页 + 主站
- 定期更新（樱花/红叶每年更新数据）

### 数据更新

| 工具 | 更新频率 | 数据源 |
|------|---------|--------|
| 樱花/红叶预报 | 年更（1月更新当年预测） | 日本气象厅 + weathermap.jp |
| 预算计算器 | 季度（跟随实体价格更新） | 实体库价格字段 |
| 交通卡选择器 | 年更（JR 调价时） | 人工维护 |
| 签证材料 | 按需（政策变化时） | 领事馆官网 |
| 汇率 | 实时 | 汇率 API |

## 三、流量→转化完整路径

```
自然搜索 → 工具页（实用价值）→ 底部引导 → 下单
     ↑                                      ↓
小红书/抖音 → 营销内容（兴趣激发）→ 主页/视频 → 下单
     ↑                                      ↓
老用户 → 查看页广告位 → 推荐给朋友 → 下单
                       → 复购其他城市圈 → 下单
```

### 转化追踪

每个入口带 UTM 参数：
- `?from=sakura_tool` — 从樱花预报来
- `?from=xhs_ramen_post` — 从小红书拉面帖来
- `?from=douyin_video_023` — 从抖音第23条视频来
- `?from=referral_TRV-A3K9M2` — 从老用户推荐来

系统记录每个订单的来源，分析哪个渠道 ROI 最高。

## 四、实现优先级

| 优先级 | 模块 | 理由 | 执行方 |
|--------|------|------|--------|
| **P0** | 营销图文生成（小红书） | 最快能产出内容开始获客 | 🧠 Opus 设计模板 → 🤖 Sonnet 实现 |
| **P0** | 视频脚本生成（抖音） | 你拍摄发布，系统出脚本 | 🧠 Opus |
| **P1** | 樱花/红叶预报页 | 季节性爆发流量 | 🤖 Sonnet |
| **P1** | 预算计算器 | 常青流量 | 🤖 Sonnet |
| **P2** | 其余工具页 | 长尾流量 | 🤖 Sonnet |
| **P2** | UTM 追踪 | 优化渠道投入 | 🤖 Sonnet |

**P0 可以和 Phase 1 并行启动**，不依赖实体数据完善。用已有数据就能生成营销内容。

## 五、和主项目的关系

```
travel-ai/
  app/domains/content_marketing/    # 后端：内容生成（独立模块）
  web/app/tools/                    # 前端：工具页（独立路由）

  # 以下是主链，完全不受影响：
  app/domains/planning/             # 决策链
  app/domains/rendering/            # 页面生成
  app/workers/                      # 后台任务
```

content_marketing 和 tools 只**读**实体库和知识包，不**写**任何主链数据。
主链挂了工具页照常运行，工具页挂了主链不受影响。
