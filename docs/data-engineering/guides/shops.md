# 宝藏店铺精选指南

> 版本: 3.0
> 更新: 2026-04-01
> 上位文档: SELECTION_PHILOSOPHY.md
> 核心改动: 按店铺类型拆数据源，提高到"精选"标准

---

## 一、定位: 轻量高信号，不追求与餐厅/酒店同等重量

店铺是手账本的"惊喜层"，不是"骨架层"。重要性低于餐厅/酒店/景点。

原则:
- 每条都有明确的"为什么逛这家"，但**不需要**数值化评分体系
- 不收"还行"的店，只收"真的有趣/独特"的
- 按店铺类型分别找最权威的来源
- 保持轻量: 6个必填字段即可，不堆与餐厅酒店同重量的schema

---

## 二、按类型拆数据源

**不同类型的店铺有不同的最佳信息来源，不能全靠小红书。**

| 店铺类型 | 最佳来源 | 辅助来源 |
|----------|---------|---------|
| 古着/中古 | 本地时尚媒体(F-STREET/装苑ONLINE/FINEBOYS) | 小红书+独立站 |
| 工艺/特产 | 协会官网、博物馆店、百货买手清单 | japan-guide/JNTO工艺专题 |
| 商店街 | 官方商店街名录、地元新闻 | 乐吃购/独立站 |
| 杂货/书店 | 设计媒体(Pen/Casa BRUTUS)、书店协会 | 小红书/独立站 |
| 市集/跳蚤 | 官方活动日历、地元旅游局 | 小红书(实拍) |
| 伴手礼 | 百货地下食品层、机场免税店名录 | 携程/独立站 |

### 日文数据源特别重要

宝藏店铺的信息，日文来源通常比中文来源更准确更全面:

| 日文来源 | 覆盖 | 特别价值 |
|----------|------|---------|
| F-STREET (f-street.org) | 古着店门户 | 大阪/东京古着店最全信息 |
| 装苑ONLINE (soen.tokyo) | 时尚/古着 | 编辑型推荐，有深度 |
| Hanako Web | 生活方式 | 街区特色店精选 |
| Japan Shopping Now (japanshopping.org) | 购物全品类 | 按店型分类，官方信息 |
| 各商店街官方网站 | 商店街 | 店铺一览、营业信息 |

## 三、三层池执行

### 发现池

```
Step 1: 按类型搜索专业来源(日文优先)
  古着/中古:
    - F-STREET "{城市}" 页面
    - 装苑ONLINE 古着特集
    - "{城市} 古着屋 おすすめ" 日文搜索
  工艺/特产:
    - "{城市} 伝統工芸 ショップ"
    - JNTO 工艺/购物页面
    - 各产业协会官网(如京焼清水焼協会)
  商店街:
    - "{城市} 商店街 おすすめ"
    - 各商店街公式サイト
  杂货/书店:
    - "{城市} 雑貨屋 おすすめ"
    - "{城市} 独立書店"

Step 2: 中文独立站
  - 乐吃购购物专题
  - 小红书 "{城市}必逛" "{城市}宝藏店铺"
  - 台湾/香港博主购物攻略

Step 3: 携程+Google Maps
  - 携程购物频道
  - Google Maps 按类型搜索
```

### 入围池

```
进入条件:
- 专业来源推荐(如F-STREET收录) + 至少1个其他来源
- 3+个独立原创推荐(去重后)
- 官方商店街名录中的特色店 + 有体验描述

不进入:
- 仅小红书1篇提及
- 纯连锁店(3COINS/LoFt不收录为"宝藏"，可能作为"便利购物"单独处理)
- 已关闭/搬迁
```

### 终选池

```
编辑判断:
- "如果逛这片区域，这家店值得专门走进去看看吗？"
- 有明确的"为什么逛这家"(独特选品/历史/氛围/只有这里有)
- 允许稀疏: 某个街区没有宝藏就不硬凑

终选标签:
  must_visit: 该类型公认最值得逛的
  recommended: 多个来源推荐
  worth_checking: 有独特之处但样本少
```

## 四、店铺类型定义

| 类型码 | 中文 | 精选标准 |
|--------|------|---------|
| vintage_shop | 中古/二手精品 | 选品品质+价格竞争力+购物体验 |
| furugi | 古着店 | 选品风格+价格+店铺氛围 |
| zakka | 杂货店 | 独特性(不收纯连锁) |
| antique | 古董店 | 货品质量+可信度 |
| craft | 手作/工艺 | 工艺水平+是否可体验/定制 |
| local_specialty | 地方特产 | 是否是该地真正的名物 |
| bookstore | 独立书店 | 选书品味+空间体验 |
| record_shop | 唱片店 | 品类深度 |
| market | 市集/跳蚤 | 定期举办+规模+有趣度 |
| shopping_street | 特色商店街 | 整体氛围+特色店密度 |

## 五、坐标获取(省Google API)

```
优先免费方式:
1. 店铺官网(日本店铺官网常有地图)
2. Google Maps手动搜索(浏览器，不用API)
3. OpenStreetMap
4. 攻略站文章中的地图信息
5. 商店街官网通常有地图

最后才用Google Places API。
```

## 六、负向编辑规则(店铺)

| 信号 | 处理 |
|------|------|
| 纯连锁(全国都有) | 不收录为"宝藏" |
| 营业不稳定 | 标注"建议出发前确认" |
| 已大众化/失去宝藏感 | 考虑不收 |
| 只适合特定人群(如只卖动漫周边) | 标注，收在对应标签下 |
| 价格虚高(游客定价) | 标注或不收 |

## 七、数据结构(最小契约)

店铺不需要和餐厅/酒店同重量的schema。**6个必填字段 + 基础定位字段**即可:

### 必填6字段

| 字段 | 含义 | 说明 |
|------|------|------|
| why_go | 为什么逛这家 | 具体理由，不是泛泛的"值得一逛" |
| what_to_expect | 进去能看到/买到什么 | 具体到品类和价位 |
| skip_if | 什么情况不用去 | 反向过滤，帮用户省时间 |
| where | 区域/走廊/最近站 | 方便行程编排 |
| when | 营业窗口 | HH:MM-HH:MM + 定休日 |
| evidence | 来源证据 | 至少2个来源，且至少1个专业/官方/编辑源 |

### 完整JSON结构

```json
{
  "id": "kyo_shop_001",
  "name_zh": "...",
  "name_ja": "...",
  "type": "shop",
  "shop_type": "bookstore/vintage_shop/furugi/...",
  "style_zh": "一句话定位",

  "city_code": "kyoto",
  "corridor_tags": ["..."],
  "address_ja": "...",
  "nearest_station": "...",
  "coord": [lat, lon],

  "when": {
    "open_hours": "HH:MM-HH:MM",
    "closed_notes": "..."
  },

  "descriptions": {
    "why_go": "为什么逛这家(具体理由)",
    "what_to_expect": "进去能看到什么",
    "skip_if": "什么情况不用去"
  },

  "best_for": ["couple", "culture_deep", "photo"],
  "selection_level": "must_visit/recommended/worth_checking",
  "tags": ["..."],

  "evidence": {
    "sources": ["f_street", "xiaohongshu", "letsgojp"],
    "source_count": 3,
    "has_professional_source": true
  }
}
```

注意: 店铺**没有**数值化评分(没有grade S/A/B/C，没有review_signals)。
用 `selection_level`(must_visit/recommended/worth_checking) 做粗粒度分层即可。
```
