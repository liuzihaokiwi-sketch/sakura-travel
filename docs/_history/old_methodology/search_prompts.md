## 任务 A：旅行体验质量的深层维度（通用）

```
你是一个研究旅行体验设计的专家。我们在做一个付费旅行手账本产品，需要搜索"如何从根本上提升旅行体验质量"的深度内容。

我们已经知道的（不用再搜，搜到了直接跳过）：
- Peak-End Rule（峰终定律）
- 同类体验连续会导致记忆干扰
- 负面峰值规避比制造正面峰值更重要
- 自由漫游/留白设计的价值

你要搜索的是我们还不知道的维度，方向包括但不限于：
- 身体舒适节奏（体力曲线管理、什么时候该坐下来休息）
- 感官丰富度（一天中视/听/味/触/嗅的切换设计）
- 独处与共享的平衡（旅行中需要安静消化的时刻）
- 意外感/发现感（被完全安排 vs 自己发现的体验，心理差异是什么）
- 时间感知（旅行中"时间变慢"的条件是什么）
- 归属感/连接感（与当地人和文化产生瞬间连接的条件）
- 叙事感（回来之后能讲出一个故事而不是流水账，怎么设计）
- 期望管理（营销文案过度渲染如何损害实际体验）
- 同行关系动态（旅伴关系如何影响个人体验质量）

搜索方法：
1. 先搜英文学术/深度文章（"memorable travel experience research"、"tourism experience quality psychology"、"travel pace design"、"sensory experience travel"），筛3-5篇真正有洞察的
2. 再搜中文深度内容（知乎/虎扑/豆瓣长文，关键词"旅行体验提升"、"怎么旅行才不像赶场"、"旅行感受力"、"旅行审美"）
3. 每轮搜索后停下来思考：哪些是已经知道的常识？删掉。哪些是真正的盲区？保留。然后调整下一轮搜索方向。
4. 如果某个方向搜了两轮都是常识，放弃这个方向，换一个。

最终输出10条左右我们真正不知道的洞察，每条写：
- 原理是什么
- 对旅行手账本产品的具体影响
- 应该由谁响应：装配层prompt / 模板编写规范 / 内容池字段

宁可少而精，不要多而泛。如果某个方向确实没有新东西，直接说"这个方向没有盲区"，比硬凑10条更有价值。
```

## 任务 B1：关西城市圈

```
你是深度了解日本关西的旅行专家。我们在做关西旅行手账本（付费产品，298元），需要理解每个城市的"灵魂"和"城市间化学反应"。

覆盖城市：京都 / 大阪 / 神户 / 奈良 / 宇治 / 姬路

【第一部分：每个城市的灵魂与正确打开方式】

对每个城市搜索并提炼：
1. 核心魅力本质（这个城市给人的独特感受，一句话，别处替代不了的）
2. 感知条件（什么时间/天气/心态/节奏下才能感受到这个魅力）
3. 常见的"打开方式错误"（游客最常犯的让体验打折的错误）
4. 当地人视角 vs 游客视角的差异（当地人觉得理所当然但游客不知道的事）
5. 这个城市无法在其他城市替代的体验（如果只能去一次，非做不可的事）

【第二部分：城市间化学反应与节奏切换】

搜索并提炼：
1. 城市顺序如何影响整体体验（先京都后大阪 vs 反过来，感受有什么不同）
2. 两城市之间的切换需要什么"过渡"（从京都的安静切到大阪的嘈杂，需要缓冲吗）
3. 哪些城市组合会互相增强体验？哪些会互相削弱？
4. 每个城市待几天体验最饱和？超过几天开始审美疲劳？
5. 奈良/宇治/姬路这种 day trip 目的地，插在哪一天效果最好？

搜索方法：
1. 先搜英文（在日本住过的外国人写的深度反思，"kyoto travel mistakes"、"osaka local tips"、"kansai itinerary order which first"、"destination fatigue kyoto"）
2. 再搜中文（小红书/知乎/马蜂窝的长文，不是攻略清单，而是"去了才懂"/"后悔没早知道"类型）
3. 每个城市搜2-3轮，搜完一个城市停下来想：这条信息对"装配一个好行程"有具体指导意义吗？如果只是"京都很美"这种废话，删掉。
4. 城市间化学反应部分，重点搜多次去关西的人写的"第二次/第三次去的感受"，他们的顺序偏好最有参考价值。

最终输出：
- 每个城市3-5条真正有指导意义的洞察
- 城市间化学反应5-8条规则
- 每条标注：该写进装配规则（结构化）还是模板编写原则（自然语言）

如果某个城市（比如姬路）确实内容有限，如实说，不要硬凑。
```

```
北海道地广，城市间距离远，跟关西完全不同。重点搜：
1. 北海道行程的核心节奏错误（贪多、赶路、没留够在一个地方的时间）

3. 每轮搜完停下来想：这条信息能直接影响行程装配吗？不能的删掉。

最终输出：
- 每个城市/地区2-4条真正有指导意义的洞察
- 北海道特有节奏规则5-8条
- 每条标注：装配规则 还是 模板编写原则
```
