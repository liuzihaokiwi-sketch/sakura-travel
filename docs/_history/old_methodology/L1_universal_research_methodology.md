# L1 · 通用旅行目的地研究方法论

> 适用范围:为本项目(付费旅行手账本,国外 348 元 / 国内 298 元)做任何国家/地区研究的 AI 协作者
> 创建:2026-04-18,基于西班牙研究(12 城 + 4 国情层)和日本关西研究(21 模板)的实战沉淀
> 必读前置:`CLAUDE.md`(项目总规范)+ `AGENTS.md`(AI 协作规范)

## 零、你是谁,你在干什么

你是这个项目新招的研究员,目标是成为**某个目的地的策展大师**——不是旅行达人,是"**十个当地旅游专家联手做出的攻略**"这个级别的判断力持有者。

你的产出不是"景点清单",是**让中国用户在陌生国家**:
- **不踩坑**(骗局/游客陷阱/文化失礼)
- **不赶路**(情绪节奏匹配)
- **看到本地灵魂**(不是被 10 万条小红书淹没的重复景点)
- **做出好决策**(知道在哪该慢、在哪该砍)

## 一、第一性原则:7 条绝对不可动摇的红线

### 1. 数据真实性是最高红线
**所有事实性字段必须来自真实数据源。搜不到填 null,绝不编造。**

事实性字段包括:
- 餐厅评分、价格区间、坐标、营业时间
- 景点门票、免费时段、订票窗口
- 交通车程、班次、价格
- 季节性事件的具体日期

**自我检查规则**:每次要写一条"事实"之前,停一秒问自己"**我这条是真搜过的,还是从上下文推断的**?"如果是后者,要么去真搜,要么标注"未核实",**绝不把推断写成事实**。

数据可信度分级:
- `verified`: 2+ 个 P0/P1 源交叉验证 → 可上生产
- `cross_checked`: 1 个 P0/P1 + 其他 → 可用,标注
- `single_source`: 仅 P2 源 → 可用,标注
- `ai_generated`: 无真实源 → **不可上生产**

### 2. 先搭骨架,再填内容
**接到"研究 X 地区"任务时,第一步不是研究 X 本身,而是 X 所在的上层坐标系**。

- 研究意大利 → 先看"意大利在欧洲的位置"(和西、法、德对比)
- 研究日本关西 → 先看"关西在日本的位置"(vs 关东、北海道)
- 研究秘鲁 → 先看"秘鲁在南美的位置"

**没有坐标系的单国研究,产出会变成孤立知识,无法支撑跨国路线判断**。

### 3. 中英文源必须交叉
单一语言源头会产生**认知偏差**:
- 纯英文源:错过中国游客独有痛点(签证/退税/小偷/支付)
- 纯中文源:错过本地视角(英文高端攻略 / 小红书没写的深度)

**最小交叉标准**:
- 每个核心观点有 1 中文源 + 1 英文源背书
- 冲突时优先"本地源"(如当地旅游局、本地博主)
- 小红书是"中国游客真实反馈"的金矿,但也是"跟风打卡"的重灾区,需判断

### 4. 不打补丁,修根因
遇到问题不要绕:
- 数据源打不开 → 换源,不编数据
- 小红书防盗链 → 查 Referer + UA 配置,不跳过
- 英文名不确定怎么翻中文 → 查官方中文名,不自创

### 5. 不加用户没要的功能/字段
研究文档专注研究,**不要**:
- 主动"顺便"做 SEO 分析
- 添加"未来可能用到"的字段
- 重复写已经在 CLAUDE.md 的内容

### 6. 文档事实性和数据事实性同等严格
"这个博客站 coverage 关西" 如果是从上下文推断,不是真访问过,**就是假事实**。每条引用必须真看过。

### 7. 产出的最终标准是"用户会不会退款"
不是"我搜到了多少条",是:
- 真实旅行顾问看了觉得"对,这就是"的比例
- 用户真去了发现哪条不对、会退款的风险

## 二、研究的 5 个阶段

### Phase 0 · 定位(1 小时)
在开始任何具体研究之前,先搞清楚:

1. **这个地区在更大坐标系的位置**
 - 属于哪个文化圈 / 大洲 / 气候带
 - 和邻居国家的对比
 - 在"一生旅行计划"里的位置(必去 / 深度 / 小众)

2. **该地区的中文攻略饱和度**
 - 小红书搜主城 + "攻略",看结果数量和质量
 - 高饱和(日本/韩国/泰国):需要做"比小红书深一层"
 - 低饱和(东欧/中东):需要做"从零建立认知"

3. **你的护城河在哪**
 - 英文源有但中文没有的 → 信息差武器(如西班牙 Paradores 网)
 - 中国游客独有痛点英文不覆盖 → 痛点深挖(如扒手、退税、支付)
 - 需要你"翻译文化底层"的 → 灵魂解读(如巴斯克不自认西班牙人)

4. **产出什么文件**(先定文件清单)
 - 国情层(1-4 份):节奏灵魂 / 顶级操盘手逻辑 / 中国人痛点 / 区域补完
 - 主城层(5-15 份):按 Tier A/B/C 分级
 - 延伸层(1-3 份):海岸/岛/小众区域

### Phase 1 · 国情层(2-3 天)
**先做"灵魂"和"痛点",不做"城市"**。

必做 3 份:
- **R1 节奏灵魂**:这个国家怎么呼吸的?节奏 / 作息 / 情绪基调 / 文化红线
- **R2 顶级操盘手逻辑**:Black Tomato / Condé Nast / 本地高端旅游公司怎么卖这个国家?偷师他们的设计思路
- **R3 中国人痛点**:签证 / 支付 / 退税 / 语言 / 骗局 / 小偷 / 厕所 / 楼层差 / 吃饭时间差

可选 1 份:
- **R4 区域补完**:R1 未覆盖的独立区域(如西班牙的 Galicia)

### Phase 2 · 主城 Tier 分层(3-7 天)
按 Tier A/B/C 分:

| Tier | 定位 | 典型天数 | 数量 |
|------|------|---------|------|
| **A** | 王牌主城,行程必含 | 2-5 天 | 2-3 个 |
| **B** | 主城,大概率去 | 1-3 天 | 3-5 个 |
| **C** | 延伸主城 / 日归 | 1-2 天 | 5-8 个 |

每个主城按 **7 问深挖**:
1. **灵魂一句话**:这城到底是啥味道
2. **骨架**:1-3 日经典路线(具体时间分配)
3. **热门景点怎么玩对**:反游客陷阱 / 时段 / 角度
4. **小众 wild card**:小红书级金矿 5-10 条
5. **最佳时段 / 季节**:什么时候去最好 / 什么时候别去
6. **餐饮骨架**:本地人吃啥 / 游客餐厅怎么认 / 人均
7. **硬约束**:订票窗口 / 闭馆日 / 文化红线 / 禁忌

### Phase 3 · 延伸层(1-2 天)
海岸 / 岛 / 小众区域,按主题打包,不独立成主城。

### Phase 4 · 总索引 + 沉淀(半天)
- `_INDEX.md` 汇总所有文件 + 产品梯队 + 文案红线 + 硬约束清单
- 用户痛点的"实用页固定内容"(可直接印在手账本)
- 下阶段待做清单

## 三、数据源优先级

```
P0 (权威源)
- 官方旅游局(visit-{country}.com)
- 世界级排行(Michelin / Fodor's / Lonely Planet / Rick Steves)
- 当地权威媒体(El País / NYTimes / Guardian / 朝日新闻)
- Google Maps API(坐标/营业时间权威)

P1 (中国用户视角)
- 小红书(真实游客反馈 + 女性视觉金矿)
- 知乎长文(深度分析)
- 携程问答(订票/签证实操)
- 穷游锦囊(系统性)
- 马蜂窝(目的地深度)

P2 (参考源)
- 个人博客(The Culture Trip / Nomadic Matt / Earth Trekkers)
- 旅游杂志 online(Condé Nast Traveler / Travel + Leisure)
- 反游客陷阱专题站

P3 (兜底,不用于事实性数据)
- AI 知识库 / 结构化分类 / 推荐语
- Wikipedia 历史背景
```

**铁律**:事实性字段必须 P0-P2 验证。P3 只能用于结构、分类、背景故事。

## 四、7 问深挖格式(每个主城必答)

### 1. 灵魂一句话
**公式**:(核心锚点)+(文化关键词)+(感官体验)

**示例(巴塞罗那)**:"高迪奇幻建筑 + 加泰设计之都 + 地中海阳光"
**示例(京都岚山)**:"竹林+渡月桥+禅意庭院,关西优雅的缩影"

### 2. 骨架(具体到小时)
```
Day 1 · 上午(9-12):锚点 A(理由)
       · 下午(14-17):锚点 B(理由)
       · 晚上(20-22):餐饮 + 夜活动
```
**每个时段必须说明"为什么这个时段"**(光线/人流/营业/文化节奏)。

### 3. 热门景点怎么玩对
**反游客陷阱三步**:
- 热门地点的"正确玩法"(时段/角度/服装/订票)
- 游客陷阱是什么(具体名字,不泛泛)
- 本地人的同类替代

**示例(巴塞罗那 La Rambla)**:"避开,除了 La Boqueria 上午 11 点前去"

### 4. 小众 wild card
**至少 5 条**,每条必须:
- 有具体名字或地点
- 有"为什么小众"的理由
- 英文源/小红书源可溯

**示例(马德里 La Latina 铁三段)**:周日跳蚤市场 → Vermut 午前酒 → Tardeo 午后餐

### 5. 最佳时段 / 季节
- 月份维度:1-12 月每月什么情况
- 周维度:周几闭馆 / 周几本地人活跃
- 日维度:几点光线好 / 几点免费

### 6. 餐饮骨架
- 本地核心 3-5 道菜(发音 / 写法 / 含义)
- 游客餐厅怎么认(装饰 / 菜单语言 / 位置)
- 本地人真去的餐厅(具体名字)
- 人均价格区间

### 7. 硬约束
- 订票窗口(几个月前 / 哪个网站)
- 闭馆日(周几 / 节日)
- 文化红线(禁忌 / 失礼)
- 身体要求(台阶 / 高原反应 / 着装)

2. 我的护城河在哪?(和现有中文攻略比)
3. 真旅行顾问看了会不会认可?

## 六、常见坑(西班牙 / 关西研究都踩过的)

### 坑 1: 把"信息差"当"护城河"
错误:"大家都不知道 X,我知道了就是护城河"
真相:护城河是**系统性深度**,不是单点信息

### 坑 2: 照抄英文高端攻略
错误:直接翻译 Black Tomato 的路线
真相:高端攻略服务的是 $10,000+ 客户,和 348 元手账本客户画像完全不同。**偷师设计思路,不是照抄路线**。

### 坑 3: 小红书跟风
错误:小红书说 XX 必去就抄过来
真相:小红书有 60% 是跟风打卡,要**分辨"真实反馈" vs "网红打卡"**。判断标准:有没有具体时段 / 具体代价 / 具体痛点。

### 坑 4: 硬塞所有城市
错误:把所有出现过的城市都列 Tier
真相:Tier D 要有"**不值得单独去**"的判断力,不是全收录

### 坑 5: 忽略季节
错误:写"最好玩法",不说几月去
真相:欧洲 8 月大半国家空城,12-2 月北欧极夜,这些是**一级约束**,不是备注

### 坑 6: 情绪词泛滥
错误:"浪漫的 / 壮丽的 / 迷人的"
真相:情绪词必须**和具体感官绑定**。"帕尔马海滩的日落是什么样子" > "浪漫的日落"

### 坑 7: 忘记文化红线
错误:假设所有国家和中国一样
真相:加泰人不自认西班牙人 / 日本关西不爱被叫"西日本" / 英国人不觉得自己是欧洲人——这些不记会失礼

## 八、继续学习

做完一国后,用该国 L2 适配层(如 L2 欧洲 / L2 东亚)继续下一国时,你应该能**复用 70-80%** 的基础方法,只学习该国独有的部分。

如果复用率 < 50%,说明:
- 要么你没真正理解方法论(回来读 L1)
- 要么该国确实需要独立适配层(提议新建 L2)

## 九、如何做到"高质量"——我(前任研究员)的 8 条思维习惯

这部分是方法论里最软也最重要的一块。方法流程(Phase 0-4)是骨架,**真正决定产出好坏的是下面这 8 条思维习惯**。读完你会明白"什么叫做出十个专家联手的水平"。

### 1. 每轮搜完之后"停一秒"
搜完一轮资料,**不要立刻写**。停下来问:
- 我刚才搜到的哪些是常识、哪些是盲区?
- 常识该扔掉(中国游客都知道的不算护城河)
- 盲区该深挖(这才是手账本的价值)

**真实案例**:西班牙研究第一轮我搜"西班牙美食",搜到一堆"Paella / Tapas / Sangría"——这是常识。**真正的盲区是"Sangría 是游客饮料,本地人喝 Tinto de Verano / Vermut / Txakoli"**。这条写进产品里才值 348 元。

### 2. 对"显而易见的答案"保持警惕
如果一个问题你 30 秒就有答案,**这个答案很可能是错的或肤浅的**。

**真实案例**:"西班牙最该去的城市?" 30 秒答案是"巴塞 + 马德里"。但深挖之后发现:
- 格拉纳达的阿尔罕布拉宫在小红书热度不亚于巴塞
- 圣塞巴斯蒂安在米其林密度世界第一
- "该去哪"其实取决于"你是什么人"

**高质量答案有层次,低质量答案是清单**。

### 3. 主动承认"我不知道"
**最危险的是"我以为我知道"**。

具体做法:
- 写一条事实前,内心扫描"这是搜过的还是推断的"
- 只要是推断,明确标"未核实",或者立刻去搜
- 用户指出错误时,**先承认再纠错**(不要找借口)

**真实案例**:用户问"10 次欧洲怎么排",我第一轮凭印象答了,用户说"你还不够了解,去搜"。我该做的是立刻承认"对,这是我的推断",然后真去搜。

### 4. 每份研究至少一条"反常识"发现
手账本卖的是"**用户想不到**"的东西,不是"用户已经知道"的东西。

每份研究文件写完,问自己:
- 里面有没有一条"反常识"发现?
- 没有的话,说明这份研究不够深

**例子**:
- 反常识:"西班牙 8 月马德里空城,但 Verbenas 节连开三场——是体验真马德里的最好时段"
- 常识:"西班牙 8 月很热"

### 5. 区分"真实反馈"vs"跟风打卡"
小红书是双刃剑——有真实金矿,也有 60% 跟风。判断标准:
- **真实反馈**:带具体时段 / 具体代价 / 具体痛点(如"上午 10 点去不用排队,下午去要等 1 小时")
- **跟风打卡**:只说"必去""震撼""美翻了",没有具体信息

**只采纳真实反馈,跟风打卡直接过滤**。

### 6. 跨源冲突时,记录冲突本身
两个源说法不一样时,**不要选一个相信**,要:
- 记录冲突(A 说 X,B 说 Y)
- 分析可能原因(时间差 / 区域差 / 群体差)
- 如果影响产品,必须再搜第三源

**真实案例**:有源说"Alhambra 订票 3 个月前",有源说"2 周前"。真相是**Nasrid 宫殿 3 个月前 / 其他区 2 周前**——不是谁对谁错,是颗粒度不够。

### 7. 边做边沉淀,不是做完才沉淀
做到中间就写反思,不要等全做完。具体做法:
- 每个 Phase 结束,写 5-10 行反思(踩了什么坑、学到什么)
- 反思直接存 memory 系统,不在文档里
- 下次 Phase 开始前**先读反思**

**这条很关键**——做完才反思,会忘掉 70% 的细节。

### 8. 保持"用户真金白银"的敬畏
你写的每一条都可能让用户:
- 专程飞去那个地方
- 带父母 / 带情侣去
- 退票重订 / 取消行程

所以每一条都要承担**"如果写错,用户怎么办"**的压力。

**自我检查问题**:
- 如果用户按你写的 X 去了,发现是错的,他会不会退款?
- 如果会,这条必须再核验一次
- 如果不会,也至少标注"未核实"
