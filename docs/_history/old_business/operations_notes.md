# 运营笔记(边做模板边记,不专门研究)

这个文件的用法:**做模板搜小红书时,遇到有运营价值的东西就顺手记一笔**。不是专门做运营研究,只是防止这些信息被遗忘。

---

## 爆款标题模式(搜模板时碰到的)

- "真不想承认自己是冤种"(金阁寺吐槽型)
- "XX 好看但真的累"(和服清水寺)
- "别穿和服去 XX"(避雷+反向)
- "怒省 XX 日元,XX 平替真的美"(祇王寺 vs 西芳寺)
- "一顿外卖的钱泡京都第一的温泉"(便宜替代)
- "原来《千与千寻》的世界是真实存在的"(IP 关联)
- "我居然在京都学会了..."(新发现)
- "8 点要离开京都,但我会 6 点去鸭川散步"(深度党/早起)
- "这片大众树林,是我每次去京都的动力"(本地人视角)
- "京都 XX 是国人的冷门景点吗"(反差/种草)

**共通规律**:
- 标题要有情绪(自嘲/反转/惊喜),不要"京都攻略""必去"
- 数字越具体越好(¥4000→¥300, 1200 尊石佛)
- **收藏数(collects)比点赞更能反映爆款程度**——很多高赞笔记 likes 只有 10-20 但 collects 1000+

## 真实用户吐槽(可用做避雷贴)

- **金阁寺**:庭园小、10 分钟走完、观景点就 1 个,但门票是平安符设计,带回家当纪念才值
- **京都塔展望台**:¥900 上去后悔,塔底 SANDO 浴场 ¥1050 泡澡才是本地玩法
- **清水寺 + 穿和服**:木屐+台阶+和服大臂不能抬,拍不出好照片(376 赞共识)
- **寺田屋**:1894 年重建不在原址,墙上弹痕是复原
- **嵯峨野小火车(早春)**:风景一般(隧道多),非旺季卖点弱
- **醍醐寺(早春)**:樱花季是主场,3 月去光秃秃
- **茶道体验**:80% 的人到了约不上(正规茶室需 3-6 个月前预约)
- **一天走金阁+龙安+仁和+北野**:体力挑战,很多人半路放弃
- **清水参道上的店**:贵 30% 以上且全是游客,下坡到高台寺附近好得多

## opencli 基本用法(作者女朋友做运营可能用得上)

```bash
cd D:/projects/projects/travel-ai/opencli-main

# 搜笔记
node dist/main.js xiaohongshu search "关键词" --limit 10 --format md

# 读笔记正文
node dist/main.js xiaohongshu note "笔记 url" --format md

# 其他能搜:feed(首页)/user(某用户主页)/comments(评论)/creator-notes(创作者自己)
```

前置:Chrome 装了"opencli Browser Bridge"扩展 + 小红书登录过。`node dist/main.js doctor` 检查连接。

**注意**:看 **collects(收藏)**比 likes 更能判断爆款。likes 可能只显示 1-10 但实际收藏几千。
