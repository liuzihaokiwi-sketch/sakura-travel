# 服务完整流程

> 最后更新: 2026-03-28
>
> 从获客到售后的完整链路。设计优先级：节省人力 > 客户体验 > 规避出错 > 降低系统开销。

---

## 流程总览

```
获客下单 → 填写表单 → 等待生成 → P13预览 → 一轮修改 → 解锁全本 → 实体交付 → 回访运营
```

---

## 阶段一：获客与需求提交

### 1.1 下单（双渠道）

| 渠道 | 流程 | 阶段 |
|------|------|------|
| 抖音小店 | 顾客咨询 → 拍下订单 → 客服发表单链接 | 当前 |
| 独立站 | 顾客浏览 → 下单支付 → 自动跳转表单 | 后期 |

**先付款再填表**：付过款的用户有动力认真填，未付款的不占系统资源。

### 1.2 填写表单

表单结构：8屏 + 1确认页（详见 `travel_form_multistep_frontend_v3.md`）

三层设计：
- **必填（1-4屏）**：去哪、什么时候、和谁、预算住宿 — 懒人1-2分钟搞定
- **选填（5-6屏）**：偏好、去过/不想去 — 不填出稳妥版
- **补充（7-8屏）**：已订内容、特殊要求 — 没有直接跳

表单始终为动态表单，支持草稿保存，用户可中途离开后继续填写。

**避雷选项（第5屏新增）**：
```
以下情况你可以接受吗？不勾选我们会帮你避开：
□ 环境比较简陋但味道好的小馆子
□ 服务态度不太好但出品稳定的店
□ 排队超过30分钟的热门店
□ 只收现金的店
```

**提交规则**：
- 点击"提交，开始这趟旅行"后表单锁死
- **提交后30分钟内可撤回重填**（此时未开始生成），页面明确提示
- 30分钟后锁死，修改需联系客服

## 阶段二：系统处理与预期管理

### 2.1 生成专属码

提交成功后系统即刻生成专属码（格式：`TRV-A3K9M2`，6位字母数字，不含姓名）。

- 页面提示截图或复制保存
- 明确告知：这是后续查阅、修改、售后返现的唯一凭证
- 后期可改为微信授权登录查看，专属码仅作客服备用

### 2.2 拉高期待值

页面不立刻出结果，显示预期时间。

**实际机制**：
- 系统真实生成时间：约30分钟
- 告诉用户：预计 2-4 小时
- 生成完成后 **2小时后推送通知**（不是秒出，也不是半天）
- 用户体验 = "说2-4小时，实际2小时就好了" → 超预期

### 2.3 后台处理

```
表单提交
  → 归一化（normalize_trip_profile）
  → 城市圈决策链（10步管线）
  → 页面生成（PlanningOutput → 章节 → 页面 → ViewModel）
  → AI文案填充（copy_enrichment）
  → 质量门控
      → 自动通过（>80%）→ 直接进入待展示
      → 未通过 → 进人工审核队列 → 审核通过后进入待展示
  → 到达预设通知时间 → 推送"您的攻略已生成"
```

**一次性生成全部手账**，前置层免费展示，每日详细层确认后解锁。原因：
- 用户不修改（预计70%）→ 直接解锁，秒完成
- 用户要修改 → 只重新生成受影响部分
- 一次生成 vs 两次生成，成本差一倍

## 阶段三：精华展示与AI交互修改

### 3.0 下单前转化（旧样本 + 视频）

用户还没付款时，通过以下方式建立信任：
- 展示其他城市圈的完整攻略样本（测试生成的旧版本）
- 攻略讲解视频（真人翻阅讲解）
- 用户评价/旅行后反馈

**下单前看到的是别人的完整攻略，下单后看到的是自己的定制精华。**

### 3.1 前置层展示（Part 1 + Part 2）

用户凭专属码进入，看到前置层全部内容。**大方展示，炫技**——让用户觉得"钱花对了"。

三段结构：

```
Part 1 (精华展示): 让用户感受到"安排得真好"
Part 2 (预约行动): 有时效，现在就要做
剧透页
Part 3 (每日执行): 确认/修改后解锁
```

**Part 1: 精华展示**（具体的，不模糊）

| 页面 | 展示什么 | 用户感受 |
|------|---------|---------|
| 封面 | 城市名 + 定制感 + 品牌 | 仪式感 |
| 地图跨页 | 完整路线 + 每天区域标注 | "整个行程一目了然" |
| 旅行气质 | 为什么这样安排 + 节奏定位 | "他们真的想过" |
| 亮点跨页 | 具体的、勾人的体验亮点 | "第三天有个惊喜晚餐" |
| 路线逻辑 | 取舍原因 + 体力曲线 | "连为什么不去某个热门点都解释了" |
| 酒店策略 | 具体酒店名 + 为什么选这家 + 区域优势 | "连住哪都替我想好了" |
| 餐厅总表 | 具体店名 + 招牌菜 + 预约方式 | "这些餐厅我查了下真的很好" |
| 预算分布 | 每天大概花多少 + 哪里值得花 | "花得明白" |
| 出发准备 | 交通卡/通信/行李/App | 实用 |
| 安全应急 | 紧急联系/礼仪/天气应对 | 安心 |

**Part 2: 预约行动清单**（有时效，放在剧透页前）

```
距离您出发还有 18 天，以下项目建议尽快预约：

🔴 紧急（3天内截止）
   [无]

🟡 本周内
   1. ○○割烹（祇園）— Tabelog预约 — 建议出发前10天
      → 链接: tabelog.com/...
      → 推荐: 午餐套餐 ¥3500

🟢 出发前完成即可
   2. 岚山小火车 — 官网购票 — 建议出发前3天
   3. ○○温泉旅馆 — 电话预约 — 可请酒店前台代订
```

**剧透页** — "从下一页开始，这趟旅行会慢慢展开。"

**前置层页数不固定**，按行程复杂度浮动：

| 行程复杂度 | 前置层页数 | 说明 |
|-----------|-----------|------|
| 5-6天简单行程 | 12-14页 | 酒店/餐厅较少 |
| 7-8天标准行程 | 14-16页 | 标准展开 |
| 9-10天复杂行程 | 16-18页 | 更多酒店/餐厅/预约项 |

### 3.2 自动提醒机制

系统在生成时计算所有预约项的截止日期，自动推送提醒：

| 时间点 | 动作 |
|--------|------|
| 生成完成时 | "您的攻略已生成，有X项需要提前预约，请尽快查看" |
| 截止日期前7天 | "○○餐厅建议本周预约" |
| 截止日期前1天 | "🔴 ○○餐厅明天就到预约截止日了" |
| 超过截止日期 | "建议预约时间已过，仍可尝试，或查看备选方案" |
| 出发前1天 | 汇总：所有预约确认状态 + 出发清单 |

推送渠道：微信服务号 > 短信 > 邮件。

### 3.2 一轮免费修改（AI辅助）

**是"一轮"不是"一次"** — 用户可以和AI多次对话直到满意。

```
用户在文本框输入修改意见（大白话）
  → AI提取需求，弹窗确认："是我理解的这样子吗？您希望……"
  → 用户补充/修正 → AI再确认
  → 循环，直到用户满意
  → 点击"确认这次修改" → 本轮修改结束
  → 页面提示：后续如需再次大改，需联系客服补差价
```

**修改边界识别**：
- **偏好调整**（AI可处理）："第二天换成购物"、"少安排寺庙"、"预算压低一点"
- **基本信息变更**（需重新生成）："其实是5天不是3天"、"去错城市了"
  → 系统识别后提示"这个改动较大，需要重新生成，请联系客服"

**AI多轮无法满足时**：自动提示"您的需求比较复杂，已转交专属客服为您处理"。

### 3.3 修改后等待

用户确认修改后，重新生成受影响部分。

- 控制在 **5分钟内**（只重新生成变动的天/页面，不是全部60页）
- 页面显示进度提示："正在根据您的意见调整中…"
- 完成后自动刷新

### 3.4 退款政策

| 阶段 | 可否退款 | 说明 |
|------|---------|------|
| 填表前/填表中 | 全额退款 | 未开始生成 |
| 前置层展示阶段 | 全额退款 | 用户看到了精华但未解锁执行细节 |
| 点击"确认"后 | 不可退款 | 已解锁全部每日执行内容 |
| 7天未操作 | 自动确认，不可退款 | |

前置层虽然展示了具体酒店/餐厅名，但不构成可执行行程（缺每日时间线、路线串联、plan B）。退款率预估 < 3%。

## 阶段四：终态交付与资产沉淀

### 4.1 解锁全本

用户确认无误（或修改完成）→ 解锁全部每日执行层。

- 凭专属码/微信登录，**永久可查看**
- 提供 **PDF下载**（主交付物，不依赖服务器长期在线）
- 查看页面长期保留，可加入广告位/复购引导

### 4.2 实体产品交付

| 步骤 | 时间 | 说明 |
|------|------|------|
| 用户确认最终版 | T+0 | 解锁全本 |
| 进入印刷队列 | T+0 | 自动触发 |
| 印刷完成 | T+3~5天 | 含手账本 + 贴纸DIY包 |
| 快递发出 | T+5~7天 | 推送快递单号给用户 |

（前期如不做实体产品，此步跳过，仅交付数字版PDF）

## 阶段五：回访与客户运营

### 5.1 旅行前提醒

| 时间点 | 动作 |
|--------|------|
| 出发前7天 | 推送预约提醒："以下餐厅/景点建议本周预约" |
| 出发前1天 | 推送出发清单："行李/证件/交通卡确认" |

### 5.2 旅行中（被动）

不主动打扰。用户可随时凭码查看攻略。

### 5.3 旅行后回访

| 时间点 | 动作 | 目的 |
|--------|------|------|
| 返程后3天 | 推送回访消息 | 收集反馈（哪些推荐好/哪些踩雷） |
| 返程后7天 | 邀请发小红书/抖音视频评价 | 内容传播 |
| 好评视频确认后 | 客服凭专属码返现 | 码消费掉，返现记录关联 |

### 5.4 长期运营（查看页面）

用户永久查看页面可用于复购引导：

| 位置 | 内容 |
|------|------|
| 页面顶部 | "这趟旅行结束了？看看下一趟→" |
| 页面底部 | 其他城市圈推荐卡片 |
| 侧边栏 | "推荐给朋友，双方各减50" |
| 季节性 | "您去年去了关西，今年北海道薰衣草季正好→" |

### 5.5 反馈数据回流

用户回访反馈直接回流到系统：
- "这家餐厅关了" → 标记实体状态
- "这家很好吃" → 加强推荐权重
- "排队太久了" → 更新排队预估
- "这个景点没意思" → 降低质量档

## 超时/异常处理

| 场景 | 处理 |
|------|------|
| 生成完用户不来看 | 2小时推送通知 → 24小时再推 → 7天未查看客服联系 |
| 质量门控未通过 | 进人工审核，用户等待时间不变（在预设通知时间内处理完） |
| AI修改多轮失败 | 自动转人工客服 |
| 付款后不填表 | 24小时提醒 → 72小时再提醒 → 7天未填联系确认 |

## 状态机

```
paid → form_filling → form_submitted → generating → quality_check
  → auto_approved → preview_ready → user_viewing
    → modification_requested → regenerating → preview_updated → user_confirmed
    → user_confirmed（无修改）
  → manual_review → reviewed → preview_ready

user_confirmed → full_unlocked → printing → shipped → delivered
  → feedback_collected → review_rewarded（码消费）
```

### 后期恢复原价

基础价上调 ¥70（7天 → ¥268），每天加价和拆册费不变。

### 页数规则

每天固定6页，不压缩。前置层按行程复杂度浮动：

```
总页数 = 前置层(12-18页) + 每日层(天数×6) + 旅后层(5页)
```

| 天数 | 前置 | 每日 | 旅后 | 总页数(取偶) |
|------|------|------|------|------------|
| 5天 | ~12 | 30 | 5 | ~48 |
| 6天 | ~14 | 36 | 5 | ~56 |
| 7天 | ~14 | 42 | 5 | ~62 |
| 8天 | ~16 | 48 | 5 | ~70 |
| 9天 | ~16 | 54 | 5 | ~76 |
| 10天 | ~18 | 60 | 5 | ~84 |

总页数取最近偶数（印刷要求）。5-6天行程加赠空白旅行内页+额外贴纸，包裹体感与7天一致。

### 拆册规则（11天+）

**必须一次性生成完整行程再拆为两册**，避免：
- 餐厅/活动重复推荐
- 酒店策略前后不一致
- 路线逻辑断裂

**拆分点选择**（系统自动）：
1. 优先：城市圈切换日（天然断点）
2. 其次：换酒店日（行李在动，自然分界）
3. 兜底：行程中点

**上册末尾加2页衔接**：
- "下册预告"：后半程路线预览 + 3个期待亮点
- "行程交接"：明天酒店/交通、行李处理、需提前做什么

**下册开头精简前置（4页）**：
- P1：独立封面（有"下册"标识）
- P2：上册回顾（1页总结 + 共享地图标注两册范围）
- P3：下册路线总览
- P4：下册酒店/预算/注意事项（不重复上册通用信息）
- P5起：每日展开

两册共享同一个专属码。

### 实体产品规格

#### 纸张选型

**内页首选：哑粉纸（无光铜版纸）128g**

| 纸张 | 彩印效果 | 书写 | 耐久 | 手感 | 结论 |
|------|---------|------|------|------|------|
| 双胶纸（白）120g | 中，颜色偏淡 | 好 | 好 | 像打印纸 | 能用但廉价感 |
| 蒙肯纸 120g | 差，吸墨暗沉 | 一般 | **会发黄** | 轻柔但廉价 | **排除** |
| **哑粉纸 128g** | **最好，饱和不反光** | 一般 | 好 | **高级杂志感** | **首选** |
| 纯质纸 120g | 较好，柔和 | 最好 | 好 | 温润高端 | 备选（贵） |

选哑粉纸的理由：
- 手绘/油画风格插图在哑粉纸上色彩还原最好，不反光，能呈现笔触质感
- 128g 够厚不透（双面印不会透到背面），铁圈装订无压力
- 用户翻开第一反应是"这纸质好"而不是"像打印的"
- 打印店一般有哑粉纸，成本比双胶纸高 ~¥4-7/本

**蒙肯纸直接排除**：会发黄长斑，产品定位是"值得保存的纪念品"，放半年变黄就毁了。

如果打印店没有哑粉纸，退选**双胶纸（象牙白/米白）120g**（不要纯白，像A4纸）。

#### 装订方式

铁圈（线圈）装订：
- ✅ 180度平摊（旅途中写字盖章）
- ✅ 360度翻折（单手拿着看）
- ✅ 适合各种页数（48-84页）
- ✅ 兼容 PVC 袋页/折叠地图/可撕明信片
- 后期量大可升级为双线圈（Twin-wire），质感更好
