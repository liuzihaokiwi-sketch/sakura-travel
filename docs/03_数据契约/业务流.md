# 业务流(从下单到交付)

> 合并 business/ops/SERVICE_FLOW.md + business/ops/OPS_GUIDE.md + business/intake/ + business/page_system/ 的跨流程部分。
> 最后更新:2026-04-20

## 文档第一性原理

这份文档回答"**从用户下单到拿到纸质手账本,完整业务流程是什么**"(含客服动作、表单结构、提醒机制、解锁机制、发货)。
- 不回答"代码怎么跑"(看 `数据流.md`)
- 不回答"字段长什么样"(看 `SCHEMA.md`)
- 不回答"客服话术具体怎么说"(看 `05_情境手册/客服应对.md`)

读完应该能:
- 清楚每个阶段谁做什么(用户/系统/客服)
- 知道 4 屏表单的结构和必填项
- 知道预览-纠偏-解锁-发货每一环的触发条件

## 精髓(唯一判断锚)

> **每一步都要让用户感受到"被认真对待",而不是"交易被机器处理"。**

付费手账本 ¥298/348 的利润空间有限,但用户对"仪式感"的期待不低。业务流的所有决策根依据:
- 省人力 > 客户体验 > 规避出错 > 降低系统开销(不是反过来)
- 每个触点都**既有系统也有人**(生成是系统,审核有人;发货是系统,关怀有人)
- 给用户"认真"的感觉比技术炫技更重要

## 一、流程总览

```
获客下单 → 填写表单 → 等待生成 → P13预览 → 一轮修改 → 解锁全本 → 实体交付 → 回访运营
```

---

## 二、阶段一:获客与需求提交

### 2.1 下单(双渠道)

| 渠道 | 流程 | 阶段 |
|---|---|---|
| 抖音小店 | 顾客咨询 → 拍下订单 → 客服发表单链接 | 当前 |
| 独立站 | 顾客浏览 → 下单支付 → 自动跳转表单 | 后期 |

**先付款再填表**:付过款的用户有动力认真填,未付款的不占系统资源。

### 2.2 填写表单(4 屏结构)

三层设计:
- **必填(1-4 屏)**:去哪、什么时候、和谁、预算住宿 — 懒人 1-2 分钟搞定
- **选填(5-6 屏)**:偏好、去过/不想去 — 不填出稳妥版
- **补充(7-8 屏)**:已订内容、特殊要求 — 没有直接跳

表单始终动态,支持草稿保存,用户可中途离开后继续填写。

**避雷选项**(第 5 屏):
```
以下情况你可以接受吗?不勾选我们会帮你避开:
□ 环境比较简陋但味道好的小馆子
□ 服务态度不太好但出品稳定的店
□ 排队超过 30 分钟的热门店
□ 只收现金的店
```

**提交规则**:
- 点击"提交,开始这趟旅行"后表单锁死
- **提交后 30 分钟内可撤回重填**(此时未开始生成),页面明确提示
- 30 分钟后锁死,修改需联系客服

## 三、阶段二:系统处理与预期管理

### 3.1 生成专属码

提交成功后系统即刻生成专属码(格式:`TRV-A3K9M2`,6 位字母数字,不含姓名)。

- 页面提示截图或复制保存
- 明确告知:这是后续查阅、修改、售后返现的唯一凭证
- 后期可改为微信授权登录,专属码仅作客服备用

### 3.2 拉高期待值

页面不立刻出结果,显示预期时间。

**实际机制**:
- 系统真实生成时间:约 30 分钟
- 告诉用户:预计 2-4 小时
- 生成完成后 **2 小时后推送通知**(不是秒出,也不是半天)
- 用户体验 = "说 2-4 小时,实际 2 小时就好了" → 超预期

### 3.3 后台处理

```
表单提交
  → 归一化(normalize_trip_profile)
  → 装配第一步(generate_plan_preview):选模板 + 选酒店
  → 装配第二步(generate_handbook_final):配餐 + AI 文案填充
  → 质量门控:
      → 自动通过(>80%)→ 直接进入待展示
      → 未通过 → 进人工审核队列 → 审核通过后进入待展示
  → 到达预设通知时间 → 推送"您的攻略已生成"
```

**一次性生成全部手账**,前置层免费展示,每日详细层确认后解锁。原因:
- 用户不修改(预计 70%)→ 直接解锁,秒完成
- 用户要修改 → 只重新生成受影响部分
- 一次生成 vs 两次生成,成本差一倍

### 3.4 客服主动消息触发(基于表单字段)

用户提交表单后由客服主动发送(不在表单页/预算页展示):

| 触发字段(表单) | 客服发送 |
|---|---|
| 屏 2 风格=亲子感 或 屏 4 未声明"不想去 USJ",且天数 ≥ 7 | USJ 门票方案对比(普通/快速通关/VIP),含当日人流预测和推荐组合 |
| 屏 4 文本框/补充项包含"和服",或风格=出片感/约会感 | 京都和服店推荐(3 家不同风格价位),含预约链接 |
| 预算页 吃饭档位 ≥ 好好享受 | 需要提前预约的餐厅清单,客服可代订或提供预约指导 |
| 天数 ≥ 6 且风格 ∈ {经典版, 出片感, 约会感}(大概率含岚山) | 嵯峨野小火车抢票时间和技巧,旺季客服可代抢 |
| 预算页 住宿档位 ≥ 享受型 或 屏 4 提及温泉/宿坊 | 推荐房型对比 + 注意事项(宿坊规矩、温泉礼仪等) |
| 所有订单(默认触发) | 关西周游卡 vs ICOCA vs JR Pass 的个性化计算和建议 |

## 四、阶段三:精华展示与 AI 交互修改

### 4.1 下单前转化

用户还没付款时,通过以下方式建立信任:
- 展示其他城市圈的完整攻略样本(测试生成的旧版本)
- 攻略讲解视频(真人翻阅讲解)
- 用户评价/旅行后反馈

**下单前看到的是别人的完整攻略,下单后看到的是自己的定制精华。**

### 4.2 前置层展示(Part 1 + Part 2)

用户凭专属码进入,看到前置层全部内容。**大方展示,炫技**——让用户觉得"钱花对了"。

三段结构:
```
Part 1 (精华展示): 让用户感受到"安排得真好"
Part 2 (预约行动): 有时效,现在就要做
剧透页
Part 3 (每日执行): 确认/修改后解锁
```

**Part 1: 精华展示**(具体的,不模糊):

| 页面 | 展示什么 | 用户感受 |
|---|---|---|
| 封面 | 城市名 + 定制感 + 品牌 | 仪式感 |
| 地图跨页 | 完整路线 + 每天区域标注 | "整个行程一目了然" |
| 旅行气质 | 为什么这样安排 + 节奏定位 | "他们真的想过" |
| 亮点跨页 | 具体的、勾人的体验亮点 | "第三天有个惊喜晚餐" |
| 路线逻辑 | 取舍原因 + 体力曲线 | "连为什么不去某个热门点都解释了" |
| 酒店策略 | 具体酒店名 + 为什么选这家 + 区域优势 | "连住哪都替我想好了" |
| 餐厅总表 | 具体店名 + 招牌菜 + 预约方式 | "这些餐厅我查了下真的很好" |
| 预算分布 | 每天大概花多少 + 哪里值得花 | "花得明白" |
| 出发准备 | 交通卡/通信/行李/App | 实用 |
| 安全应急 | 紧急联系/礼仪/天气应对 | 安心 |

**Part 2: 预约行动清单**(有时效,放在剧透页前):

```
距离您出发还有 18 天,以下项目建议尽快预约:

🔴 紧急(3 天内截止)
   [无]

🟡 本周内
   1. ○○割烹(祇園)— Tabelog 预约 — 建议出发前 10 天
      → 链接: tabelog.com/...
      → 推荐: 午餐套餐 ¥3500

🟢 出发前完成即可
   2. 岚山小火车 — 官网购票 — 建议出发前 3 天
   3. ○○温泉旅馆 — 电话预约 — 可请酒店前台代订
```

**剧透页**:"从下一页开始,这趟旅行会慢慢展开。"

**前置层页数按行程复杂度浮动**:
| 行程复杂度 | 前置层页数 |
|---|---|
| 5-6 天简单 | 12-14 页 |
| 7-8 天标准 | 14-16 页 |
| 9-10 天复杂 | 16-18 页 |

### 4.3 自动提醒机制

系统生成时计算所有预约项的截止日期,自动推送:

| 时间点 | 动作 |
|---|---|
| 生成完成时 | "您的攻略已生成,有 X 项需要提前预约,请尽快查看" |
| 截止日期前 7 天 | "○○餐厅建议本周预约" |
| 截止日期前 1 天 | "🔴 ○○餐厅明天就到预约截止日了" |
| 超过截止日期 | "建议预约时间已过,仍可尝试,或查看备选方案" |
| 出发前 1 天 | 汇总:所有预约确认状态 + 出发清单 |

推送渠道:微信服务号 > 短信 > 邮件。

### 4.4 一轮免费修改(AI 辅助)

**是"一轮"不是"一次"** — 用户可以和 AI 多次对话直到满意。

允许的修改范围:
- 调整节奏/密度
- 换酒店区域
- 换餐厅偏好
- 增减某类体验(博物馆/购物/户外)
- 换城市分配比例

**不允许**(硬规,需二次下单或客服定制):
- 完全换地区
- 改天数
- 超预算档位升级

### 4.5 解锁全本

用户确认后,Part 3 每日执行层解锁,整本手账本可下载 PDF + 进入实体印刷队列。

## 五、阶段四:实体交付

- **印刷**:A5 纸质,全彩,60 页左右(按行程浮动)
- **贴纸 DIY 包**:随本附赠,用户可自己贴照片/装饰
- **发货**:顺丰,3-5 天到
- **打包**:品牌包装 + 手写卡片(仪式感)

### 5.1 客服跟进节点

| 时间点 | 动作 |
|---|---|
| 发货 | 物流单号 + "您的手账本已发出" |
| 到货 | 询问"收到了吗""第一眼感觉怎么样" |
| 出发前 3 天 | "准备好了吗,有什么要帮忙的" |
| 回来后 3-7 天 | 询问体验,收集反馈,邀请发小红书 |

---

## 六、阶段五:回访与运营

- **售后反馈**:回来后问 3 个核心问题(最满意/最失望/会不会推荐朋友)
- **小红书激励**:发小红书带指定标签 → 返现 10-20 元
- **二次转化**:积累老客户,下次去新地区可享折扣

---

## 七、产品边界(不走主流程,直接客服定制)

- 第二次来某地区的用户(要小众深度)
- 奢华游(酒店 ¥3000+/晚)
- 已预订酒店的用户
- 预约失败场景(任天堂没抽中)
- 恢复日特殊需求

服务 < 10% 特殊用户的功能默认留给客服定制线,主产品标准化。
