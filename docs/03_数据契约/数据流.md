# 数据流 + 运行时流

> 合并 architecture/FLOW_DATA.md + FLOW_RUNTIME.md。
> 最后更新:2026-04-20

## 文档第一性原理

这份文档回答"**数据从用户填表到纸质本子交付,中间流过哪些环节**"——包含用户视角(数据流)+ 代码视角(运行时调用链)。
- 不回答"字段长什么样"(看 `SCHEMA.md`)
- 不回答"业务流程怎么跑(表单/客服/发货)"(看 `业务流.md`)
- 不回答"具体代码实现"(看 `app/` 源码)

读完应该能:
- 一眼看清从填表到收本子,用户经历哪 6 步
- 知道运行时代码的主链路(API → 装配 → 预算 → 渲染)
- 知道每个环节读的是哪份数据(事件 / 模板 / 事实层)

## 精髓(唯一判断锚)

> **每一步:用户在做什么 + 系统在做什么 + 读的是哪份数据 — 三个问题必须同时能回答。**

这份文档的作用是**让任何接手系统的人,30 秒内知道数据在哪流动**——若某一步答不上这三个问题,说明系统在这一步有黑盒。

## 一、用户视角:6 步

```
填表 → 生成方案 → 预览确认 → 选预算 → 渲染手账本 → 交付
```

| 步 | 用户在做 | 系统在做 |
|---|---|---|
| 1. 填表 | 4 屏轻表单(日期/城市/人群/节奏) | 收集表单数据 |
| 2. 生成方案 | 等待 | 装配引擎生成 N 日方案 |
| 3. 预览确认 | 看方案,调城市/风格/节奏 | 应用纠偏,重新装配 |
| 4. 选预算 | 选经济/中档/高档 | 计算升降档混搭 |
| 5. 渲染手账本 | 等待 | 60 页 PDF 生成 |
| 6. 交付 | 收到纸质手账本 + 贴纸 DIY 包 | 客服发货 |

## 二、运行时视角:代码主链路

```
POST /v2/trips
  ↓
创建 TripRequest → 入队 generate_trip job
  ↓
─── 装配阶段 ───
读表单约束 → 硬筛候选池(模板+实体) → Opus 装配 → 输出方案草稿
  ↓
GET /v2/trips/{id}/plan-preview
  ↓
─── 用户纠偏(可选多轮) ───
POST /v2/trips/{id}/plan-actions
  ↓
应用 action → 局部/全量重装配
  ↓
POST /v2/trips/{id}/plan-confirm
  ↓
─── 预算阶段 ───
GET /v2/trips/{id}/budget-options → 吐三档
POST /v2/trips/{id}/budget-confirm → 选定档位
  ↓
─── 渲染阶段 ───
装配输出 → 页型映射 → magazine 排版 → PDF
  ↓
job 完成,通知客服发货
```

## 三、每一步对应的代码入口

| 运行步 | 代码入口 | 读哪份数据 |
|---|---|---|
| 入队 | `app/api/trips_v2.py` | — |
| 装配(第一步)| `app/workers/jobs/generate_plan_preview.py` | 模板 + 事实层 |
| 装配(第二步)| `app/workers/jobs/generate_handbook_final.py` | 模板 + 事实层 |
| 候选池硬筛 | `app/domains/planning_v2/opus_assembler.py` | L0 事件 + `when` 字段 |
| 纠偏 | trips_v2.py `plan-actions` 端点 | — |
| 预算 | `app/domains/planning_v2/budget_calculator.py` | policy + 餐厅池 |
| 渲染 | `app/domains/rendering/` | 事实层(票价/时间) |
| Worker 入口 | `app/workers/jobs/generate_trip.py` | — |

> ⚠️ 当前重构(2026-04-20)后这些代码路径的 import 还指向老数据位置(content/kansai/、data/events/),下一步代码重构时统一修。

## 四、运行时读的"活数据"位置

不是代码也不是文档,是装配/渲染真实读取的数据:

| 事实源 | 位置(新结构) | 谁读 |
|---|---|---|
| L0 事件(樱花/红叶/祭典) | `{大洲}/{圈}/events/` | 装配(判断 `when.event_ref`) |
| 票价/营业时间/预约 | `{大洲}/{圈}/live_facts__{city}.json` | 渲染(填进手账本) |
| 模板库 | `{大洲}/{圈}/{season}/*.json` | 装配 |
| entity 静态数据 | `{大洲}/{圈}/entities__{city}.json` | 装配 + 渲染 |
| 内容池(餐厅/酒店/店铺) | `{大洲}/{圈}/{池名}__{city}.json`(未来) | 装配 |
| 圈注册表(code/name/tier) | `data/circle_registry.json` | 装配启动时读一次 |
| 客服追问模板 | `data/followup_templates.json` | 客服工具调用 |
