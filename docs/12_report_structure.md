# 04 攻略生成与交付架构

> 版本：v2.1  
> 更新时间：2026-03-21  
> 角色：产品 / 方案设计 / 工程 / 模板 / 渲染  
> 权威性：最高（交付结构以本文件为准）

## 总原则
不要用 token 堆页数，要用**结构设计 + 条件页 + 模板块 + 少量高价值 AI 解释**堆价值感。

---

## 一、整份攻略 = 3 层结构

### 第 1 层：总纲（必须有）
放在整份攻略最前面，建立“这不是拼出来的”感觉。

固定包含：
1. 为什么这份攻略这样设计
2. 一眼看懂总览
3. 关键预订提醒
4. 出发前准备 / 总注意事项

### 第 2 层：每天固定骨架（每天必须有）
每天固定 4 页：
1. 今日执行总览  
2. 为什么今天这样排  
3. 今日亮点展开  
4. 今日注意事项 + Plan B

### 第 3 层：条件页（按需触发）
只在有价值时出现，不机械重复。

---

## 二、总纲详细定义

### 1. 为什么这份攻略这样设计
目标：让用户感到“有人替我认真想过”。

内容：
- 为什么先去这个区，再去那个区
- 为什么哪天轻、哪天重
- 为什么住这个区域
- 这套路线比普通游客路线好在哪里
- 这份方案的 3–5 个小巧思

### 2. 一眼看懂总览
内容：
- 总路线示意
- 每天所在城市 / 区域
- 强度分布
- 酒店更换情况
- 全程亮点

### 3. 关键预订提醒
内容：
- 需要提前订的酒店 / 餐厅 / 门票
- 最迟什么时候订
- 哪些不订会影响体验

### 4. 出发前准备 / 总注意事项
内容：
- eSIM / 支付 / 交通卡
- 行李与穿着建议
- 日本基础注意事项
- 紧急联系与常用 App
- 季节性提醒

---

## 三、每天固定骨架：详细定义

### 1. 今日执行总览
这是每天最重要的一页。  
内容：
- 今日时间轴
- 今日城市 / 区域
- 今日亮点
- 今日强度（轻松 / 均衡 / 偏满）
- 步行 / 换乘风险
- 今天最值得期待的 1–2 件事

### 2. 为什么今天这样排
只写 3–5 个判断，不写成长作文。  
例如：
- 早上先去这里，因为人少、光线好、体力最好
- 下午排这条线，因为顺路、适合慢逛
- 没选另一个热门点，因为这趟更偏轻松 / 出片 / 亲子

### 3. 今日亮点展开
每天只展开 1–3 个真正重要的点：
- 核心景点 / 活动
- 拍照建议
- 体验重点
- 附近顺便可逛的点

### 4. 今日注意事项 + Plan B
内容：
- 排队风险
- 营业时间与易踩坑点
- 穿衣 / 鞋子建议
- 如果下雨 / 体力不够 / 晚点，先砍什么
- 当天替代方案

---

## 四、条件页（按条件触发）

### 1. 酒店页
仅在以下情况触发：
- 新入住
- 换住宿区域
- 温泉旅馆 / 高端酒店 / 特色酒店
- 酒店本身明显影响当天体验

### 2. 餐厅页
仅在以下情况触发：
- 重点晚餐 / 纪念日晚餐
- 某餐是体验核心
- 某餐对路线价值影响明显

### 3. 交通页
仅在以下情况触发：
- 跨城日
- 复杂换乘日
- 机场 / 早班机 / 晚到达
- 温泉区或郊区进出

### 4. 出片页
仅在强视觉日触发：
- 富士山
- 京都古街
- 夜樱
- 温泉街
- 高审美酒店
- 强景观街区 / 海边 / 山景

### 5. 顺路补充页
只有真的有价值时出现。  
不能为了显得“厚”而硬塞。

### 6. 预算感知页（可选条件页）
适合强调：
- 今天哪里容易花冤枉钱
- 今天哪段消费偏高
- 今天更适合把预算花在哪里

---

## 五、哪些内容必须模板化 / 规则化

### 静态块（不应每次重写）
- 签证提醒
- eSIM / 支付 / 交通卡
- 常用 App
- 医疗 / 紧急联系方式
- 通用安全须知
- 出片基础建议
- 通票基础说明

### 规则块（结构化生成）
- 酒店选择理由
- 餐厅推荐理由
- 路线排序理由
- 交通方式解释
- 预算偏向解释
- 条件页触发逻辑

### AI 块（只留高价值）
- 整份攻略开头的总设计思路
- 每天亮点解释
- 复杂取舍说明
- 用户画像相关备注
- 少量语气润色

---

## 六、页数控制建议
不追求“每天很多页”，追求“看起来厚但不累”。

### 参考范围
- 5 天：26–32 页
- 7 天：34–42 页
- 9 天：40–50 页
- 11–14 天：46–60 页

### 7 天版本理想构成
- 总纲：4 页
- 每天固定骨架：28 页
- 条件页：0–6 页
- 总附录：2–4 页
- 合计：34–42 页

---

## 七、一日体验版结构
不是“缩水版”，而是“完整样片 + 钩子”。

### 必须给
- 1 天完整执行样片
- 路线、交通、餐食建议、预约提醒、当天 Plan B
- 1–2 句专业判断

### 后面只给
- 后续 2–7 天高光预告
- 全程路线脉络感
- 解锁 CTA

---

## 八、交付方式
### 正式交付
- PDF（正式版）
- 网页（补充版）

### 交付说明
- PDF 加轻度个性化水印
- 仅供本人非商业使用
- 网页适合在线查看与自助微调
