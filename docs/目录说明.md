# docs 目录说明

> 2026-04-20 重构后的结构。按"读者要回答什么问题"分类。

## 5 个主目录(数字前缀 = 阅读顺序)

| 目录 | 回答什么问题 | 何时读 |
|---|---|---|
| `01_项目定位/` | 做任何决策前,判断标准是什么 | 新人第一份必读 |
| `02_历史决策/` | 为什么过去这样做,推迟了什么 | 改架构前必读 |
| `03_数据契约/` | 字段/数据流/业务流长什么样 | 做开发前必读 |
| `04_操作SOP/` | 模板/数据/研究怎么做 | 做事前必读 |
| `05_情境手册/` | 遇到 X 怎么办(客服/运营/新圈) | 触发式查阅 |

## 每份文档开头都带

- **文档第一性原理**:这份回答什么、不回答什么
- **精髓**:唯一判断锚,用户视角两句话
- **内容**:吸收旧文档精华 + 搜索补齐 + 自己的思考

## 01_项目定位

- `项目是什么.md` — 产品定位、9 条体验设计原则、6 条禁止行为、商业模式

## 02_历史决策

- `DECISIONS.md` — 架构决策日志(D1-D35,最新 D35 docs/ 重构)
- `DEFERRED.md` — 推迟事项清单

## 03_数据契约

- `SCHEMA.md` — 字段唯一权威源(带"字段还在演化"警告)
- `数据流.md` — 用户视角 6 步 + 代码视角调用链(合并 FLOW_DATA + FLOW_RUNTIME)
- `业务流.md` — 从下单到发货完整业务流程(合并 SERVICE_FLOW + intake + page_system)

## 04_操作SOP

- `模板写作.md` — **最常用**。Part 0 集合策展 + Part 1 前置搜索 + Part 2 字段规则 + Part B 交付自检(带"好模板精髓"两句话判断锚)
- `数据采集.md` — AI 不写事实红线、三轴判断、三层池、效率规则
- `研究方法.md` — 5 阶段研究流程、7 问深挖、MTE 学术框架、搜索策略

## 05_情境手册

- `客服应对.md` — 订单管理、异常 SOP(当前版本偏操作,未来会按"对产品做好"视角重写)
- (运营选题.md / 新圈开城.md — 待写,不急)
