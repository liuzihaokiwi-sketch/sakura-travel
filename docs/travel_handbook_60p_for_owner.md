# 定制旅行手册 60 页完整页码方案（给老板看版）

> 适用版本：7天 / A5 / 全彩 / 实体手册  
> 设计原则：  
> 1. **前半段不剧透**：先给用户必须提前知道的信息，再用“剧透提示页”分隔。  
> 2. **右页重执行，左页轻情绪**：右边放重要内容；左边放插图、故事、出片、提醒、照片位、心情位。  
> 3. **不是堆字，而是堆值感**：路线、酒店、餐厅、理由、预算、Plan B、故事、记忆位，共同组成“值”。  
> 4. **这不是普通攻略**：它既要能执行，也要值得保存。

---

# 一、整本结构总览

- **P1–P12：不剧透前置层**
- **P13–P54：7天详细行程层（每天 6 页 × 7 天 = 42 页）**
- **P55–P60：旅后记忆与附录层**

合计：**60 页**

---

# 二、P1–P12 不剧透前置层

## P1 封面
**页型**：整页  
**内容**
- 城市名称 / 城市圈名称
- 主插图（城市地标 / 气质型插图）
- 品牌名称
- 一句友好互动文案
- 一句带浪漫感的旅程祝福语

**推荐文案方向**
- 「愿这趟旅程，替你把日常之外的心动，一页页收好。」
- 「愿你在这座城市里，走到喜欢的街角，也遇见更松弛的自己。」
- 「这不是一份普通攻略，而是我们替你认真想过的一段旅行。」

---

## P2–P3 总地图跨页
**页型**：跨页  
**内容**
- 旅行区域总地图
- 酒店所在区
- 大致移动范围
- 用户可自行画线的留白区
- 小图例：酒店 / 核心区 / 机场 / 换乘点

**注意**
- 这里只给“范围感”，不把每日细节讲透。

---

## P4 这趟旅行的气质页
**左页（轻）**
- 插图 / 色块
- 三个关键词
- 一句题记

**右页（重）**
- 这趟旅行的整体气质
- 节奏定位（轻松 / 均衡 / 偏充实）
- 适合怎么打开这趟旅行
- 适合怎样的心态来玩

---

## P5–P6 行程亮点跨页
**页型**：跨页  
**内容**
- 不剧透式亮点
- 4–6 个体验型亮点
- 例如：
  - 一顿值得专门留胃的晚餐
  - 一段很适合慢慢走的街区
  - 一个会想停下来拍照的傍晚
  - 一页适合贴上票根和照片的夜晚
  - 一次不用赶路也会觉得值的留白

**注意**
- 不要写“Day 3 去哪里、Day 5 吃什么”，只卖期待感。

---

## P7 这趟旅行为什么这样打开
**左页（轻）**
- 体力曲线小图
- 区域分布小图
- 小提示

**右页（重）**
- 为什么住这些区域
- 为什么有些热门点没放进去
- 为什么这趟不是“打卡越多越好”
- 为什么某些天故意留白
- 为什么这趟会先松后紧 / 先熟悉再深入

---

## P8 住宿总览页
**左页（轻）**
- 酒店区域小地图
- 行李小插图
- 小提醒：可否寄存行李 / 晚到注意事项

**右页（重）**
- 每一晚住在哪里
- 为什么住这里
- 到最近车站 / 核心区域的便利度
- 哪天适合换酒店
- 哪天不建议折腾换酒店

---

## P9 餐厅与预订总览页
**左页（轻）**
- 招牌菜手绘
- 「这些餐值得留胃口」小字块

**右页（重）**
- 需要提前预订的餐厅
- 推荐预订方式
- 招牌菜与推荐点法
- 支付方式
- 排队风险
- 临时休息风险
- 哪些餐“值得认真吃”，哪些餐“可随缘”

---

## P10 预算页
**左页（轻）**
- 钱包 / 小票 / 咖啡 / 拉面等小插图
- 一句预算提示语

**右页（重）**
- 整体预算感知（低 / 中 / 高）
- 这趟钱主要花在哪
- 哪几天消费偏高
- 哪几顿值得花
- 哪里容易花冤枉钱
- 省钱建议（若适用）

**说明**
- 这是“预算感知页”，不是精确会计表。
- 让用户知道该把钱花在何处。

---

## P11 行前准备事项页
**左页（轻）**
- 行李 / eSIM / 交通卡 / 插头 / 穿搭小插图

**右页（重）**
- 出发前 7 天检查项
- 网络 / 支付 / 交通卡
- 穿搭建议
- 常用 App
- 证件 / 保险 / 预约单
- 需要提早下载或处理的事项

---

## P12 安全 / 应急 / 剧透提示跨页中的前半页
**左页（轻）**
- 温泉 / 电车 / 寺社 / 餐厅礼仪提示
- 小图示

**右页（重）**
- 紧急联系电话
- Japan Visitor Hotline
- 台风 / 地震 / 大雨基本原则
- 保险 / 使领馆信息填写位

---

## P13 剧透提示页
**页型**：整页  
**内容建议**
- 标题：**从下一页开始，这趟旅行会慢慢展开。**
- 文案：
  - 如果你想把惊喜留到路上，现在合上也来得及。
  - 从下一页开始，会看到每天的节奏、餐厅、风景和故事。
  - 你可以现在继续翻，也可以到了旅途中再一页页打开。

> 从这一页之后，开始进入详细日程。

# 五、补充说明：预算页应该放哪里

我的建议是：

- **前置预算页（P10）**：告诉用户“这趟大概会把钱花在哪里”
- **旅后预算回顾页（P57）**：让用户自己回头看“这趟钱花得值不值”

这样预算既服务出发前，也服务旅后回味。

# 六、版式原则总结

## 1. 右页一定要稳
右页只放重要内容：
- 路线
- 时间线
- 酒店 / 餐厅
- 选择理由
- 预算
- 风险
- Plan B

## 2. 左页不等于废页
左页是让这本手册变“有温度”的关键：
- 插图
- 故事
- 出片机位
- 礼仪提示
- 小地图
- 照片位
- 心情位

## 3. 不是每页都必须左右分栏
可以适当做：
- 跨页地图
- 跨页亮点
- 整页剧透提示
- 整页照片页
- 整页封面 / 封底

## 4. 每天统一 6 页，用户会更舒服
这样用户很容易建立阅读习惯，也更方便程序自动渲染。

---

# 七、这版 60 页方案的核心价值

这不是一本“信息很多的攻略”，而是一本同时满足：
- **出发前能用**
- **路上能翻**
- **回来还想留**
的旅行手册。

它既能执行，也能保存，还能让用户觉得：
**“这趟旅行，真的像是有人替我认真想过。”**
