# 08 长期产品路线

> 版本：v2.0  
> 更新时间：2026-03-21  
> 角色：创始人 / 产品 / 技术 / 数据  
> 权威性：高（路线图以本文件为准）

## 总原则
先打透日本，再考虑复制。  
不要一开始做平台，要先把一个高毛利、可复用、可扩展的样板市场跑通。

---

## 阶段 1：先打透日本
### 目标
- 跑通从内容 / 搜索 / 分享到成交的闭环
- 跑通预览 → 付费 → 交付 → 自助微调 → 精调
- 建立日本核心城市和主题家族
- 建立标准版与尊享版的边界
- 让 PDF + 网页交付稳定

### 完成标志
- 有稳定成交
- 体验版到付费转化可衡量
- 标准版交付质量稳定
- 自助微调能明显减少人工

---

## 阶段 2：把日本做成高毛利样板市场
### 目标
- 完善第二次日本 / 一地深玩
- 完善高客单主题线
- 完善城市上下文数据
- 完善静态块 / 规则块 / AI 块分层
- 建立离线评测与线上验收

### 完成标志
- 复购开始出现
- 高价单稳定出现
- 内容、增长、交付、私域形成闭环
- 人工只处理高风险或高价值单

---

## 阶段 3：轻工具化
### 目标
- 更强的自助微调
- 更轻的旅中模式
- 更轻的分享与协作
- 更稳的 PDF + H5 双交付体系

### 不是目标
- 不是先做重 App
- 不是做开放式旅行平台
- 不是给用户无限拖拽编排自由

---

## 阶段 4：复制到第二目的地
### 前提
- 日本模型已稳定
- 毛利、交付、复购都有正反馈
- 方案树 / 数据结构 / 文档体系可复制
- 前台与后台映射关系已跑通

---

## 真正的长期护城河
不是 prompt，不是单篇内容，也不是“我会用 AI”。

真正的护城河是：
1. 前台用户输入结构
2. 后台主题家族与方案树
3. 城市上下文数据
4. 规则与条件页体系
5. 预览成交逻辑
6. 自助微调闭环
7. 复购与私域闭环

---

## 不该过早做的事
- 多国家同时展开
- 做成 OTA
- 做复杂开放平台
- 过早追求重 App
- 过早追求复杂 agent 架构
- 过早为极少数需求扩大量 SKU
