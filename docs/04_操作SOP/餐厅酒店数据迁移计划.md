## 一、背景

模板里的 `meal_area` + `cuisine_hint` + `hotel_area_note` 是查询条件，装配时 Opus 需要从餐厅/酒店池里找出最匹配的 A/B 推荐。

原始数据已存在于 `_deprecated/kansai_raw/`，但：
1. 放在 deprecated 目录，未被任何系统消费
2. `area` 字段是 Tabelog 抓取的原始日文地址（带距离），不能直接匹配模板的 `meal_area`
3. 缺少装配所需的关键字段（`name_zh`、`ab_role`、`editor_note` 等）

## 三、目标格式

### 餐厅字段（参考 `_legacy/v1_content/kyoto/restaurants.json`）

```json
{
  "id": "unique_id",
  "name_ja": "店名（日文）",
  "name_zh": "店名（中文音译或译名）",
  "area": "higashiyama",
  "cuisine_tag": "kaiseki",
  "vibe_tags": ["romantic", "quiet"],
  "budget_tier": "high",
  "price_lunch_jpy": 3000,
  "price_dinner_jpy": 8000,
  "score": "tabelog:3.85",
  "ab_role": "A_safe",
  "meal_role": "audience_day_showcase",
  "editor_note": "一句话推荐理由（30字内，朋友口吻）",
  "reservation_required": true
}
```

**关键字段说明**：
- `area`：必须对齐模板的 `meal_area`（见第四节 area 映射表）
- `ab_role`：`A_safe`（默认推荐）/ `B_surprise`（惊喜牌）
- `meal_role`：对应 `产品原则.md` 第4节的 meal_role 枚举
- `budget_tier`：`economy` / `mid` / `high` / `showcase`
- `editor_note`：**装配时 Opus 读这句话**，不能空，不能写"暂无"

### 酒店字段（参考 `_legacy/v1_content/osaka/hotels.json`）

```json
{
  "id": "unique_id",
  "name_ja": "酒店名（日文）",
  "name_zh": "酒店名（中文）",
  "city": "kyoto",
  "area": "gion",
  "hotel_type": "convenient",
  "budget_tier": "mid",
  "price_range_jpy": {"low": 15000, "high": 35000},
  "vibe_tags": ["romantic", "traditional"],
  "facility_tags": ["onsen", "free_breakfast"],
  "quality_bias": "experience",
  "editor_note": "一句话推荐理由（30字内）",
  "audience_fit": {"couple": 5, "family": 2, "friends": 3}
}
```

## 五、执行步骤

### Step 1：整理输出目录

目标文件位置：
```
japan/kansai/
├── restaurants__kyoto.json
├── restaurants__osaka.json
├── restaurants__kobe.json
├── restaurants__nara.json
└── hotels__kansai.json        ← 酒店按城市合并成一个文件
```

### Step 2：迁移餐厅数据

**每个城市执行**：
1. 读原始文件（`_deprecated/kansai_raw/restaurants/{city}.json`）
2. 参考 `_legacy/v1_content/{city}/restaurants.json` 里已处理的样本理解字段语义
3. 逐条转换字段：
   - `area`：用第四节映射表将原始日文地址转为标准 area
   - `price_tier` → `budget_tier`（映射：`local_good`→`economy`，`premium`→`high` 等）
   - `price_lunch/dinner`：从字符串"700-1200"转为 `price_lunch_jpy: 700`（取下限）
   - 补充 `name_zh`（中文译名，可用日文音译）
   - 补充 `ab_role`：tabelog ≥ 3.8 + 评价词含"本地人"→ `A_safe`；小众/创意→ `B_surprise`
   - 补充 `editor_note`：基于 `brief_note` 改写成朋友口吻一句话
   - 补充 `meal_role`：根据 cuisine + area 判断（怀石→`audience_day_showcase`，到达日→`arrival_recovery`）
4. 写入目标文件

**优先级**：京都 → 大阪 → 神户 → 奈良

### Step 3：迁移酒店数据

1. 合并所有酒店文件到 `hotels__kansai.json`
2. 字段转换：
   - 补充 `name_zh`
   - 补充 `budget_tier`（convenient 型默认 economy/mid，experience 型默认 high）
   - 补充 `editor_note`
   - 补充 `audience_fit`（温泉旅馆 couple+5，商务酒店 friends+3 等）
3. 去重（同一酒店可能在多个文件出现）

### Step 4：验证

```bash
# 检查 area 覆盖
python3 -c "

d = json.load(open('japan/kansai/restaurants__kyoto.json'))
areas = set(r['area'] for r in d['restaurants'])
print('区域:', sorted(areas))
"

# 检查必填字段
python3 -c "
import json
d = json.load(open('japan/kansai/restaurants__kyoto.json'))
missing = [r['id'] for r in d['restaurants'] if not r.get('editor_note')]
print('缺 editor_note:', len(missing), '条')
"
```

- [ ] 每条餐厅都有 `area`（标准值）、`ab_role`、`editor_note`、`budget_tier`

- [ ] 无重复 id
- [ ] 文件 UTF-8 编码，JSON 合法

---

## 七、注意事项

1. **不要删原始数据**——`_deprecated/kansai_raw/` 保留不动，迁移是"复制+转换"
2. **editor_note 不能空**——这是 Opus 装配时的核心依据，写"暂无"没有意义
3. **area 必须严格对齐**——自己发明的 area 名装配时匹配不到
4. **v1_content 里的已处理版是最好的参考样本**，先读懂格式再开始批量处理
