## 一、背景与目标

### 为什么要做事实层

模板（`japan/kansai/*/xx.json`）负责"体验逻辑"——哪天去哪、怎么走、情绪弧线。  
事实层（`entities__kyoto.json` / `live_facts__kyoto.json`）负责"客观事实"——票价、时间、交通、官网链接。

**两层分离的理由**：
- 票价会变，模板不该随票价变动而频繁重写
- 同一个景点被多个模板引用，数据只维护一份
- 未来装配代码从 entity_id 读事实，自动注入模板

### 当前状态

- 模板层：**170个 entity_id** 已在模板中被引用
- 事实层：**只有3个 entity** 已完成（嵯峨野竹林、天龙寺、渡月桥）
- 差距：**167个 entity 尚无事实数据**

## 四、数据质量标准

### 可信度分级（必须标注）

| 级别 | 条件 | 是否可用 |
|---|---|---|
| `verified` | 2个以上 P0/P1 源交叉核实 | ✅ 可用 |
| `cross_checked` | 1个 P0/P1 + 1个其他源 | ✅ 可用 |
| `single_source` | 仅1个 P2 源 | ⚠️ 标注后可用 |
| `ai_generated` | 无搜索，纯推断 | ❌ 不可上生产 |

### 数据源优先级

- **P0（最高可信）**：景点/寺社官网、京都市公式観光 Navi、japan.travel（JNTO）
- **P1（高可信）**：Travel Japan、Japan Guide、Lonely Planet、NHK 相关报道
- **P2（参考）**：Wikipedia JA/EN、TripAdvisor、携程官方页（非 UGC）
- **P3（不使用）**：博主游记、小红书攻略帖、AI 推断

### 绝对禁止

- ❌ 写入 AI 推断的价格（必须搜索核实）
- ❌ 把 P3 源当成事实写入
- ❌ 票价写成"约 XX 日元"而不核实（除非真的找不到官方来源，需标 single_source）
- ❌ 营业时间用"一般为 9:00–17:00"等模糊表达，必须给官方来源

| 城市/区域 | 文件路径 |

| 高野山/吉野 | `japan/kansai/entities__other.json`（如不存在则创建） |

### 格式要求

- UTF-8 编码，2空格缩进
- entity_id 作为 JSON key
- 不同 entity 之间加空行
- 同名重复 ID（如 `kyo_nijojo` 和 `kyo_nijo_castle`）：**选一个保留，另一个在 `_alias` 字段标注**

```json
"kyo_nijo_castle": {
  "entity_id": "kyo_nijo_castle",
  "_alias": ["kyo_nijojo"],
  ...
}
```

## 七、执行流程

### 每个 entity 的操作步骤

1. **搜索官网** — 先找官方 URL（Google: `[景点名] 公式 site:kyoto.travel 或 site:japan.travel`）
2. **核实必填字段** — 特别是票价、开放时间、定休日
3. **交叉核实** — 至少找2个源核对票价（价格容易过时）
4. **判断可信度** — 标 verified/cross_checked/single_source
5. **写入对应 entities JSON**
6. **标注 last_verified 为今日日期**

### 完成标准

每个 entity 必须满足：
- [ ] 必填字段全部填写
- [ ] data_confidence 已标注
- [ ] data_sources 至少1个（P0/P1优先）
- [ ] last_verified 已填日期
- [ ] 如有票价，已核实官方来源

## 九、注意事项

1. **票价最容易过时** — 先查官网，不要靠记忆或旧资料
2. **开放时间有季节差异** — 注意夏季延长/冬季缩短，最好标注正常期和特殊期
3. **部分景点免预约变预约制** — 特别是热门景点（醍醐寺/仁和寺/任天堂博物馆等），必须确认当前是否需要预约
4. **重复 ID 要处理** — 见第六节，选定规范 ID 后需要通知负责人更新模板引用
5. **不要批量 AI 生成** — 每个 entity 的票价/时间必须搜索核实，ai_generated 的数据不可用
6. **季节特别参观要单独核** — 红叶/樱花夜间特别参拜的票价每年会变，不要写成固定值，用 `seasonal_notes` 写入并标注"以当年公告为准"
