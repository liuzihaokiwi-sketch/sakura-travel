> 2026-04-22 D36 决策已定（[DECISIONS.md](../02_历史决策/DECISIONS.md) D36），字段定稿在 [SCHEMA.md](../03_数据契约/SCHEMA.md)。

## 第一阶段：Demo 验证（必须先做，1 窗口）

### Step 1.1 demo 模板：岚山 core

文件：[templates/common/kyo_arashiyama_core_full.json](../../japan/kansai/templates/common/kyo_arashiyama_core_full.json)

按新 SCHEMA 完整改造一遍：

- [ ] 砍 25+ 旧字段
- [ ] 写顶层 `note`（设计 note，不展示）：路线为什么这个顺序、体验为什么这样配、情绪节奏怎么走
- [ ] 写顶层 `description`（用户钩子）
- [ ] 写 `curators_notes` 1-4 条策展小巧思
- [ ] `slots` 改成 `{packed, balanced, relaxed}` 三档完整序列（不是删 packed，是写作者重新想 balanced/relaxed 怎么排）
- [ ] slots 数组按 time 时间正序
- [ ] meal slot 不写具体餐厅 entity，只写 meal_area + meal_type
- [ ] 检查 contingencies 是否有真内容（rain_heavy/crowd 等），有就保留有内容子项
- [ ] 写 `applicable_dates: []`（常年模板）

### Step 1.2 demo 5 个景点 entity 补字段

[facts/entities__kyoto.json](../../japan/kansai/facts/entities__kyoto.json) 中：
- `kyo_arashiyama_bamboo` 竹林
- `kyo_tenryuji` 天龙寺
- `kyo_kameyama_park` 龟山公园
- `kyo_jojakkoji` 常寂光寺
- `kyo_togetsukyo` 渡月桥

每个 entity：
- [ ] 字段瘦身到 SCHEMA §2.1.1 必填 8 + 可选 7
- [ ] 把原模板里写过的 photo_spots / fun_facts / nearby_shops 搬过来（按"有比较好的才写"过滤）
- [ ] 写 `notes` 自由文本：装跨景点衔接 tip / 票价指引细分 / 避坑 / 旧 slot.note 里"模板用不到但跟该景点相关"的所有内容
- [ ] 砍 access_difficulty / weather_sensitive / data_confidence 等已废弃字段

### Step 1.3 用户审 demo

跑通后给用户审：
- 模板 JSON 干净度
- entity notes 自由文本读起来是否清晰
- 三档 slots 是否真的是写作者想清楚的三套，而不是同一套删了点

**审通过 = 进入第二阶段批量改造**。审不通过 = 调整 SCHEMA 或写作方法。

## 第三阶段：景点附属内容迁事实层（3-5 窗口）

跟第二阶段并行做。每改造一个模板，把它的 photo_spots / fun_facts / nearby_shops / slot.note 里的景点信息**提炼**到对应 entity。

预估约 80-100 个景点 entity 需要扩字段。优先级：

1. 高频出现的景点（清水寺/金阁/伏见稻荷/天龙寺/二条城/奈良大仏/东大寺等）
2. 季节灵魂景点（哲学之道/醍醐寺/常寂光寺/平野神社/吉野山等）
3. 其他

每个 entity 工作：
- [ ] photo_spots 按"有比较好的才写"过滤
- [ ] fun_facts 按"朋友会忍不住讲的冷知识"过滤
- [ ] nearby_shops 按"懂当地朋友会指给你看的小店"过滤
- [ ] notes 自由文本写跨景点衔接 / 票价 / 避坑

## 第四阶段：填充模板装配花名册（2-3 窗口）

文件：[assembly/模板装配.md](../../japan/kansai/assembly/模板装配.md)

98 个模板每个写一段元属性（季节 / 打分 / 人群加成 / execution_risk / min_days / day_type / exclusive_with / no_pace_downgrade / night_options / selectable_tag）。

打分要按 D36 的乘数公式重打：
- core_experience（0-60）
- audience_bonus（情侣/朋友/家庭，-15 ~ +20）
- execution_risk（0/-1/-2/-3）

打分依据要搜索（小红书 collects > likes 排名 + 英文站 + 官方），不能凭感觉。每个模板 5-8 分钟搜索 + 10 分钟写入。

## 跨阶段并行机会

可以并行的工作：
- 第二阶段（模板改造）+ 第三阶段（entity 扩字段）：每改一个模板顺手改对应 entity
