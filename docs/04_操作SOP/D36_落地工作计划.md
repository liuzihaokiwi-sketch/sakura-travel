模板 `applicable_dates` 直接写月日范围，跟 events/ 里的窗口数据语义重叠。樱花/红叶每年窗口变 → 改 events 还是改模板的 applicable_dates？两份不同步。

### 解决

`applicable_dates` 元素加 `event_ref` 字段，引用 `assembly/templates/data/` 里的 event_def_id：

```json
"applicable_dates": [
  {"event_ref": "kansai_sakura_kyoto", "label": "京都樱花期"}
]
```

装配代码读到 `event_ref` 就去 events 拉当年窗口；没 event_ref 就用 start/end 月日范围。

## 十三、跨阶段并行机会

可以并行：
- **第二阶段**（模板改造）+ **第三阶段**（entity 扩 notes）：每改一个模板顺手改对应 entity
- **第六/七阶段**（餐厅/酒店迁 markdown）：跟其他阶段无依赖，独立窗口跑

不可并行：
- 第一阶段必须最先（demo 不通过其他都白做）
- 第八阶段必须等第二+三阶段完成（代码要适配新数据格式）
- 第四阶段（花名册重打分）建议第二阶段完成后做（模板字段稳定后）
- 第九阶段（event_ref）建议第二阶段开始前做（不然樱花/红叶模板要返工写 applicable_dates）
