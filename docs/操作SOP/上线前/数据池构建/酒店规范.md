## 一、第一性原理

### 1.1 酒店在产品里的角色

**Sleep first, play second**——酒店是用户每天必住·决定动线半径·决定一整天体力分配。
跟餐厅/stops/entity 三类的差别：
- 个体差异最大（同档酒店体验差距 3-10 倍）
- 决策成本最高（用户最焦虑的环节·参考小红书爆款"京都车站 vs 四条" collects 1139）
- 试错代价最高（餐厅难吃换一家·酒店踩雷整晚毁掉）

### 1.2 我们卖什么

「**有人替我想过 + 敢对结果负责**」的确定感。这条是文档所有取舍的总闸：

| 决策 | 第一性追问 | 结论 |
|---|---|---|
| 收不收胶囊/民宿 | 替用户想过 = 不让用户踩雷·民宿房东失联谁负责？ | **不收** |
| 收不收三星经济连锁 | 替用户想过 = 用户睡得舒服·三星体验单调谁负责？ | **不收·底线四星** |
| 50% 数据池给顶奢难订老铺·还是给「准顶级但好订」 | 实际推用户也用不上 = 把负担推给用户 | **50% 主力是 worry_free** |
| 数据源靠搜索还是靠专家清单穷举 | 搜索结果 = 小红书汇总·算「替用户想过」吗？ | **专家清单先行·搜索补漏** |
| 见到不在任何「值的入口」的店 | 没有任何「值」的瞬间·收来干嘛 | **不收** |

### 1.3 工作纪律（元规则）

- **A 类**（装配跑不动 / 字段缺失） → 停下报告
- **B 类**（用户体验硬伤：地址错·档位错·价格离谱） → 停下报告
- **小事**（房型不全 / 含早细节 / 评论摘要） → 标 verified 或留空·**继续做不停**
- 想加新字段 → **停下问用户**（铁律：字段不准膨胀）
- 想收第 1.2 节列的「不收类」 → **停下问用户**

## 二、要不要收（准入）

### 2.1 不收的（一次说完）

#### 整类不收（铁律·不需逐家判断）

| 类型 | 原因 |
|---|---|
| 胶囊酒店 | 体验差·跟「替你想过」脱节 |
| 个人民宿 / Airbnb | 不可控·无客服救济·房东失联风险 |
| 青旅 / 寄宿家庭 | 同上 |
| 三星 / 经济连锁 | 体验单调·**底线四星起** |

**例外**：公司化运营町家（Nazuna / 葵 KYOTO STAY 等）有客服+标准化 = 等同酒店·收。

#### 红线（任一触发即不收·过准入前先扫）

| 红线 | 例 |
|---|---|
| **位置安全坑** | 大阪难波以南/西成区·中国客差评提"流浪汉/治安" ≥3 条 |
| **主源 OTA 任一评分 < 4.3 或评论 < 30** | 主源 = 携程/一休/楽天/JTB/じゃらん/Yahoo·**任一过线即可**（避免日本宿坊被携程覆盖少误剔） |
| **接待外国客不友好** | 不会英文 / 不接受 / 评论提"被冷落" |
| **品牌翻车信号** | 近 1 年小红书爆款差评 ≥1 篇·进 skeleton 观察 |
| **装修/改名/闭店** | 立即降 skeleton 标注·不删 |
| **OTA 长期满房无法订** | 婚庆专用/商务包场·降 skeleton |

### 2.2 收的（双线准入）

#### city 便利型——5 入口（OR·任一强命中即过）

> 总闸门：「这家酒店能不能让用户睡这一晚时·至少有一个『值』的瞬间·或第二天醒来想跟人讲？」

| 入口 | 含义 | 客观信号（携程优先） |
|---|---|---|
| **位置顶**（location） | 走出门就是动线核心·首次客硬通货 | 携程「位置」≥4.7 + 距核心地标 ≤8 分 |
| **品牌顶**（service） | 国际高端品牌或本地顶级老铺 | 主源 ≥4.8 + 国际五星 / Forbes / MICHELIN Keys |
| **体验打包**（package） | city 中带温泉/含早等附加体验 | 主源「服务」≥4.7 + 温泉大浴场 / 含早评价好 |
| **性价比**（value） | 同档位价/质明显胜出·准顶级但好订 | 主源同档前 20% + 价格低于该档中位 20%+ |
| **设计独特**（design） | 非标本·有灵魂·出片 | 小红书 collects > likes 且 ≥500·或大师设计 |

> 命名跟餐厅 5 入口（味道/仪式/独家/招牌/景观）刻意错开·避免 A/B/C 字母混淆。

命中弱时再看是否有第二入口辅助。

#### experience 体验型——两路径（套 entity 准入·OR）

**路径 A：普世价值**——4 维任一头部 + 1-2 个搭配过线：

| 维度 | 例 |
|---|---|
| 氛围/在地 | 俵屋·兵卫向阳閣 |
| 食事 | 西村屋本馆（蟹会席）·高野山宿坊精进料理 |
| 空间 | 翠岚（保津川景）·智积院（名勝庭園貸切） |
| 历史/独特性 | 柊家（200 年）·丸福楼（任天堂创业地）·智积院（国宝障壁画） |

**路径 B：精准价值**——一句话「**谁应该来 + 为什么不可替代**」写进 `note.简介`：
- 智积院 = 「佛教文化+国宝艺术爱好者·京都唯一可住进等伯国宝障壁画寺院的宿坊」
- Nazuna 椿通 = 「想体验整栋町家+半露天风吕·全京都最完整町家群」
- BnA Alter = 「艺术爱好者·每间房 = 一件艺术作品」

**4 维 + 路径 B 全不命中** → 不收（归 city 重新评估）。

### 2.3 数据源（**先穷举种子·后搜索补漏**）

> 跨数据类型方法论 → [搜索方法.md](搜索方法.md)·酒店落实见 [§4.1 酒店映射](搜索方法.md#41-酒店--见-酒店规范md-23)。

**酒店 6 类种子**（关西预估总量 200-300 名字·阶段 1 穷举完才能进阶段 2 搜索补漏）：

| 种子类型 | 穷举范围 |
|---|---|
| 大集团目录 | 万豪/凯悦/IHG/Hilton/Mitsui/Hoshino/Prince/JR Hotels·含副牌 |
| 权威认证 | MICHELIN Keys / Forbes / 一休 100 选 / プロが選ぶ 100 選 |
| 协会/组合 | 有马/城崎旅館協同組合 / 高野山宿坊組合 / 京都老舗旅館会 |
| 类型专题 | 京都市内宿坊穷举 / 京都町家公司化运营商目录 |
| 锚点反向 | S/A 级 entity 周边 1-3 km 查现存酒店 |
| 用户推荐 | 用户/客服主动推荐（如「智积院不错」） |

**关键原则**（详见搜索方法.md §1.3）：数据池以「专家清单穷举」为骨架·搜索结果只做补漏。反例：智积院这类「专家知道、用户搜不到、AI 也搜不到」的宿坊·按搜索路径**永远收不到**。

## 三、怎么写（字段 + 真实性）

### 3.1 字段定稿（9 系统 + 9 note + 3 元数据·不准膨胀）

```json
{
  "id": "kyo_kawaramachi_kyoto_hotel_okura",
  "city": "京都",
  "area": "河原町",
  "near_attractions": [{"entity_id": "kyo_pontocho", "walk_min": 3}],
  "tier": "品质",
  "type": "city",
  "price_cny_per_night": [800, 1500],
  "season_months": null,
  "depth": "full",
  "note": {
    "店名": "京都大仓酒店（京都ホテルオークラ / Kyoto Hotel Okura）",
    "简介": "1888 年创业老牌·河原町御池路口最大体量·鸭川景观房型是签名。",
    "亮点": ["河原町最大格局", "鸭川夜景客房", "1888 年创业"],
    "地址": "京都市中京区河原町御池（地铁东西线京都市役所前站直结）",
    "房型": "标准双人 / 鸭川景观双人 / 行政套房",
    "含早": "可选·¥3,500/人·和洋自助",
    "价格": "淡季最低 ¥600（1月平日）/ 平季中位 ¥800 / 旺季 ¥1500",
    "预约": "官网 / 一休 / 楽天 / 携程·提前 30 天最优惠 15-20%",
    "到店提醒": "Check-in 15:00 / Check-out 12:00·樱花季提前 60 天约必须"
  },
  "可信度": "verified",
  "数据来源": ["https://..."],
  "最后核实": "2026-04-25"
}
```

#### 系统字段（9 个）

| 字段 | 必填 | 说明 |
|---|---|---|
| `id` | ✅ | `{城市拼音}_{区域}_{酒店名}` |
| `city` | ✅ | 中文城市名 |
| `area` | ✅ | 跟 area_registry 对齐·不在 → FAIL |
| `near_attractions` | ✅（≥1） | `[{entity_id, walk_min}]`·entity 必须真实存在 |
| `tier` | ✅ | 4 档枚举·**按平季中位 RMB 归档**·见下表 |
| `type` | ✅ | `city`（便利型·离开酒店玩） / `experience`（体验型·住宿即体验） |
| `price_cny_per_night` | ✅ | `[平季中位, 旺季中位]`·精度 50 或 100·淡季最低写 note.价格 |
| `season_months` | 选填 | 季节限定（如蟹宿冬季）·常年=null |
| `depth` | 选填 | `skeleton` / `verified` / `full` |

**tier 4 档**（按平季中位 RMB·底线四星·**D47 边界对齐 trip.com 关西地区 6 档**·**枚举权威 → [字段权威.md §2.4](../../项目核心/字段权威.md#24-hotels酒店)**）：

| tier | 平季中位 RMB | trip 关西档 | 装配 | 典型 |
|---|---|---|---|---|
| `comfort`（舒适） | 0-600 | 1+2 档 | 装配主力（首次客） | 万豪 Courtyard·Aloft·蒙特利·Tokyu Stay·Mitsui Garden·OMO5 |
| `quality`（品质） | 600-1,250 | 3+4 档 | 装配主力（享受档） | 凯悦嘉轩·THE BLOSSOM·京都格兰维亚·Hotel Kanra·Ace Hotel |
| `luxury`（奢华） | 1,250-2,050 | 5 档 | 高预算 | 京都四季·京都柏悦·三井 MUNI·希尔顿京都·翠岚·空庭テラス别邸 |
| `top`（顶奢） | ≥ 2,050 | 6 档 | **0 次·定制启用** | 京都丽思·Aman·Bvlgari·虹夕诺雅·俵屋·柊家·嘉佩乐·华尔道夫 |

**边界依据**：trip.com 关西地区筛选 6 档（关西用户最常用筛选页）。**后台对照锚·不写入用户可见 note**。淡旺季信息归 `note.价格` 自由文本·tier 永远按平季中位归档。

**体验型跨城兜底**：高野山宿坊一泊二食 ¥800 / 城崎温泉旅馆 ¥1,200 这类按 RMB 可能落 comfort/quality·靠 `type=experience` + `note.亮点` 6 组兜底·不动 tier 边界。

- 价格只决定 tier 不决定 type。智积院 ¥600 = experience+comfort·虹夕诺雅 ¥6000 = experience+top。
- **comfort 下限 ¥300**：大阪/奈良/神户旺季可订到·京都旺季会超档（note.价格 标「樱花/红叶季 ¥800+」·装配按月份切档·不是档位错）。
- **top 边界 ¥4000+**：体验型 ¥4000+（虹夕诺雅/俵屋/Aman）也归此档·**统一不进自动装配**。

**体验型 6 组**（不加字段·写在 `note.亮点` 第一项·**validator FAIL 强约束**·装配按关键词归组）：

| 组 | 含义 | 关西代表 |
|---|---|---|
| `温泉旅馆` | 一泊二食含温泉的传统旅馆 | 兵卫向阳閣·西村屋本馆 |
| `老铺旅馆` | 顶级日式老铺·住宿即体验 | 俵屋·柊家·炭屋·丸福楼 |
| `宿坊` | 寺院宿坊·精进料理+晨课 | 智积院会馆·高野山宿坊群 |
| `町家` | 公司化运营整栋町家·**个人民宿不收** | Nazuna·葵 KYOTO STAY·Hotel Kanra |
| `温泉度假` | 现代温泉度假酒店 | 翠岚·神户六甲山 |
| `设计精品` | 设计师作品·非标本·住一晚出片 | Ace Hotel·Hotel She·OMO 系·BnA Alter |

#### note 字段（9 个）

| key | 必填 | 说明 |
|---|---|---|
| `店名` | ✅ | 中文（本地语言/英文） |
| `简介` | full 必填 | 100-200 字·"懂当地的朋友讲口吻"。**path B 精准价值必写「谁应该来 + 为什么不可替代」** |
| `亮点` | full 必填 | 3-5 个短词标签·体验型第一项写 6 组之一 |
| `地址` | full 必填 | 含最近交通+步行分钟 |
| `房型` | full 必填 | 列主流房型 |
| `含早` | 选填 | "可选/含/无"·查不到写"以官网为准" |
| `价格` | full 必填 | 一句话含 3 时段：淡季最低 / 平季中位 / 旺季 |
| `预约` | full 必填 | 主推渠道 + 提前预约建议 |
| `到店提醒` | 选填 | Check-in/out·特殊政策（如京都旺季提前 60+ 天约） |

#### 元数据（3 个）

| 字段 | 必填 | 说明 |
|---|---|---|
| `可信度` | ✅ | `verified` / `cross_checked` / `single_source` / `ai_generated`（禁上生产） |
| `数据来源` | ✅（≥1 URL） | 数组·携程优先 |
| `最后核实` | ✅ | YYYY-MM-DD |

### 3.2 价格 3 时段法

OTA 真实价格·3 时段中位数：
- **淡季**：1 月平日 / 6 月梅雨平日
- **平季**：5 月/9 月平日
- **旺季**：樱花/红叶/黄金周/年末年始

`price_cny_per_night` = `[平季中位, 旺季中位]`·淡季最低写 `note.价格`·不写淡季最低价进字段（误导）。

**京都旺季订不到**：note.价格 标「樱花/红叶季约 ¥X·常订不到·需提前 60+ 天」·装配按月份切档（不是数据池档位错）。

### 3.3 三轴可信度

每个 full 必须**三轴中至少两轴有真实证据**：

| 轴 | 信号源 |
|---|---|
| quality | MICHELIN Keys / Forbes / 集团官方品牌 / 老铺背书 |
| traveler_fit | 携程评分（首选）+ 小红书 collects > likes / 中文实测博客 |
| execution | OTA 3 时段真实价 / 营业稳定 / 接待外国客 / 交通可达 |

三轴全过 → `verified`·两轴 → `cross_checked`·一轴 → `single_source`·零轴 → `ai_generated`（禁上生产）。

**数据真实性硬规**：价格必须 OTA 真实抓取·含早/房型查不到写"以官网为准"·禁跨店比较句（"X 城最豪华"）。详见 [数据采集.md §一](../数据采集.md)。

### 3.4 校验

`scripts/validate_hotels.py`·跑法：`python scripts/validate_hotels.py`·粘**完整输出**。

- 系统字段白名单（9）+ note 字段白名单（9） → 未知字段 FAIL
- area / near_attractions.entity_id 不在 registry → FAIL
- tier 枚举（`comfort`/`quality`/`luxury`/`top`）+ type 枚举（`city`/`experience`） → 不在 FAIL
- price 必须 `[平季中位, 旺季中位]`·禁 min=max → FAIL
- price 跟 tier 矛盾（`comfort` 但 [2000, 3000]） → 警告
- type=experience 但 note.亮点 第一项不在 6 组（温泉旅馆/老铺旅馆/宿坊/町家/温泉度假/设计精品） → **FAIL**
- depth=full 但 note 字段缺 → FAIL（建议降 skeleton）

## 四、池子长什么样（分布）

### 4.1 总量与每区上限

| 范围 | 上限 | 说明 |
|---|---|---|
| 全圈 full | **100-120 家** | D40 旧池 387 家是全收录·砍 70% 才到精选 |
| 每区 city | 4 档 × 1 首推 + 1 备选 = **最多 8 家** | |
| 每区 experience | ≤ 8 家代表 | 温泉镇/宿坊镇/景区一泊 |
| 京都町家 | ≤ 10 家 | 跨区合集 |
| 顶奢档 | 每城 3-5 家 | 名片即止·不进自动装配 |

### 4.2 区域基因（不平均·按真实需求·**采集时按这个铺**）

不同区域类型酒店分布天然失衡·这是真实的：

| 区域类型 | 关西归属（例） | type 偏向 | tier 偏向 |
|---|---|---|---|
| 商务城市中心 | 梅田/心斋桥/京都站 | 95% city | 舒适/品质偏厚 |
| 古都老镇区 | 祇园/东山/河原町 | 60% city + 40% experience（町家/老铺） | 4 档均匀 |
| 温泉宿坊镇 | 有马/城崎/高野山 | 95% experience | 品质/奢华主力·舒适 ≈ 0 正常 |
| 景区方向 | 岚山/姬路/宇治 | 一泊型 experience 主力 | 看具体景区 |
| 奢华定制目的地 | 京都顶级品牌区 | 看品牌定位 | 顶奢偏多 |

具体归类 → `japan/kansai/这是什么.md` §酒店区域归类。

### 4.3 「省心度」第二轴（X 轴·控制可达性）

> 中国客酒店痛点 #1 不是「贵或便宜」·是「订完会不会踩雷」。

| 省心档 | 含义 | 字段判定（系统自动算·不入库） |
|---|---|---|
| `worry_free` 省心 | 国际连锁 / 携程客服可处理 / 英文 OK | 国际五星集团 OR 携程 ≥4.5+评论 ≥100·有英文服务 |
| `local_friendly` 本地友好 | 日系连锁但接待外国客·中文/英文 OK | 日系大集团（Mitsui/Hoshino/Prince）OR 中文小红书攻略热度高 |
| `effort_required` 需努力 | 需日文沟通·或预约麻烦·或位置偏 | 老铺旅馆/宿坊/町家·常需介绍信或电话 |

#### 配比硬规

| 省心档 | 占比目标 |
|---|---|
| `worry_free` | **50%**（核心·绝大多数用户首次来） |
| `local_friendly` | 30-35% |
| `effort_required` | 15-20%（绝不是主力） |

违反处理：
- `worry_free` < 40% → 优先补 worry_free
- `effort_required` > 25% → 必须剔除部分难订老铺·宁可总数变少

### 4.4 边界 case

- **京都町家**：普通町家公司化运营 → city；高端贷切町家（一栋一组 ¥5000+） → experience
- **国际五星纯城市便利型**（如 Park Hyatt 城市定位 ¥4500+）→ city + 顶奢档（贵但没打包食宿）
- **平价含 2 食温泉旅馆**（¥1500-2500/晚）→ experience + 品质档
- **复合体**（一家有 ryokan 房间也有普通房间）→ 按主营定·不拆两条记录

## 附 历史教训（精简）

- **D40 五档失败**：套话简介 + area 错标 + 价格用合并默认值 → D43 重做（tier 四档 + type 双标签 + OTA 真抓）
- **D43 字段瘦身**：D40 字段超 25 个 → 精简到 9 系统+9 note·细节塞 note 自由文本
- **D46 重写**：单一准入「评分+负向信号」漏宝藏（智积院类） → tier 砍三星·准入双线·种子库穷举先行·worry_free 50% 主力·整类不收（胶囊/民宿/青旅/三星）
- **type 边界判断**：「住宿费打包体验吗」——打包了 experience·没打包 city。价格只决定 tier 不决定 type。
