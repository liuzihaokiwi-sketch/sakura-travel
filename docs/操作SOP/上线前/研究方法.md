## 文档第一性原理

这份文档回答"**怎么对一个新目的地做深度研究**"(国家/大洲/圈/主城)。
- 不回答"怎么采集单条数据"(看 `04_操作SOP/数据采集.md`)
- 不回答"怎么写模板"(看 `04_操作SOP/模板写作.md`)
- 不回答"某个具体地区的特有规则"(看对应地区层)

读完应该能从零开始,对一个新国家/圈/主城做出可支撑策展的完整研究——产出"**国情层 + 主城层 + 延伸层**"三层文件。

### 🔗 横向联动(研究过程中顺手做的事)

研究会产生两类"顺带的战果"——都直接可用于内容运营,不要浪费:

- **搜到他人爆款笔记**(opencli 小红书搜索时,看到 collects > 100 / likes > 500 的) → 填 `marketing/{地区}/爆款参考.md`(记结构、选题视角、可借鉴套路)
- **发现本地视角/痛点/小众洞察**(英文源有中文源没写过的,或中国游客独有痛点英文攻略不覆盖的) → 填 `marketing/{地区}/素材库.md`

研究不只产出模板决策依据,也产出**内容运营原材料**——两边都记,研究一次产品+运营双丰收。

## 研究的精髓(用户视角,唯一判断锚)

研究最终通过模板到达旅行者。判断一份研究做得好不好,看两种感受——

> **① 用户翻开手账本,觉得"这本书懂这个地方,不是 AI 拼的"。**
>
> **② 用户走一趟回来,觉得"我好像真的懂了这个国家",不是只打了几个卡。**

**第一句是下限**:研究够扎实 → 模板推荐不踩坑 / 文案不空洞 → 用户信任
**第二句是上限**:研究够深 → 模板触达"本地灵魂" → 用户不只是游客,还懂了点东西

### 两句话拆解

**"懂这个地方"**——
- 事实都对(数据真实性红线)
- 细节能支撑(营业时段、文化礼仪、交通真相)
- 有本地视角(中国游客看不到的那一层)

**"真的懂了"**——
- 节奏灵魂(这个国家怎么呼吸的)
- 文化底层(不只是建筑,还有气质)
- 痛点预判(中国游客独有的不适,英文攻略不讲的)
- 顶级操盘手逻辑(Black Tomato / Condé Nast 怎么卖这个国家)

### 两句话怎么用

每做一条研究笔记,问:"**这条能支撑'懂这个地方'还是'真的懂了'?两个都不支撑的话,这条是废内容。**"

- 支撑"懂":事实数据、规则、流程、硬约束
- 支撑"真的懂":节奏、痛点、文化底层、本地视角
- 两者都没 → 常识 / 水贴 / 百科抄录 → 删

### 质量验证

研究笔记交付时,**每条洞察必须能回答**:
- 这是从哪里搜到的?(URL 或平台 + 文章类型)
- 这是真实旅行者/研究者说的,还是 AI 从训练数据推断的?
- 如果是推断,停下来真搜,或标"未验证/待核实"

**每条洞察都通过这三问 = 研究做到精髓**。

### 所有研究铁律都是这两句话的展开

| 铁律 | 服务哪一面 |
|---|---|
| 数据真实性红线(AI 不写事实) | 下限:不让"看起来像真的"摧毁信任 |
| 中英文源交叉 | 上限:中文看痛点 + 英文看灵魂,两面都懂 |
| 先搭坐标系再做单国 | 上限:研究法国不先懂欧洲 = 研究出来是孤立知识 |
| 偷师顶级策展社 | 上限:Black Tomato 的 Feelings Engine 比 AI 更懂"怎么让用户记住" |
| 7 问深挖 | 下限+上限:逼你既有事实又有灵魂 |
| 每条洞察有来源 | 下限:防止推断写成事实 |
| 常识删掉 | 简洁:小红书都有的不算研究成果 |

## 一、研究前置:方法论

### 我们做的是什么

**信息整合 + 好的思考**,不是自己从头设计。

- 找市面上最好的方案(顶级策展社、旅行杂志、本地权威媒体、高赞旅行者游记)
- 提炼结构性洞察(节奏、灵魂、痛点、玩法差异)
- 用自己的判断力做筛选和组合

**不做**:
- 不从 AI 知识库直接产出(会生成"看起来像真的")
- 不做学术综述(我们不是期刊)
- 不做景点清单(那是攻略,不是研究)

### 7 条不可动摇的红线

**1. 数据真实性是第一红线**
所有事实性字段必须来自真实数据源。搜不到填 null,绝不编造。

- 餐厅评分/价格/坐标/营业时间
- 景点门票/免费时段/订票窗口
- 交通车程/班次/价格
- 季节性事件的具体日期

**自检规则**:每次要写"事实"前,停一秒问自己"我这条是真搜过的,还是从上下文推断的?"——如果是后者,要么真搜,要么标注"未核实",**绝不把推断写成事实**。

可信度分级:
- `verified` 2+ 个 P0/P1 源交叉验证 → 可上生产
- `cross_checked` 1 个 P0/P1 + 其他 → 可用,标注
- `single_source` 仅 P2 源 → 可用,标注
- `ai_generated` 无真实源 → **不可上生产**

**2. 先搭骨架,再填内容**
接到"研究 X 地区"任务时,第一步不是研究 X 本身,而是 **X 所在的上层坐标系**。

- 研究意大利 → 先看"意大利在欧洲的位置"(vs 西、法、德)
- 研究日本关西 → 先看"关西在日本的位置"(vs 关东、北海道)
- 研究秘鲁 → 先看"秘鲁在南美的位置"

没有坐标系的单国研究,产出会变成**孤立知识**,无法支撑跨国路线判断。

**3. 中英文源必须交叉**
单一语言源头产生**认知偏差**:
- 纯英文源:错过中国游客独有痛点(签证/退税/小偷/支付)
- 纯中文源:错过本地视角(英文高端攻略 / 小红书没写的深度)

**最小交叉标准**:
- 每个核心观点有 1 中文源 + 1 英文源背书
- 冲突时优先"本地源"(当地旅游局、本地博主)
- 小红书是"中国游客真实反馈"的金矿,但也是"跟风打卡"的重灾区,需判断

**4. 不打补丁,修根因**
遇到问题不要绕:
- 数据源打不开 → 换源,不编数据
- 小红书防盗链 → 查 Referer + UA 配置,不跳过
- 英文名不确定怎么翻中文 → 查官方中文名,不自创

**5. 不加用户没要的功能/字段**
研究文档专注研究,不要:
- 主动"顺便"做 SEO 分析
- 添加"未来可能用到"的字段
- 重复写已经在 CLAUDE.md 的内容

**6. 文档事实性和数据事实性同等严格**
"这个博客站 coverage 关西"如果是从上下文推断,不是真访问过,**就是假事实**。每条引用必须真看过。

**7. 产出的最终标准是"用户会不会退款"**
不是"我搜到了多少条",是:
- 真实旅行顾问看了觉得"对,这就是"的比例
- 用户真去了发现哪条不对、会退款的风险

## 二、研究的 5 个阶段

### Phase 0 · 定位(1 小时)

在开始具体研究前,先搞清楚:

**1. 这个地区在更大坐标系的位置**
- 属于哪个文化圈 / 大洲 / 气候带
- 和邻居国家的对比
- 在"一生旅行计划"里的位置(必去 / 深度 / 小众)

**2. 该地区的中文攻略饱和度**
- 小红书搜主城 + "攻略",看结果数量和质量
- 高饱和(日本/韩国/泰国):需要做"**比小红书深一层**"
- 低饱和(东欧/中东):需要做"**从零建立认知**"

**3. 你的护城河在哪**
- 英文源有但中文没有的 → 信息差武器(如西班牙 Paradores 网)
- 中国游客独有痛点英文不覆盖 → 痛点深挖(如扒手、退税、支付)
- 需要你"翻译文化底层"的 → 灵魂解读(如巴斯克不自认西班牙人)

**4. 产出什么文件**(先定文件清单)
- 国情层(1-4 份):节奏灵魂 / 顶级操盘手逻辑 / 中国人痛点 / 区域补完
- 主城层(5-15 份):按 Tier A/B/C 分级
- 延伸层(1-3 份):海岸 / 岛 / 小众区域

### Phase 1 · 国情层(2-3 天)

**先做"灵魂"和"痛点",不做"城市"**。

**必做 3 份**:
- **R1 节奏灵魂**:这个国家怎么呼吸的?节奏 / 作息 / 情绪基调 / 文化红线
- **R2 顶级操盘手逻辑**:Black Tomato / Condé Nast / 本地高端旅游公司怎么卖这个国家?偷师他们的设计思路
- **R3 中国人痛点**:签证 / 支付 / 退税 / 语言 / 骗局 / 小偷 / 厕所 / 楼层差 / 吃饭时间差

**可选 1 份**:
- **R4 区域补完**:R1 未覆盖的独立区域(如西班牙的 Galicia)

### Phase 2 · 主城 Tier 分层(3-7 天)

| Tier | 定位 | 典型天数 | 数量 |
|---|---|---|---|
| **A** | 王牌主城,行程必含 | 2-5 天 | 2-3 个 |
| **B** | 主城,大概率去 | 1-3 天 | 3-5 个 |
| **C** | 延伸主城 / 日归 | 1-2 天 | 5-8 个 |

每个主城按 **7 问深挖**:
1. **灵魂一句话**:这城到底是啥味道
2. **骨架**:1-3 日经典路线(具体时间分配)
3. **热门景点怎么玩对**:反游客陷阱 / 时段 / 角度
4. **小众 wild card**:小红书级金矿 5-10 条
5. **最佳时段 / 季节**:什么时候去最好 / 什么时候别去
6. **餐饮骨架**:本地人吃啥 / 游客餐厅怎么认 / 人均
7. **硬约束**:订票窗口 / 闭馆日 / 文化红线 / 禁忌

### Phase 3 · 延伸层(1-2 天)

海岸 / 岛 / 小众区域,按主题打包,不独立成主城。

### Phase 4 · 总索引 + 沉淀(半天)

- `_INDEX.md` 汇总所有文件 + 产品梯队 + 文案红线 + 硬约束清单
- 用户痛点的"实用页固定内容"(可直接印在手账本)
- 下阶段待做清单

## 三、数据源优先级(通用)

```
P0 (权威源)
- 官方旅游局(visit-{country}.com)
- 世界级排行(Michelin / Fodor's / Lonely Planet / Rick Steves)
- 当地权威媒体(El País / NYTimes / Guardian / 朝日新闻)
- Google Maps API(坐标/营业时间权威)

P1 (中国用户视角)
- 小红书(真实游客反馈 + 女性视觉金矿)
- 知乎长文(深度分析)
- 携程问答(订票/签证实操)
- 穷游锦囊(系统性)
- 马蜂窝(目的地深度)

P2 (参考源)
- 个人博客(The Culture Trip / Nomadic Matt / Earth Trekkers)
- 旅游杂志 online(Condé Nast Traveler / Travel + Leisure)
- 反游客陷阱专题站

P3 (兜底,不用于事实性数据)
- AI 知识库 / 结构化分类 / 推荐语
- Wikipedia 历史背景
```

**铁律**:事实性字段必须 P0-P2 验证。P3 只能用于结构、分类、背景故事。

**地区特定的数据源**(如日本的 Tabelog/楽天/一休)放 `japan/日本特有规则.md`。

## 四、搜索策略

### 4.1 维度框架先从综述文献入手

不要直接搜"体验有哪些维度"——AI 会直接从知识库回答。

**正确做法**:搜系统综述(systematic review)论文,从真实文献提取框架。

**有效关键词**:
- `"memorable tourism experience" dimensions framework systematic review`
- `tourism experience quality psychology research 2023 2024`
- `travel experience [具体方向] research`

**关键参考**:Kim et al. (2012) MTE 七维度(279 篇文献综述确认)是目前最有学术共识的框架:享乐 / 刷新 / 本地文化 / 意义感 / 知识 / 参与 / 新奇。新维度研究要在这个基础上找补充,不是另起炉灶。

### 4.2 旅行者深度内容:找"去了才懂"而非攻略

**不要搜**:
- "关西旅游攻略"
- "京都必去景点"
- "大阪美食推荐"

**要搜**:
- `[城市] travel mistakes locals vs tourists`
- `[城市] second visit what I learned`
- `[城市] authentic experience beyond [著名景点]`
- `"后悔没早知道" [城市]`
- `[城市] 当地人视角`
- `[城市] 去了才知道`

**平台优先级**:
- 英文:旅行者个人博客 > reddit r/JapanTravel > japan-guide.com 论坛 > Tripadvisor 论坛
- 中文:知乎长文 > 马蜂窝游记 > 小红书深度帖(不是图片帖)
- 排除:旅游局官网、OTA 推广内容、泛泛攻略清单

### 4.3 区分两类问题,用不同策略

| 问题类型 | 可以用 AI 知识 | 必须真搜 |
|---|---|---|
| 逻辑性规则(为什么这样排) | 部分可以 | 需要真实旅行者验证 |
| 具体数据(班次/时间/价格) | 不可以 | 必须找一手来源 |
| 心理机制(体验为什么好) | 不可以 | 找学术文献 |
| 常识性建议(早起人少) | 不需要搜,直接排除 | — |

### 4.4 迭代搜索:每轮之间停下来判断

**第一轮**:宽泛关键词,找方向
**停下来问**:哪些是常识(删掉)?哪些是真盲区(保留)?哪些方向还没覆盖?

**第二轮**:针对盲区缩小关键词,找具体内容
**停下来问**:每条洞察有来源吗?逻辑自洽吗?能直接影响装配决策吗?

**第三轮(如需要)**:针对需要验证的具体数据补搜

### 4.5 识别"AI 生成冒充搜索结果"的信号

以下情况**高度可疑,需要追问来源**:
- 给了具体数字但没有 URL(如"30-40%"、"14:40 末班")
- 逻辑太完整、太整齐,像是从常识直接推导
- 洞察方向和搜索关键词高度重合(说明没真正找到意外发现)
- "多位旅行者反馈"但没有具体链接

## 五、7 问深挖格式(每个主城必答)

### 1. 灵魂一句话
**公式**:(核心锚点)+(文化关键词)+(感官体验)

- 巴塞罗那:"高迪奇幻建筑 + 加泰设计之都 + 地中海阳光"
- 京都岚山:"竹林 + 渡月桥 + 禅意庭院,关西优雅的缩影"

### 2. 骨架(具体到小时)

```
Day 1 · 上午(9-12):锚点 A(理由)
       · 下午(14-17):锚点 B(理由)
       · 晚上(20-22):餐饮 + 夜活动
```

**每个时段必须说明"为什么这个时段"**(光线/人流/营业/文化节奏)。

### 3. 热门景点怎么玩对(反游客陷阱三步)

- 热门地点的"正确玩法"(时段/角度/服装/订票)
- 游客陷阱是什么(具体名字,不泛泛)
- 本地人的同类替代

**示例(巴塞罗那 La Rambla)**:"避开,除了 La Boqueria 上午 11 点前去"

### 4. 小众 wild card

- 小红书级金矿 5-10 条
- 每条必有:**位置 + 为什么值得 + 怎么玩 + 人流量 + 季节限制**
- 不是"评分最高的",是"朋友带你去的那种"

### 5. 最佳时段 / 季节

- 什么时候去最好(月 + 气候 + 人流)
- 什么时候别去(梅雨/假期/节庆爆满)
- 樱花/红叶等浮动日期 → 不写具体日期,写"参照 JMA/koyo.walkerplus 当年预测"

### 6. 餐饮骨架

- 本地人吃啥(主食/街头/家常)
- 游客餐厅怎么认(位置/装修/菜单语言/店员态度)
- 人均(本地档 / 中档 / 高档 各一例)

### 7. 硬约束

- 订票窗口(几周前开始)
- 闭馆日(定休/年末/维修)
- 文化红线(禁穿和服 / 禁拍 / 禁入)
- 禁忌(温泉纹身 / 电车声音 / 小费)

## 六、体验维度参考框架(学术锚)

Kim et al. (2012) 的 **MTE 七维度**(279 篇文献综述),可作为判断"一个体验是不是好体验"的框架:

1. **Hedonism 享乐** — 感官愉悦
2. **Refreshment 刷新** — 脱离日常
3. **Local culture 本地文化** — 接触地方
4. **Meaningfulness 意义感** — 与自我/他人联结
5. **Knowledge 知识** — 学到东西
6. **Involvement 参与** — 亲身动手
7. **Novelty 新奇** — 第一次体验

**用法**:一个模板/一条动线/一个活动,能同时打中 2-3 个维度 = 好;只打中 1 个 = 平庸;一个都打不中 = 废。

**补充维度(近年研究)**:
- **Time perception 时间感知**(slow down psychology)
- **Cognitive load 认知负荷**(itinerary decision fatigue)
- **Peak-end rule 峰终定律**(Kahneman)
- **Place attachment 地方依恋**(短期旅客也会产生)
