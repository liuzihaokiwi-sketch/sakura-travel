## 一、装配是什么

用户付 ¥298 买的是"专家替我想过的成品行程"·不是"工具拼装"。

装配引擎要做的事：
1. **读用户表单**（季节/天数/人群/预算/偏好）
2. **匹配预制方案**（plans/）—— 多日方案选 1 个
3. **按方案的每天槽位**装：
   - 选模板（templates/{城市}/{动线}/）
   - 挑餐厅（restaurants/{城市}/{区域}/）
   - 挑停留点（stops/{城市}/{区域}/）
   - 挑酒店（hotels/{城市}/{区域}/）
4. **应用装配硬规**（跨日去重 / 跨城互斥 / 节奏调整）
5. **输出完整行程** → 渲染成 60 页手账本

## 二、行程矩阵覆盖（user-facing 视角）

行程矩阵 = "我们的产品覆盖哪些用户场景"。3 个维度组合：

### 2.1 archetype 维度（用户人群×节奏）

主流 archetype（关西样板·其他圈类比）：
- 情侣慢节奏
- 闺蜜出片重
- 家庭友好（亲子）
- 默认（朋友/单人）
- 特种兵密度

每个 archetype 有自己的"决策偏好":
- 情侣 → 高级餐厅 + 出片机位 + 体验型酒店
- 闺蜜 → 古着+咖啡+甜品多 + 出片
- 家庭 → 互动度高 + 不排长队 + 早睡早起
- 特种兵 → 高密度景点·砍弹性段
- 默认 → 平衡

### 2.2 季节维度

主流季节窗口（每地区不同·关西样板）：
- 樱花（3/底-4/初）
- 红叶（11/中-12/初）
- 春秋常规
- 夏祭
- 冬蟹/雪
- 黄金周/年末年始（旺季）

季节决定：
- 模板可用性（夜樱模板只在樱花季可装）
- entity 季节看点（红叶名所只红叶季有意义）
- 旺季加价（酒店价格段上浮）

### 2.3 天数维度

主流天数：3 / 5 / 7 / 9 日（关西样板）

天数决定：
- 包不包含日归延伸（5 日以上才推奈良/姬路）
- 包不包含温泉一泊（7 日以上才推有马/城崎）
- 主场天数分配（京都+大阪比例）

### 2.4 矩阵规模

理论上：5 archetype × 6 季节 × 4 天数 = **120 套方案**

实际不是 120 份独立 plans·而是：
- 主骨架 ~10 套（plans/）
- 装配按 archetype/季节/天数动态调整变体

## 三、装配运行时硬规（草案）

> 这段还在迭代·关西样板正在沉淀。

### 3.1 跨城互斥

- **showcase 餐厅每城 ≤ 1·全程 ≤ 2**（餐厅池容量可超·装配执行硬顶）
- 多出的 showcase 降 high 档或不推

### 3.2 跨日去重

- 同 entity 不在不同天重复出现（除非 slot 注明"D1 漫游 / D2 黄昏 peak-end"用途不同）
- 同菜系连两天避免（如不连推两天怀石）
- 同 type stops 不连两天（不连推两天古着街）

### 3.3 节奏三类（关西样板·已有）

详见 `japan/kansai/assembly/engine.md`：
- adaptive 默认（按用户出门档 8/9/10 平移）
- fixed_early（出片早起·一行程 ≤ 1）
- deep_stay（温泉/宿坊·一行程 ≤ 2）

### 3.4 退化路径

数据池缺口时怎么降级：
- 餐厅 showcase 满 2 → 降 high
- 用户勾"想体验"但 experience 酒店不可订 → 推荐 city 高档
- 该 entity 临时关闭 → 找替代 entity 同类型

### 3.5 跨地区化学反应（特化层）

每个圈有自己的"跨城序列规则"：
- 关西 7 化学反应（先京都后大阪 / 奈良作为桥 / 姬路+神户一日 等）→ `japan/kansai/这是什么.md`
- 欧洲特定（待）
- 中国特定（待）

通用层只定"装配引擎要消费这些规则"·具体规则在地区特化层。

## 四、装配跟数据采集的关系

**装配视角的硬规·部分指导采集**（不是全部）：

| 硬规 | 在哪里写 | 采集时怎么用 |
|---|---|---|
| 餐厅 4 档比例 25/50/25 | [数据池构建/餐厅规范.md §三.4](数据池构建/餐厅规范.md) | 采集时铺池目标 |
| 餐厅区域基因 | 餐厅规范 §三.5 + 地区特化文档 | 采集时按基因偏向 |
| 餐位需求分级 4 类区域 | 餐厅规范 §三.6 | 采集时定 full 池目标量 |
| showcase 跨城互斥 ≤2 | 本文 §三.1 | 池子 5-8 家·不滥采 |
| 酒店 4 档 + city/experience/luxury | 数据池构建/酒店规范.md | 采集时按真实档位 |

**装配运行时硬规**（跨日去重 / 节奏调整 / 退化路径）→ 不指导采集·只在装配时执行。
