# 内容运营 + 客服策略

> Phase 9·上线后启动。
> 本文是**策略层**——讲气质 / 节奏 / 路径决策。
> 客服日常操作（看板·触发表·异常 SOP·FAQ）见 [客服应对.md](客服应对.md)。

---

## 第一性原理

**运营和客服都是产品的延伸·不是产品之外的东西**。

用户买的是「**有人替我想过 + 敢对结果负责**」的确定感。运营让没买的人看到这份确定感·客服让买了的人感受到这份确定感。

如果运营文案变成「网红打卡 9 个必去」·客服话术变成「按规定一轮修改已用完」·**这两件事都让产品掉价**——它本质是把产品降级成「免费攻略 + 标准客服」。

**判断锚**:运营 / 客服的每一句话·让用户感觉到的是「**这家懂行 + 真心对你**」·还是「**又一个流量号 + 又一家电商**」?

## 一、内容运营策略

### 1.1 矩阵分工

| 平台 | 受众 | 内容定位 | 节奏 |
|---|---|---|---|
| **小红书** | 25-40 岁女性·主力客群 | 真心推荐·懂行女博主气质·攻略 + 生活方式 | 每周 2-3 篇 |
| **抖音** | 25-45 岁泛旅游人群 | 视觉冲击·真人翻手账·情绪 + 价值感 | 每周 1-2 条 |
| **B 站** | 深度玩家 / 极客 | 长图文 / 解说·策展逻辑 / 行业批判 | 每月 1-2 条 |

**当前由女朋友主理小红书 / 抖音矩阵**·B 站作为差异化补充。

### 1.2 内容养号原则

- **不蹭热点**·只做「这趟旅行你应该知道的事」
- **不做对比测评**·不踩同行（专业感来自「我们考虑了什么」·不来自「他们差在哪」）
- **不接广告**·呼应 [项目定位 §独立](../../项目核心/项目定位.md) 不和任何商家合作的承诺
- **真人出镜 / 真本子展示**·每条内容都有「实物 + 实拍」·区别于 ChatGPT 攻略号
- **本地视角优先**·素材库的「懂当地朋友才知道」放运营顶位（[marketing/{圈}/素材库.md](../../../marketing/)）

### 1.3 内容素材来源

```
模板写作过程     ─┐
数据采集发现     ─┼─→ marketing/{圈}/素材库.md（沉淀池）
pilot 反馈金句   ─┘                     │
                                       ↓
                              小红书 / 抖音 / B 站脚本
```

详见 [模板写作.md §联动](../上线前/模板写作.md) 和 [数据采集.md](../数据采集.md)·所有写模板/采数据时挖到的本地知识都要实时沉淀。

### 1.4 爆款研究

每篇运营内容前·先看 [marketing/{圈}/爆款参考.md](../../../marketing/)·学结构不抄内容。
判断爆款标准:**collects > likes**——收藏说明真有用·点赞可能只是好看。

## 二、客服策略（不是 FAQ·是气质）

> FAQ 和操作流程在 [客服应对.md](客服应对.md)·这里讲**客服该用什么气质对话**。

### 2.1 客服角色定位

> **客服不是「执行规定」·是「让用户觉得被认真对待的那个人」**。

机器拒绝「一轮修改已用完」·客服可以说「这种情况我们破例给您安排」。客服工作的 90% 是**系统做不到的那 10% 的温度**。

### 2.2 三条气质硬规

#### ① 不踢皮球
用户问的事·客服自己解决到底。需要后台/数据团队配合的·客服自己跑腿·**用户感受到的是「一个人」不是「一个流程」**。

#### ② 不官腔
不说「按公司规定」「您这种情况不在范围」「请您理解」。
说「您这事这么处理」「我帮您看下」「我去问下数据团队」。

#### ③ 留温度
- 用户出发前 3 天主动关心一次（不是话术·是真问）
- 用户回来后聊一句体验（这是 [pilot反馈](pilot反馈.md) 入口）
- 用户拍小红书 @我们·跑去评论一句（运营 + 客服联动）

### 2.3 升级处理路径

遇到客服无法当场解决的·按以下路径升级·**不让用户重复说一次问题**:

```
客服 → 数据团队（事实数据问题）
     → 模板团队（节奏/动线问题）
     → 印刷厂（实物质量问题）
     → 顺丰客户经理（物流问题）
     → 运营负责人（重大投诉/媒体相关）
```

每次升级·客服自己复述用户问题给团队·不让用户重新讲。

### 2.4 老客户激活

老客户去新地区·享 8 折（D14 后续）。客服在以下时机主动激活:

- 老客户咨询新地区时（直接说「您是老客户·这次有 8 折」·不让用户问）
- 新地区开放时主动私信老客户名单（节制·一年 1-2 次）
- 老客户发小红书带标签时·客服评论致谢 + 透露新地区计划

### 2.5 反馈/二次转化链路

```
回来后 3-7 天 [客服]:
  ↓ 发起 diary 采集 → [pilot反馈.md]
  ↓ 收集最满意 / 最失望 / 推荐意愿
  ↓
最满意的安排   → marketing 素材库
最失望的安排   → 反馈池分类处理
会推荐的形容   → 小红书脚本测试
  ↓
6 个月后[运营]:
  → 老客户名单·新地区开放主动通知
  → 老客户带新客 → 双方各得 ¥30 红包（待定）
```

## 三、运营 + 客服 + 反馈的闭环

```
运营吸引       ───→ 用户下单
                    ↓
                  客服承接
                    ↓
                  系统生成
                    ↓
                  客服审核·确保「钱花对了」
                    ↓
                  实体交付·客服关怀
                    ↓
                  用户旅行
                    ↓
                  客服 diary 采集 → [pilot反馈.md]
                    ↓                    ↓
                  反馈分类处理         素材沉淀回运营
                    ↓                    ↓
              改 entity / 模板 / 池    新一轮内容产出
                    └─────────┬───────────┘
                              ↓
                          下一波用户
```

**这个闭环跑起来·产品就有了真正的护城河**——不是字段设计·不是技术·是「专家策展 + 用户验证 + 持续迭代」的复利效应。
