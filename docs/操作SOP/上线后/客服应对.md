## 文档第一性原理

这份文档回答"**客服遇到具体情况时该怎么做**"(后台操作、订单异常、用户追问、修改请求、发货问题)。
- 不回答"业务流程整体长什么样"(看 `03_数据契约/业务流.md`)
- 不回答"表单设计规范"(看 `03_数据契约/业务流.md`)
- 不回答"运营/内容策略"(看 `运营选题.md`)

读完应该能:
- 遇到一个具体订单异常,知道先查哪里、如何推进
- 遇到用户追问,知道什么能改什么不能改
- 遇到数据/生成问题,知道是走人工审核还是回到表单

## 精髓(唯一判断锚)

> **客服的角色不是"执行规定",是"让用户觉得被认真对待的那个人"**。

同样一件事(比如用户要改酒店),机器可以拒绝"一轮免费修改已用完",客服可以说"这种情况我们破例给您安排"。客服工作的 90% 是**系统里做不到的那 10% 的温度**。

判断客服动作好不好:
- 用户挂断电话 / 消息断开时,他会不会推荐给朋友?
- 用户的问题是不是被"踢皮球"到另一个部门?(不该踢的要独立解决)
- 客服的每一次回复,是不是让用户感到"这家店的人靠谱"?

## 一、后台订单管理

**后台地址**:`{DOMAIN}/admin`(需登录)

### 订单看板(7 列对应生命周期)

| 列 | 含义 | 客服操作 |
|---|---|---|
| 新建 | 用户刚提交 | 可创建详细表单 |
| 表单中 | 等用户填写详细表单 | 等待(可微信提醒) |
| 待生成 | 表单已提交,等排队 | 点"生成"触发 |
| 生成中 | AI 正在跑 | 等待,可查 Trace |
| 待审核 | 生成完需人工审 | 进 Review 审核 |
| 已发布 | 审核通过 | 可查看/下载 PDF |
| 已交付 | 用户已收到 | 完成 |

### 关键操作

**触发生成**:
1. 确认详细表单已提交(状态="待生成")
2. 点击"生成行程"
3. 系统入队 `generate_trip`
4. 在 Trace 页面追踪进度

**手动推进状态**:仅异常情况(系统卡住)。点击"推进状态" → 选择目标 → 确认。

## 二、用户主动消息(客服发起)

用户提交表单后,客服按**表单字段触发器**发送对应消息(不在前端展示):

| 触发字段(表单) | 客服发送 |
|---|---|
| 屏 2 风格=亲子感 或 屏 4 未声明"不想去 USJ",且天数 ≥ 7 | USJ 门票方案对比(普通/快速/VIP),含当日人流预测 |
| 屏 4 文本框/补充项含"和服",或风格=出片感/约会感 | 京都和服店推荐(3 家不同风格),含预约链接 |
| 预算页 吃饭档位 ≥ 好好享受 | 需提前预约的餐厅清单,客服可代订 |
| 天数 ≥ 6 且风格 ∈ {经典版, 出片感, 约会感} | 嵯峨野小火车抢票时间和技巧,旺季可代抢 |
| 预算页 住宿档位 ≥ 享受型 或 屏 4 提及温泉/宿坊 | 推荐房型对比 + 注意事项 |
| 所有订单(默认) | 关西周游卡 vs ICOCA vs JR Pass 的个性化建议 |

**触发时机**:用户提交后 1-2 小时内(在系统生成期间,不要等方案出来才发)。

## 三、跟进节点

| 时间点 | 动作 | 话术方向 |
|---|---|---|
| 表单提交后 30min | 预期管理 | "您的攻略大约 2-4 小时内生成" |
| 攻略生成完 | 推送通知 | "您的攻略已生成,请凭专属码 XXX 查看" |
| 用户看完前置层 | 主动问 | "觉得怎么样?有想调整的地方吗?" |
| 发货 | 物流单号 | "您的手账本已发出,顺丰 3-5 天到" |
| 到货 | 关怀 | "收到了吗?第一眼感觉怎么样?" |
| 出发前 3 天 | 提醒 | "准备好了吗?有什么要帮忙的?" |
| 回来后 3-7 天 | 回访 | 问 3 个问题:最满意/最失望/会不会推荐朋友 |

## 四、异常处理 SOP

### 4.1 生成失败(进人工队列)

**症状**:"生成中"卡超过 2 小时,或直接变"待审核"但质量分低。

**处理**:
1. 查 Trace(后台 `/admin/trace/{id}`)看具体哪一步失败
2. 常见原因:
   - 用户表单硬约束冲突(如预算太低 + 酒店档位太高)→ 联系用户调表单
   - API 限流 → 等 10 分钟重试
   - Opus 返回异常格式 → 转人工审核,客服手动装配
3. 无法自动恢复 → 推到 Review 审核队列,编辑手动完成

### 4.2 方案质量差(用户不满意)

**症状**:用户看完前置层觉得"一般""不对味"。

**处理**:
1. 启动"一轮免费修改"(AI 辅助对话)
2. 常见修改类型:
   - 调节奏/密度 → 可直接改
   - 换酒店区域 → 可直接改
   - 换餐厅偏好 → 可直接改
   - 增减某类体验(博物馆/购物/户外) → 可直接改
3. **不在修改范围内**(需二次下单或客服定制):
   - 完全换地区
   - 改天数
   - 超预算档位升级

### 4.3 预约失败(餐厅/USJ 等抢不到)

**症状**:推荐的餐厅预约不上,或 USJ 快速通关没抢到。

**处理**:
1. 提供 1-2 个同等级备选
2. 如果备选都约不上 → 走客服定制线,用户确认后手动替换
3. 不要让系统"默默替换"——必须经用户确认

### 4.4 发货问题

- **物流卡住** → 联系顺丰,24 小时内给用户反馈
- **损坏/丢失** → 免费补寄,不追究责任
- **寄错地址** → 补寄到正确地址,运费公司承担

## 五、用户常见追问 FAQ

| 追问 | 应答方向 |
|---|---|
| "我能改日期吗?" | 生成前可以(撤回表单重填);生成后需重新下单 |
| "能加一天吗?" | 需补差价(¥50/天左右)+ 重新生成 |
| "我已经订了酒店,能不改酒店吗?" | 走客服定制线,把酒店作为硬约束传给装配 |
| "我上次买过,这次打折吗?" | 老客户下次去新地区享 8 折 |
| "PDF 能发我电子版吗?" | 已发布状态后可提供下载 |
| "没收到手账本" | 先查物流 → 补寄 |
| "照着走发现餐厅关门了?" | 致歉 → 当场提供备选(客服微信即时响应) → 记录 issue 反馈给数据团队 |
