# 项目定位

> 项目哲学唯一权威·新 AI / 新人入场第一份必读·读完应能在灰色判断时调用。
> 最后更新:2026-04-25

---

## 对用户的口号

> ### **一本为你写好的旅行手账**
> **旅行时带着走·旅行后留作回忆。**

所有用户可见文字（手账正文 / 文案 / 客服话术 / 小红书运营）的语气基线·都呼应这两句。

---

## 第一性原理

项目必须同时满足四件事:

1. **内容来自专家策展与本地人视角**·不做信息堆砌。
2. **交付物必须是一本精美、可随身翻阅的纸质手账**·而不是一份普通行程单。
3. **用户体验必须是「照着走就对了」**·尽量少思考、少二次决策、少踩雷。
4. **结果不只要好用**·还要让这趟旅行在结束后变成值得保存和反复翻看的回忆。

### 核心承诺

四件事最终都是为了兑现对用户的一句承诺:

> **「有人替我想过 + 敢对结果负责」**

这就是对外口号「**为你写好 + 留作回忆**」的内部翻译——
- 「为你写好」= 有人替我想过（对应四件事 ①②③）
- 「留作回忆」= 敢对结果负责（对应四件事 ④）

任何决策回到这句:这件事是不是在加强这份承诺?

## 给用户带来的价值

**用户的处境**:

> 你想去关西——
> 打开小红书·30 个攻略不知道哪个能信。
> 吃什么·去哪住·哪天去哪个景点·全要自己拼。
> 不想花一整周做攻略·又不想到了现场踩雷。

**这就是我们解决的事**·靠三条价值兑现:

- **品质**——专家策展 + 本地人视角写出来的·不是 AI 拼贴·不是搜索引擎扩写。精美的纸质手账让每一页都值得保存。
- **确定性**——用户不用自己比较、纠结、判断哪条攻略可信。**照着走就对了**·我们替你做完所有判断·并对结果负责。
- **独立**——**不和任何商家合作**·不收恰饭·不接返佣·不做品牌植入。每一家餐厅 / 酒店 / 景点的推荐·都是因为它真的好·不是因为它给我们钱。

60 页纸质手账本 + 贴纸 DIY 包。关西样板间为主战场·其他地区圈按关西节奏复用。

## 目标用户

**在乎旅行品质的任何人。**

不挑年龄·不挑预算档·不挑同行人。共同特征是:

- 不愿意把旅行变成「打卡 + 排队」
- 不想花一整周做攻略·但要求结果有品味
- 愿意为「不踩雷 + 有惊喜 + 实体收藏」付一份钱

## 我们不是什么·避免误认

新 AI 写文案 / 客服回话术时·避免把产品搞混:

| 不是 | 真正区别 |
|---|---|
| 小红书攻略 / 知乎清单 | 攻略要用户自己挑·我们是**写好的成品** |
| 携程 / Klook / OTA | 他们卖**预订**·我们卖**一本书** |
| 旅行社 / 团游 | 团游集体走固定路线·手账是**个性化但有专家把关** |
| ChatGPT 行程单 / AI 攻略 | AI 行程单 30 秒能做·我们是**专家策展 + 本地人视角 + 实体交付** |
| 私人定制 / 旅行管家 | 定制按你说的来·策展是**替你想过 + 敢负责** |

## 产品边界·什么不做

以下场景**不走主产品·走客服定制**:

- **二刷某地区用户** → 要小众深度·模板是大众广谱
- **奢华游**（酒店 ¥3000+/晚）→ 走定制线
- **已预订酒店的用户** → 酒店反向约束动线·主产品装不了
- **预约失败场景**（任天堂没抽中等）→ 客服处理

**原则**:服务 < 10% 特殊用户的功能默认走客服·主产品保持标准化简洁。

**也不做**:
- 不做社交、不做工具、不做 SaaS
- 不做「让用户自己拼」的界面
- 不做免费攻略扩写——只做「专家策展 + 实体交付」

## AI 工作时的判断锚

灰色判断时·按这几句对自己问:

1. **这件事在加强「替我想过 + 敢对结果负责」的承诺吗?** —— 第一性原理「核心承诺」的反向自检·任何决策的起点
2. **真专家会不会点头?** —— 质量验收的唯一标准·不是「跑通不报错」
3. **我是不是在让用户做选择?** —— 策展产品 ≠ 定制工具·替用户决策才是付费价值
4. **这是事实还是我的推断?** —— 推断标注「未核实」·绝不写成事实
5. **这场景该不该走客服?** —— < 10% 特殊用户默认走客服·不塞主产品
